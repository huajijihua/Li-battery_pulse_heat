## 1. 来龙去脉

本项目是车载低温脉冲加热系统工程建模工作区，位于 `E:\agentwork_codex_0509`，是 git 仓库（远端 `https://github.com/huajijihua/Li-battery_pulse_heat.git`，分支 `main`）。

项目分两个阶段：

- **M4（已冻结）**：CMI（Combined Module Invert，文献原称）架构下的开环低频 d 轴脉冲电流自加热路线。M4 完成了通用 Reference/Platform 基线、可审计电气/热账本、参数敏感性和控制策略比较，识别到**电池侧有效电流过低**的架构性卡点（电机侧 Id=40A，电池侧 I_batt RMS 仅约 5.5A）。M4 模型文件夹 `车载脉冲加热技术simlink模型/` 已于 2026-07-22 冻结为只读参考基线，不在其中做新开发。
- **M5（当前阶段）**：dSVPWM（dead-zone Space Vector PWM，死区空间矢量脉宽调制）换流型脉冲自加热路线。M5 保持与 M4 相同的首版硬件边界（单包、三相三线、标准两电平逆变器、PMSM、DC-link），改用死区换流重构电池侧正负交变电流形成机制。M5-00（文献调研、分级、评估、接口契约）已完成；**M5-01 已准入但尚未启动，本指令即为正式启动授权**。

M5 不是脱离 M4 的全新课题，而是沿同一条电池-DC-link-逆变器-PMSM-热响应链条，在控制层改变电流形成机制，验证电池侧电流路径能否被控制层重构。

## 5. M5-01 具体实施计划（建议步骤）

以下步骤按"小步闭环"执行，每步完成后读回验证再进入下一步。

### 步骤 1：基线分析与迁移规划

- 用 `model_overview` + `model_read(depth=0/1)` 读取 M4 冻结基线 `车载脉冲加热技术simlink模型/02_模型/pulse_heating_official_spine_v04.slx`，记录 root 六模块结构、`System_Audit` 支撑层、`Connection Label` 测量骨架（`I_batt_meas/V_batt_meas`）、`BatteryDC=[V_batt,I_batt,T_pack,SOC]`、固定窗口账本（`Compute_DCLink_Subledger_V4H`、`Energy_Balance_Audit`、`SystemKPI`）和实际门极路径。
- 对照 `M5_v2_准入与接口契约_v01.md` 第 2/3 节，明确哪些物理路径、审计口径和接口迁移，哪些 M4 控制方式不迁移。
- 产物：一份迁移清单（可写入本指令文件附录或新建 `M5-01_执行记录_v01.md`），记录保留/排除项。

### 步骤 2：v2 模型骨架建立

- 在 `车载脉冲加热仿真平台_v2/02_模型/` 建立 v2 唯一 `.slx`（命名建议 `pulse_heating_v2_platform_v01.slx` 或沿用项目约定）。
- 首版结构以 M4 六模块边界为基础复制：电池、DC-link 与逆变器接口、PMSM、FOC 控制器、热管理、System_Audit。
- **控制方式暂留空或占位**：M5-01 阶段不实现 dSVPWM，只迁移物理路径和测量/账本接口；控制层可保留 M4 的 FOC 电流环骨架但不接 M4 的低频 `Id_ref` 脉冲发生器，或以常量占位，等待 M5-02 替换。
- 保存后执行 `model_check`，修复所有结构 warning。

### 步骤 3：测量与账本接口读回

- 迁移 M4 的 `I_batt_meas/V_batt_meas` 测量骨架、`BatteryDC`、`Compute_DCLink_Subledger_V4H`（固定窗口 2e-6 s 采样、`RMS_Window_s=0.01 s`）、`Energy_Balance_Audit`、`SystemKPI(44/45)` 账本字段语义。
- 保持 M4 功率正方向、固定窗口口径和未知项传播规则（未获来源的电池包线、风险代理、局部损耗输入显式保持 `UNKNOWN/NaN`）。
- 编译模型，运行一个短时 Reference 工况（如 0.04 s），确认：
  - root 和关键子系统 `healthy`
  - 实际六路门极配对 `(1,2)/(3,4)/(5,6)` 严格互补（即使控制层是占位）
  - `BatteryDC=[V_batt,I_batt,T_pack,SOC]` 可读回
  - 完成窗口 `[0.03,0.04] s` 的账本字段可读回，未完成窗口保持 `NaN/0`

### 步骤 4：M5-01 完成判据核验

逐项核验：
- [ ] v2 唯一 `.slx` 存在且结构 `healthy`、可编译
- [ ] M4 测量/账本接口已迁移且可读回
- [ ] 未引入 M4 的 CMI 开环低频 d 轴脉冲控制
- [ ] 未把 M4 短时账本或 CMI 卡点升级为 v2 结论
- [ ] 参数来源标签完整，设备参数缺口保持 `UNKNOWN/PLACEHOLDER`

核验通过后，M5-01 完成，可进入 M5-02（dSVPWM 控制最小闭环）。
