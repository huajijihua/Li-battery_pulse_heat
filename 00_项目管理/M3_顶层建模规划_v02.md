## 2. 顶层研究目标与完成图景

M3 的顶层目标不是完成某一个可运行工况，也不是只把现有 V4 修到无 warning。M3 要形成一个通用低温脉冲加热系统集成设计与参数匹配模型，使研究者能够在 Reference / Platform 工况下回答三类根问题：

1. **凭什么能发热**：低温脉冲加热的能量来源、能量路径、电气激励、设备损耗和热源分配是否能够用统一系统模型解释。
2. **发的热是否合理**：电池、DC-link、高压路径、逆变器、PMSM、机械边界和热网络之间的能量、储能、损耗、排热和残差是否可审计，是否符合参数量级和物理约束。
3. **怎么控制发热**：电流参考、波形、频率、PWM/采样、限幅、降额、保护和候选路径选择如何影响发热能力、设备应力、热分配和系统边界。

顶层完成图景是：

| 维度 | 完成后应具备的能力 |
|---|---|
| 系统集成设计 | 能在一个统一主骨架中表达电池、高压路径、DC-link、逆变器、PMSM、控制器、机械边界和热管理的责任关系 |
| 参数匹配 | 能说明电压、电流、功率、热、频率、采样、保护和机械约束之间是否形成共同参数包线 |
| 发热机理 | 能区分电池内部损耗、逆变器损耗、PMSM 铜耗/铁耗、高压路径损耗、储能变化和未建模项 |
| 控制策略 | 能比较不同参考、波形、频率、限幅和保护策略对发热、损耗、热源分配和约束触发的影响 |
| 设计建议 | 能为后续设备参数选择、实验方案、控制策略开发和工程化实施提供方向性建议，但不直接输出产品定型或安全放行结论 |

因此，M4-00/M4-01/M4-02/M4-03 不是整个项目的全部目标，而是服务上述顶层目标的近期能力建设路线。

## 3. 总体研究路线

M3 顶层研究路线按“系统设计 -> 参数匹配 -> 机理审计 -> 策略比较 -> 工程建议”的逻辑推进：

1. **系统设计空间定义**：固定通用系统边界和候选路径接口，保留可替换电池、高压路径、逆变器、PMSM、控制器、热管理和机械边界。
2. **官方资产与机理证据对齐**：优先复用 MathWorks 官方案例和 Simulink 模块级资产；官方资产不能覆盖时，采用文献或明确物理化学原理方程；经验等效模块只作为最后手段。
3. **首轮集成主线建立**：以电池-DC-link-逆变器-PMSM 零速/锁止电流激励路径作为首轮主线，其他路径保留为插件接口和反例审计。
4. **参数基本匹配**：先确认电气、控制、热、机械四类关键匹配是否能形成共同包线，避免单独优化某一类参数。
5. **能量与热源审计**：形成能量路径与热源分配账本，判断发热来自哪里、是否合理、哪些项受参数和控制支配。
6. **敏感性与策略研究**：在账本语义稳定后，研究主导参数、边界条件和控制策略对发热效果及约束触发的影响。
7. **工程化指导建议**：输出参数选择、数据需求、实验设计、控制策略和后续模型保真度提升建议。

## 7. 近期实施次序与重构条件

近期实施次序服务于顶层研究目标，不代表整个项目只有四个步骤：

1. 资料检索与官方案例复用复核（M4-00）。
2. 在现有 6 大模块内部做最小必要改造，先补齐接口、语义和审计信号。
3. 形成系统集成设计及参数基本匹配后的能量路径与热源分配账本（M4-01）。
4. 在账本稳定后，再进入参数敏感性分析（M4-02）。
5. 最后进入控制策略比较（M4-03）。

只有在满足下列条件之一时，才考虑对顶层 6 模块进行重构：

- 现有架构无法表达首轮候选加热路径。
- 现有架构无法完成能量路径和热源分配审计。
- 现有架构导致参数身份、接口方向或责任边界混乱。
- 官方案例复用要求调整模块边界，且调整后更清晰、更可验证。
- 重构能显著降低后续验证、参数替换或证据追溯成本。

当提出重构时，必须同时给出：

1. 相比现有 6 模块架构的具体优势。
2. 需要修改的模块、接口和责任域。
3. 对 M4-00/M4-01/M4-02/M4-03 的影响。
4. 增加的验证成本和追溯成本。
5. 不重构时会造成的明确缺陷。

## 8. 工况与结论边界

当前阶段只使用以下三类工况：

1. Structural / Interface：检查模块、端口、单位、方向、连接和信号维度。
2. Blocked / Contract：验证配置契约不满足时为何不能运行，例如关键字段仍为 `PLACEHOLDER`。
3. Reference / Platform：使用 V4-REF、官方案例参数、文献量级或工程默认值，用于机理走通、能量路径分析、参数敏感性和控制策略比较。

在缺少明确设备参数和实验数据前，不进入 Active Engineering Configuration，也不进入 Validation / Comparison。Reference / Platform 结果只能作为技术机理、参数趋势、策略影响和工程化实施建议，不能作为设备能力或产品级结论。

### 9.1 M4-01：系统集成设计及能量账本

M4-01 的目标是把“系统集成设计及参数基本匹配”转化为可审计的模型结果。它不是单独做一个能量统计模块，而是要求 V4 六大模块在统一端口方向、统一参数身份和统一热源归属下，能够解释首轮主路径为什么能发热、热在哪里产生、哪些能量进入储能项、哪些损耗或残差仍未建模。

M4-01 的前置条件为：

1. M4-00 已完成官方资产复用矩阵和模型影响清单，并明确哪些结构直接复用、哪些只作参考、哪些需要物理方程补齐。
2. 当前 V4 的 7 条 warning 已进入台账，能直接修复的 warning 应修复；暂不影响 M4-01 账本和接口语义的 warning 必须形成可审计保留理由。
3. 首轮主路径固定为电池-DC-link-逆变器-PMSM 零速/锁止电流激励路径，其他候选路径只保留接口和反例审计。
4. 工况等级限定为 Structural / Interface、Blocked / Contract 和 Reference / Platform，不进入工程配置或验证比较结论。

M4-01 的建模重点为：

| 重点 | 指导要求 |
|---|---|
| 系统接口 | 明确电池、高压路径、DC-link、逆变器、PMSM、控制、机械锁止和热网络之间的端口方向、单位和信号含义 |
| 参数基本匹配 | 检查电压、电流、功率、热、频率、采样、保护和机械边界是否能形成同一 Reference / Platform 包线 |
| 发热机理 | 区分电池内部损耗、逆变器损耗、PMSM 铜耗/铁耗、高压路径损耗、热网络输入和未建模项 |
| 机械边界 | 明确零速/锁止、反作用转矩、机械功率和允许微动的口径，避免把锁止转矩误记为机械能输出 |
| 审计输出 | 每个账本字段必须有来源、单位、采样口径、适用工况和是否可用于结论的标记 |

M4-01 的账本最低字段包括：

| 字段族 | 必须覆盖的内容 |
|---|---|
| 电池 | 电池端功率、电池内部损耗、电池热源和储能变化 |
| DC-link | DC-link 端口功率、电容储能变化和瞬态能量项 |
| 高压路径 | 连接压降、等效 R/L 损耗和高压路径热源 |
| 逆变器 | DC 输入、AC 输出、导通/开关或等效损耗、逆变器热源 |
| PMSM | 电磁输入、铜耗、铁耗、机械功率、机械储能和电机热源 |
| 热网络 | 热源输入、热储能、排热和环境/冷却边界 |
| 审计 | 未建模项、能量残差、单位、采样口径、来源和适用工况 |

M4-01 的输出应至少包括：

1. 主路径能量流向图或等价结构说明。
2. 六大模块与账本字段的映射表。
3. 热源分配表，区分电池、逆变器、PMSM、高压路径和热网络。
4. 参数基本匹配表，说明哪些参数属于 Reference / Platform，哪些仍为 `PLACEHOLDER` 或数据缺口。
5. warning 台账处理结果。
6. M4-02 和 M4-03 能够直接复用的指标接口。

M4-01 的通过标准为账本可解释性，不设置产品级残差、温升或效率阈值。通过的含义是：能量项完整、正方向一致、储能/损耗/热源分清、残差定义可复算、未建模项明示。M4-01 不输出产品级发热性能、安全边界、寿命、析锂、NVH 或量产可行性结论。

### 9.2 M4-02：参数敏感性分析

M4-02 的目标是在 M4-01 账本语义稳定后，识别低温脉冲加热系统集成设计中最关键的参数和约束。它服务于参数匹配和工程化实施建议，不是参数拟合，也不是全局优化。

M4-02 的前置条件为：

1. M4-01 已固定账本字段、端口方向、热源归属和残差定义。
2. 待扫描参数均有来源身份和允许范围；若没有真实设备数据，只能采用官方案例、文献量级或平台参考范围。
3. 每次敏感性分析必须说明所属工况等级，默认只在 Reference / Platform 工况下解释趋势。
4. 扫描不得改变系统边界、候选路径接口或账本口径。

M4-02 的参数族范围为：

| 参数族 | 关注内容 |
|---|---|
| 电池 | OCV、等效阻抗、温度/SOC 依赖、允许电流、热容、热阻和初始状态 |
| DC-link 与高压路径 | 电容、初始电压、连接 R/L、传感位置、压降和路径损耗 |
| 逆变器 | 电压/电流能力、调制裕度、开关或等效损耗、采样/PWM 相关参数和保护限制 |
| PMSM | 相电阻、电感、磁链、极对数、铜耗、铁耗、机械惯量、锁止反力和热参数 |
| 热网络 | 部件热容、热阻、冷却边界、环境温度和排热路径 |
| 控制与保护 | 电流参考、频率、波形、限幅、降额、停机条件和采样时间 |

M4-02 的分析方式以局部灵敏度、单因子扫描和少量关键联动对比为主。优先回答：哪个参数决定能否形成目标电流路径，哪个参数主导热源分配，哪个参数最容易触发电压/电流/热/机械/保护限制，哪个参数最值得后续实测或供应商数据确认。

M4-02 的输出应至少包括：

1. 参数分组和来源身份表。
2. 扫描范围和不能扫描的原因。
3. 对能量账本、热源分配和控制约束最敏感的参数列表。
4. 主导限制因素排序。
5. 后续设备参数获取优先级。
6. 对 M4-03 控制策略比较的参数固定建议。

M4-02 的通过标准是能够形成可追溯的参数敏感性结论和工程建议。它不要求找到最优参数，不做参数拟合，不声称某一参数组合代表真实设备，也不把平台参考趋势升级为产品级设计值。

### 9.3 M4-03：控制策略比较

M4-03 的目标是在同一系统边界、同一账本口径和可追溯参数范围内比较不同发热控制策略对能量路径、热源分配、设备应力和约束触发的影响。它服务于“怎么控制发热”，不是量产控制器标定。

M4-03 的前置条件为：

1. M4-01 已提供可复用的能量账本和热源分配指标。
2. M4-02 已明确需要固定的关键参数、可扫描参数和主要约束边界。
3. 每个控制策略必须共享同一端口方向、同一热源定义和同一保护/降额口径，否则不允许直接比较。
4. 策略比较仍限定在 Reference / Platform 工况，不输出量产控制性能结论。

M4-03 的策略族范围为：

| 策略族 | 比较重点 |
|---|---|
| 电流参考 | d/q 参考、幅值、相电流约束、目标电池端电流和电机侧电流的对应关系 |
| 波形与频率 | 脉冲、正弦、方波或其他候选波形；频率与电池阻抗、逆变器开关、PMSM 损耗和采样关系 |
| 限幅与降额 | 电池、逆变器、PMSM、热网络和机械锁止限制如何削弱或中止发热 |
| 候选路径插件 | 主路径与其他候选路径接口的可比性、反例路径的能量闭合检查 |
| 保护与退出 | 软限制、硬限制、故障、停机和复位条件对能量账本和热源分配的影响 |

M4-03 的比较指标应至少覆盖：

1. 电池端功率、电池内部热源和电池侧约束触发。
2. 逆变器损耗、PMSM 损耗、高压路径损耗和热源占比。
3. DC-link 储能波动和电压/电流裕度。
4. 机械锁止反作用、机械功率口径和失锁风险标记。
5. 能量残差、未建模项和账本可解释性。
6. 控制可实现性、限幅频率、降额触发和停机原因。

M4-03 的输出应至少包括：

1. 策略集合和对比条件。
2. 各策略的能量账本和热源分配差异。
3. 主要约束触发原因和裕度排序。
4. 对系统集成设计和参数匹配的控制侧建议。
5. 不建议继续推进的策略及其排除原因。

M4-03 的通过标准是能够基于同一账本口径比较策略趋势，并给出后续工程化实施建议。它不要求选出唯一最优策略，不做量产控制标定，不评价真实 ECU 软件权限，也不替代后续台架或实测验证。
