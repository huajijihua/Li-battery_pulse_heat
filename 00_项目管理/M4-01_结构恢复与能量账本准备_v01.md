## 1. 本轮目标与边界

本轮目标是恢复并审计低温脉冲加热系统首轮主路径：

```text
电池物理端 -> HV 路径/DC-link -> 三相逆变器 -> PMSM -> 机械边界与热网络
```

本轮只做活动 V4 内的最小结构恢复和接口读回，不改变六模块顶层边界，不运行工程工况，不形成设备、安全、析锂、寿命或量产结论。

## 4. 当前接口语义审计

当前 `blk_27` 已有逆变器侧 DC 测量候选：

| 量 | 当前测点 | 当前解释 |
|---|---|---|
| `I_dc` | `blk_591 DC_Current_Sensor`，位于 `DC_plus` 到逆变器正端内部支路 | 逆变器侧 DC 电流候选，不等同于电池支路电流 |
| `V_dc` | `blk_592 DC_Voltage_Sensor`，跨逆变器正负 DC 端 | 逆变器侧 DC 电压候选，不等同于电池端电压 |
| `i_phase` | PMSM/电流检测链和 `blk_182 From_i_abc` | 电机相电流，必须单独定义峰值、平均值和窗口 RMS |
| `I_batt` | `blk_761 Battery_Current_Sensor`，串入 `blk_20` 的 HV 正路径 | 电池支路电流候选；方向和单位仍需模型编译/视觉复核，不得用 `I_dc` 或 `i_phase` 替代 |

`blk_20` 保留一个仅作兼容发布的 `BatteryDC_legacy_non_authoritative` 输入，并在内部直接 Terminator；它不参与电压、电流或功率映射。`InverterDCFeedback[1]` 经选择器输出为 `V_dc_meas`，供驱动和 KPI 使用。`BatteryLoad` 的旧电池输入、`blk_20` 输出、选择器和 root Terminator 均已删除；电池物理端口是唯一能量路径，`BatteryDC` 只保留为 `[V_batt,I_batt,T_pack,SOC]` 发布向量，不能暗示信号级功率传递。

### 4.2 DC-link 测点和剩余语义

此前直接对 `blk_20` conserving port 进行多分支重连时曾出现结构回退；改用 `Connection Label` 后，当前四组节点和传感器路径已闭合。读回关系为：

- `blk_685 Compute_I_rms_V4G` 已是实际时间窗口算法，`blk_666 Energy_Balance_Audit` 已改为未知项感知账本。

### 4.3 窗口 RMS、账本与 KPI 字段

`blk_685` 的输入为 `I_batt`、`I_dc`、`i_phase`、`I_heat_equiv`、实际时间 `t` 和 `window_s`。它在窗口内使用梯形积分；窗口未完成、输入无效或窗口长度无效时，相关 RMS 保持 `NaN`。`I_heat_equiv` 当前无权威生产者，明确接入 `kpi_value_unknown`，不得与电池、母线或相电流混同。

`blk_666` 的账本将电池端功率、DC 输入功率、HV 损耗、DC-link 储能、热储能、排热和高压热源分开。`P_charge`、`P_discharge` 为独立的非负量；当所需字段已知时，`residual_dc_link = P_batt_terminal - P_hv_loss - P_dc_link_storage - P_dc_input`。缺少任一必要来源时，残差和依赖字段保持 `NaN`，`ledger_valid=0`。

`SystemKPI` 的 `u1-u24` 保持原映射；新增字段如下：

| 端口 | 字段 |
|---|---|
| `u25-u28` | `I_phase_rms`、`RMS_Window_Elapsed_s`、`RMS_Window_Complete`、`RMS_Window_s` |
| `u29-u35` | `P_batt_terminal`、`P_dc_input`、`P_hv_loss`、`P_dc_link_storage`、`E_dc_link`、`DeltaE_dc_link`、`DeltaE_batt` |
| `u36-u44` | `P_loss_known`、`P_loss_total`、`P_mech`、`P_thermal_storage`、`P_heat_rejection`、`Q_hv_gen`、`P_unmodeled`、`residual_dc_link`、`Energy_Ledger_Valid` |

### 4.4 `System_Audit` 支撑层与候选公式边界

`System_Audit` 在 root 上没有 Simulink 接口端口；其三个子系统经既有 `Goto/From` 标签接收测量量或发布审计字段。因此 root 画布上不存在该支撑层的主路径连线是设计结果，不是待修复的连接缺陷。

- `RMS_And_Measurement_Audit` 只负责区分和统计 `I_batt`、`I_dc`、`i_phase`、`I_heat_equiv`；最后一项仍无权威生产者。
- `Energy_And_Thermal_Audit` 提供字段框架和局部损耗/热估算接口。未有来源的字段保持 `NaN`；当前不保留任何未准入的局部系数或经验损耗数值，因此不形成物理闭合结论。
- `Battery_Envelope_Screening` 只保留能力与风险审计接口，输出保持 `UNKNOWN/NaN`。它不替代化学体系、阻抗、温度梯度、SOC、充电状态或实验验证，不得用于设备限流、安全或析锂判定。

## 5. 恢复安全记录

本轮发现此前使用含糊范围的清除操作会扩大影响，因此采用以下恢复纪律：

1. 不再使用 `disconnect "? -> ?"` 或未限定 scope 的物理清除。
2. 每次 `model_edit` 只作用于一个明确 scope，失败或 `partial` 后立即 `model_read` 与 `model_check`。
3. 先用实际端口 ID/端口号确认连接，再进行定向断开和重连。
4. 结构恢复后立即保存，并用 root 和关键子系统双重检查。
5. 视觉快照只作为结构审查的第二证据，不把画布连通误判为参数或物理正确。
