## 1. 本轮目标与边界

本轮目标是恢复并审计低温脉冲加热系统首轮主路径：

```text
电池物理端 -> HV 路径/DC-link -> 三相逆变器 -> PMSM -> 机械边界与热网络
```

本轮只做活动 V4 内的最小结构恢复和接口读回，不改变六模块顶层边界，不运行工程工况，不形成设备、安全、析锂、寿命或量产结论。后段实施的一次 `20 ms` Reference 短时仿真仅用于核对 PWM 门极、FOC 坐标与基本电流闭环；它不是能量账本、热、设备能力或安全验证。用户已于 2026-07-17 授权在此基础上继续建立通用系统架构集成、Reference / Platform 参数契约和最小必要验证工况；参数来源可为官方案例、文献或工程默认值，但每项都须保留身份、单位、适用范围和替换影响。

### 4.3 窗口 RMS、账本与 KPI 字段

`blk_685` 的输入为 `I_batt`、`I_dc`、`i_phase`、`I_heat_equiv`、实际时间 `t` 和 `window_s`。它在窗口内使用梯形积分；窗口未完成、输入无效或窗口长度无效时，相关 RMS 保持 `NaN`。`I_heat_equiv` 当前无权威生产者，明确接入 `kpi_value_unknown`，不得与电池、母线或相电流混同。

`blk_666` 的账本保留电池端功率、DC 输入功率、HV 损耗、DC-link 储能、热储能、排热和高压热源的完整账本字段。`P_charge`、`P_discharge` 为独立的非负量；完整 `ledger_valid` 继续要求电池储能、热储能、排热和高压热源等字段，不因电气子账本有限而变为有效。

`E_dc_link` 与 `DeltaE_dc_link` 按能量状态和初值差定义；不使用连续 `Derivative`。`blk_888` 以 `2e-6 s` 实际时间样本和固定 `RMS_Window_s` 边界，对 `P_batt_terminal=V_batt I_batt`、`P_hv_loss`、`P_dc_input=V_dc I_dc` 进行梯形积分，对跨边界步长作线性插值，并以有限能量差计算 `P_dc_link_storage`。四项量和 `residual_dc_link=P_batt_terminal-P_hv_loss-P_dc_link_storage-P_dc_input` 始终来自同一已完成窗口；未完成或任一输入为 `NaN` 时均保持 `NaN`。

`SystemKPI` 的 `u1-u24` 保持原映射；新增字段如下：

### 4.4 `System_Audit` 支撑层与候选公式边界

`System_Audit` 在 root 上没有 Simulink 接口端口；其三个子系统经既有 `Goto/From` 标签接收测量量或发布审计字段。因此 root 画布上不存在该支撑层的主路径连线是设计结果，不是待修复的连接缺陷。

- `RMS_And_Measurement_Audit` 只负责区分和统计 `I_batt`、`I_dc`、`i_phase`、`I_heat_equiv`；最后一项仍无权威生产者。
- `Energy_And_Thermal_Audit` 现以 `blk_897 Compute_Reference_Thermal_Subledger_V4I` 补齐 M4-01C 的固定窗口 Reference 热账本；`blk_888` 继续独立闭合电池端到 DC-link/逆变器输入之间的电气子账本。两者都不构成设备热路径、热性能或安全结论。
- `Battery_Envelope_Screening` 只保留能力与风险审计接口，输出保持 `UNKNOWN/NaN`。它不替代化学体系、阻抗、温度梯度、SOC、充电状态或实验验证，不得用于设备限流、安全或析锂判定。

### 4.7 账本浮点边界修复（2026-07-21）

M4-02B 全量扫描首轮在 T 系列（200ms、`[0.16,0.20] s` 窗口）失败，表现为 `DeltaE_batt_AbsError=0.220955 J`、`P_dc_link_storage_AbsError=6.720807 W`、`Residual_dc_link_AbsError=6.720807 W`，而 `P_batt_terminal`/`P_dc_input`/`P_hv_loss` 仍吻合到 1e-13。

根因为模型 bug，不在运行器：`blk_888 Compute_DCLink_Subledger_V4H` 与 `blk_897 Compute_Reference_Thermal_Subledger_V4I` 用累加方式跟踪窗口边界 `window_end = t_window_start + active_window`，每次锁定后 `t_window_start = window_end`。经 4 次累加 `0.04`，浮点漂移使 `window_end = 0.2000000000000000111` 略大于 DigitalClock 在 `n=100000` 处的 `t = 0.19999999999999998335`，差 `2.78e-17 s`，导致 `t >= window_end` 判定为假，`[0.16,0.20] s` 窗口从未锁定。模型最后发布的 `P_dc_link_storage`、`DeltaE_batt` 仍是 `[0.12,0.16] s` 窗口的值，而运行器独立复算的是 `[0.16,0.20] s`，故比较的是不同窗口。

B 系列（`[0.04,0.08] s`，1 次累加，`t == window_end` 恰好相等）与 M4-01C（`window_s=0.01`、`[0.03,0.04] s`，3 次累加恰好对齐）不受影响，因此此前均通过。`blk_685 Compute_I_rms_V4G` 用 `window_start = t` 重置模式而非累加，无此 bug。

修复：两块 `if t < window_end` 改为 `if t < window_end - 1e-9`（1 ns 浮点容差，远大于漂移 2.78e-17，远小于 PWM 2kHz / Id 50Hz 任何动态）。锁定仍在最近的 2e-6 网格命中点触发，`alpha ≈ 1.0 ± 1.5e-15`，`E_boundary` 误差约 3e-14 J。对已正确锁定的窗口（B 系列、M4-01C）无行为改变。

小步闭环验证（保存、关闭、重载后 root 与 `blk_843` `model_check(all)` 均 `healthy`，`Dirty=off`）：
