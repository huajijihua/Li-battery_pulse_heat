## 4. 收口工作包与顺序

### M4-01B：窗口化电气子账本

目标是在不虚构电池/热数据的前提下，闭合已具备来源和物理路径的电气子账本。

1. 明确固定时间窗口 `[t1,t2]` 中 `P_dc_link_storage=(E_dc_link(t2)-E_dc_link(t1))/(t2-t1)` 的正方向、单位、适用条件和缺失数据处理。
2. 以同一窗口复算 `P_batt_terminal`、`P_hv_loss`、`P_dc_link_storage` 和 `P_dc_input`，并定义可复算的 `residual_dc_link=P_batt_terminal-P_hv_loss-P_dc_link_storage-P_dc_input`。
3. 将电气子账本有效性与完整热/能量账本有效性分开；电池储能和热字段没有来源时继续保持 `UNKNOWN/NaN`。
4. 对每个有限字段记录生产者、物理位置、符号、单位、窗口、参数来源和适用工况。

M4-01B 的通过条件是：受控 Reference / Platform 窗口内的 DC-link 储能功率与有限时间能量差一致，电气子账本残差可独立复算，且完整账本仍不会因未知项被误报为有效。

执行结果（2026-07-20）：通过。`blk_888 Compute_DCLink_Subledger_V4H` 以 `RMS_Window_s=0.01 s` 的固定时间边界，对 `V_batt*I_batt`、`P_hv_loss`、`V_dc*I_dc` 做实际时间的梯形积分均值，并以 `E_dc_link` 的有限差分给出 `P_dc_link_storage`。时钟采样时间为与测量信号一致的 `2e-6 s`；跨边界步长按线性插值拆分，避免窗口漂移或脉冲混叠。`SystemKPI.u29-u32/u43` 发布同窗口四项功率和残差，`u45=Electrical_Ledger_Valid`，原 `u44=Energy_Ledger_Valid` 保持完整账本语义。

在默认闭环 `0.04 s` Reference 工况的最后固定窗口 `[0.03,0.04] s`，模型输出 `P_batt_terminal=105.669282315 W`、`P_hv_loss=1.04430323048 W`、`P_dc_link_storage=-18.0257968322 W`、`P_dc_input=121.856391176 W`、`residual_dc_link=0.794384740542 W`、`Electrical_Ledger_Valid=1`、`Energy_Ledger_Valid=0`。以原始 `logsout` 独立复算五个量，绝对差均不超过 `1e-13`。未完成窗口的 `0.005 s` 工况保持 `P_dc_link_storage=NaN`、`Electrical_Ledger_Valid=0`、`Energy_Ledger_Valid=0`。这些是 Reference / Platform 电气路径审计事实，不是设备功率、热或安全结论。

### M4-01C：热源与完整账本

目标是以有来源身份的 Reference / Platform 参数补齐首轮主路径的热源、储能和排热语义。

1. 为电池低温阻抗/热源、HV 路径热源、逆变器导通/开关/死区损耗、PMSM 铜耗/铁耗以及各自热网络去向建立参数契约。
2. 明确电池、逆变器、PMSM、高压路径和热网络的热容、热阻、冷却/环境边界及适用范围；缺少可辩护来源的字段继续阻断完整账本。
3. 建立 `DeltaE_batt`、`P_thermal_storage`、`P_heat_rejection`、`Q_hv_gen` 和完整残差的统一符号与窗口口径。
4. 形成参数基本匹配表，区分 `OFFICIAL`、`LITERATURE`、`ENGINEERING_DEFAULT`、`PLATFORM_DEFAULT` 与仍阻断结论的 `PLACEHOLDER`。

M4-01C 的通过条件是：顶层规划第 9.1 节最低字段均有可解释状态，能量项完整、方向一致、储能/损耗/热源分开、残差可复算、未建模项明示；不设置产品级效率、温升或安全阈值。

执行结果（2026-07-21）：通过。`blk_897 Compute_Reference_Thermal_Subledger_V4I` 以 `0.01 s` 固定窗口发布 `DeltaE_batt`、`P_thermal_storage`、`P_heat_rejection` 和 `P_unmodeled`；`blk_647` 扩展为 Reference 导通/缩放开关/死区损耗，`blk_639` 将 `P_iron=0` 限定为锁止规则，`blk_666` 将 DC 边界残差与系统未建模项分开。`runM401CThermalLedger.m` 的 `0.005 s` 工况保持热字段 `NaN` 与两个有效标志 `0`；`0.04 s` 的 `[0.03,0.04] s` 窗口中，两个有效标志均为 `1`，`DeltaE_batt=-1.28540044107756 J`、`P_thermal_storage=-111.739818340343 W`、`P_heat_rejection=211.755632881919 W`、`P_unmodeled=46.7653709933744 W`。电池能量、DC-link 储能、热平衡、系统残差和 DC 边界残差的独立复算最大绝对差为 `2.27373675443232e-13`。详见 `M4-01_结构恢复与能量账本准备_v01.md` 和唯一 CSV `M4-01C_thermal_ledger_v01.csv`。

### M4-02B：账本闭合后的参数敏感性

在 M4-01C 后复用或重跑 M4-02A 中的电气物理项，并把电池低温阻抗、热容/热阻、冷却边界、逆变器损耗、保护限制及其来源范围纳入受控扫描。输出必须覆盖能量账本、热源分配、约束触发、主导限制因素和数据获取优先级，而非仅报告代理变化。

执行结果（2026-07-21）：通过。运行器 `runM402BCompleteLedgerSensitivity.m` 完成 26 工况（B 系列 19 个 40 ms + T 系列 7 个 200 ms），全部到达 StopTime、门极严格互补、闭环有效、零速、`EnergyLedgerValid=1`、`ElectricalLedgerValid=1`，全部独立复算 AbsError 与热平衡误差 < 1e-10（最大 5.68e-13），`DeviceProtection_Evaluable=0`。首轮 T 系列失败暴露 `blk_888`/`blk_897` 浮点窗口边界 bug（4 次累加 0.04 后 `window_end` 比 DigitalClock `t` 大 2.78e-17 s，`[0.16,0.20] s` 窗口未锁定），已修复为 `if t < window_end - 1e-9`（详见 `M4-01_结构恢复与能量账本准备_v01.md` 4.7 节），修复后回归 B00/M4-01C 无变化。主导参数：`DCLink_C` 对母线纹波与电池 RMS 最敏感，`Rs` 对铜耗最敏感，`HVPath_R` 仅改 HV 损耗系数。T 系列发现短窗口热时间常数简并（`τ=C_inv*Rth_inv` 相同则热响应相同，C_inv/Rth_inv 不可独立分辨）。`T_batt` 全部保持 263.15 K。详见 `M4-02_参数敏感性分析计划_v01.md` 第 8 节和唯一 CSV `M4-02B_complete_ledger_sensitivity_v01.csv`。

### M4-03B：账本闭合后的控制策略比较

在 M4-01C 和 M4-02B 后，以同一端口方向、热源定义、保护/降额口径比较频率、波形、幅值、限幅、降额和停机策略。

执行结果（2026-07-21）：通过。运行器 `runM403BControlComparison.m` 完成 11 工况（C00-C06 频率/幅值/占空比偏置 + C07-C10 限幅/保护模式/停机），全部到达 StopTime、门极严格互补、闭环有效、零速、`EnergyLedgerValid=1`、`ElectricalLedgerValid=1`，全部独立复算 AbsError 与热平衡误差 < 1e-10，`DeviceProtection_Evaluable=0`。详见唯一 CSV `M4-03B_control_comparison_v01.csv`。

关键结果：

- **频率趋势**（与 M4-03A 一致）：25 Hz 跟踪误差最低（11.45 A）但电池端功率最高（102.3 W）；100 Hz 跟踪误差最高（17.96 A）但电池端功率最低（98.3 W）。该相互制约关系在完整账本下仍成立，不支持基于本轮选出"最佳频率"。
- **幅值趋势**：20 A 至 60 A 时，`P_cu` 从 9.57 W 升至 68.48 W，`P_inv` 从 26.51 W 升至 67.19 W，热排散从 215 W 基本稳定——能耗随幅值平方增长，热排散受固定环境/冷却边界限制。
- **限幅与停机策略**：C07-LimitId_Low（LimitId=20）、C08-LimitDuty_Low（LimitDuty=25）、C09-LimitMode_Active（LimitMode=3）、C10-LimitStatus_Off（LimitStatus=0）四组工况与 C00_Base 结果一致，说明当前模型中 `LimitConfig` 块为被动监控层，不主动约束脉冲参考。`ControlSaturation_Any=0` 在所有工况保持。
- **账本一致性**：所有工况 `P_thermal_balance_Error_W < 1e-10`，模型输出与独立复算最大绝对差 1.4e-12。

结论：M4-03B 以共同完整账本和保护口径完成了频率、幅值、占空比偏置和限幅/停机策略比较。保护/限幅配置在当前 Reference 路径中为被动信息层，不改变控制行为。所有结论限定为 `Reference / Platform`，不升级为设备控制策略、频率选型或安全结论。
