### 3.2 当前 V4 的关键读回事实

| 事实 | 读回证据 | M4-00 解释 |
|---|---|---|
| V4 根层把 `Battery_pos/Battery_neg` 物理连接到电驱 `DC_plus/DC_minus` | 活动 V4 root `model_overview(full)` | 存在物理端口连接，但不能据此称为已建成 DC-link |

| 电驱有 `DC_plus/DC_minus` 物理端口、DC 电压/电流传感器和 `InverterDCFeedback` Mux | `model_read(scope=blk_27, depth=1)` | 具备测量接口候选，但 `I_dc/P_dc` 的有效性和方向仍需 M4-01 复核 |

## 6. 首轮主路径结论

首轮主路径的推荐复用组合为：

```text
OF-BAT-01 电池包/冷却结构
    -> OF-HV-01 官方物理电气元件构成的高压路径与 DC-link
    -> OF-DRV-01 三相逆变器/PMSM/热/机械结构
    -> OF-MCB-01 的 Control_System 子系统
    -> V4 PulseHeating_Command_And_Limits 与 KPI_And_Logging 薄层
```

该组合是“官方资产候选拼接 + V4 工程接口适配”，不是把三个官方顶层模型直接级联。当前复核没有发现可以直接覆盖以下完整边界的单一官方模型：

```text
电池包 -> 受控高压路径/DC-link 储能 -> 逆变器 -> PMSM -> Park/Lock
```

因此，M4-01 必须把 DC-link 物理闭合、端口方向、测量位置、储能变化和高压路径损耗作为独立修改对象；不能因为 `PMSMDriveThermal` 采用电池直连而把它当作 V4 DC-link 已完成的证据。

## 7. 参数身份纪律

本矩阵中的官方示例参数只能按以下方式进入后续配置：

| 情形 | 允许身份 | 不允许的解释 |
|---|---|---|
| 直接来自官方模型或初始化文件 | `OFFICIAL`，`identity_scope=official_example` | 具体目标设备参数、产品额定值 |
| 从官方模型提取单位、端口或参数关系 | `OFFICIAL`，`verification_status=结构读回/文档比对` | 已验证 M3 设备能力 |
| 用于建立自洽的平台参考组合 | `PLATFORM_DEFAULT`，须另行给出组合逻辑 | 伪装成官方设备参数 |
| 官方示例未覆盖的低温脉冲保护或机械边界 | `PLACEHOLDER` 或 `ENGINEERING_DEFAULT` | 从快充、热驱动或历史案例推断通过 |
| 历史 M2/V4 参数 | `HISTORICAL_CASE` 或 `V4-REF` | 自动升级为 M3 默认配置 |

特别注意：官方电池和驱动初始化文件中的所有数值、拓扑、时间设置、损耗与热边界均是官方示例身份。它们可以支撑参数字段和适用范围设计，不能直接填入 M3 Active 配置，也不能用于反推当前电池拓扑、平台电压等级或控制硬件角色。

## 8. 复用决策与重构判断

### 8.1 当前推荐决策

1. 保留 V4 六模块顶层骨架。
2. 允许在六模块内部采用官方子系统和官方物理块的最小适配。
3. 采用 `OF-BAT-01 + OF-DRV-01 + OF-MCB-01` 作为 M4-01 首轮主候选。
4. 将 `OF-HV-01` 作为必须补齐的官方物理块候选，而不是声称已有完整官方 DC-link 复用资产。
5. 将 `OF-DRV-02`、`OF-DRV-03`、`OF-BAT-02` 作为参考或反例，不改变首轮主线。

### 8.2 本轮未触发顶层重构

M4-00 尚未发现必须改变六模块顶层边界的证据。当前问题集中在 `DC_Link_And_Inverter_Interface` 内部物理表达、跨模块接口语义、参数身份、机械锁止和 KPI 账本字段，原则上可先按最小内部改造处理。

如果 M4-01 证明 DC-link 物理闭合、机械边界或能量账本无法在现有边界内表达，必须另行提交重构提案，说明新增边界、责任域、验证成本和不重构缺陷。
