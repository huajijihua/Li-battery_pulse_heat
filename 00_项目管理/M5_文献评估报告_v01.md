## 1. 执行结论

### 1.1 文献不是同一层级的路线

10 份 PDF 分析文件提供的是不同类型的证据：

1. **控制层证据**：2024templock 与 2024 IEEE TTE 解释如何通过 dSVPWM（dead-zone Space Vector PWM，死区空间矢量脉宽调制）换流在无新增硬件条件下形成电池侧交流电流，并提供温度/电压控制结构。
2. **电热模型证据**：2026bit NPBH 与 Zhang 2015 提供频率相关阻抗和产热模型，但 NPBH 同时引入了中性点开关拓扑。
3. **安全判据证据**：Jian 2023 提供负极电位、端电压和边界频率组合判据，解决“有热不等于安全”的缺口。
4. **基线/架构证据**：2021 iScience 解释 CMI 与 DMSI 的电流路径和性能差异，帮助定位 M4 卡点。
5. **工程替代和辅助证据**：BYD 四线制、CN679、CN173 和 ATIC 分别提供整车替代拓扑、RMS 标定闭环、开环对照和第三方方法论。

因此，控制、热模型、安全判据和拓扑不能写成 A/B/C/D 四条互相并列的首版路线。M5 应采用分层路线：先确定硬件边界，再选择控制机制，再接入电热模型和安全判据，最后评估硬件拓扑替代。

### 1.2 M5 首版路线选择

**首版主线：单包、三相三线、标准三相两电平逆变器和 PMSM 的无新增硬件车载电驱路径，在此硬件边界内采用 dSVPWM（死区空间矢量脉宽调制）换流型脉冲自加热控制。**

- **首版控制主参考**：2024templock + 2024 IEEE TTE。
- **首版热模型主参考**：Zhang 2015 的频域热生成/集总热平衡结构，结合 2026bit NPBH 的二阶 ECM 形式和双频产热分解思路。
- **首版安全主参考**：Jian 2023 的负极电位判据与 TVCS/NEPCS 组合框架；由于当前电池化学体系和原件页码数据不完整，先建立可审计的候选判据路径，满足数据资格后才允许安全标志变为有效。
- **首版控制对照**：CN114789679A 母线电流 RMS 闭环。它不与 dSVPWM 主环同时作用于同一调制执行量。
- **远期拓扑评估**：2026bit NPBH、2025 BYD 四线制、2021 iScience DMSI。三者不进入首版默认硬件。

### 1.3 两篇 Claude 深度分析文献的作用

用户指出 `2024templock_dSVPWM温度锁定_文献分析.md` 和 `2026bit_NPBH驱动电路重构_文献分析.md` 由 Claude 模型深度分析，质量明显高于其他 DeepSeek Flash 初析文件。本报告保留这一事实，并将其体现在证据权重中：

- `2024templock` 设为 **A1 核心主参考**，因为它既有全文视觉核验，又直接适配首版无硬件改动控制层。
- `2026bit NPBH` 设为 **A2 核心备选与模型参考**，因为其分析完整、公式和拓扑已视觉复核，且频域电热模型对 v2 很有价值；但其新增中点开关和电池包拆分使它不能直接成为首版 CMI 拓扑。

分析模型来源不能单独替代原件证据。Jian 和 Zhang 虽然初始分析深度低于两篇 Claude 深度分析，但由于分别承担安全判据和热模型的关键职责，仍列为 A2；两篇原件视觉核验现已完成，数值仍受单电芯、正弦波、化学体系和目标参数缺失等适用性门槛约束。CN173、ATIC 不能因包含仿真方法或析锂字样而升级为核心安全依据。

## 3. M4 卡点与文献证据

### 3.1 M4 卡点的正确定位

M4 的活动 V4 是标准电池-DC-link-逆变器-PMSM 路径，正式路线为 **CMI（Combined Module Invert，文献原称）架构下的开环低频 d 轴脉冲电流自加热路线**。在零速锁止下，电机相电流主要表现为电感储能和局部无功交换，DC-link 电容进一步隔离高频分量，导致电机侧电流与电池支路 RMS 电流不等价。M4 已独立读回电池 RMS、DC-link 纹波、门极互补和电气/热账本字段，电池侧有效电流过低是可追溯的 Reference / Platform 卡点。

### 3.2 2021 iScience 的定位

2021 iScience 的 CMI 模式与 M4 同源，可用于解释为什么同一类标准驱动电路方案的电池支路电流受电机绕组、PWM 矢量和回路阻抗限制。其 DMSI 模式通过电池包拆分和继电器改变回路，不能把 DMSI 的 4C 或温升速率视为 CMI 的控制改进结果。

### 3.3 dSVPWM 的关键价值

2024 IEEE TTE 和 2024templock 的共同价值不在于某个固定频率，而在于 dSVPWM 换流机制：利用死区矢量和补偿矢量，在标准驱动硬件内形成正负母线电流分量。其控制对象分别偏向电压峰谷或温度锁定，不能直接把电机相电流、母线电流和电池支路电流混为同一个量。

这正好对应 M4 的卡点：首版 v2 应先改变电池侧交流电流形成机制，再讨论更长时热响应和安全边界；不能继续只扫描低频 `Id_ref` 的幅值、频率和占空比来推断车载加热能力。

## 4. 关键技术矛盾及处理决定

### 4.1 dSVPWM 与 CN679 RMS 闭环不是两个可叠加主路线

- dSVPWM 使用零矢量/死区/补偿矢量时序、`bn` 或 `d0` 调节换流强度。
- CN679 使用 `Ud_req`、`Id_ini` 前馈和 RMS 电流闭环。
- 两者如果都直接修改同一 PWM 调制层，控制权会冲突，无法把两套 PI 简单串接成“更完整”的控制器。

**决定：**首版使用 dSVPWM 作为主调制机制；电压峰谷闭环作为快速约束，温度锁定作为较慢监督层；CN679 仅作为独立的 RMS 闭环对照工况或后续替代控制器，不与主控制器同时闭环。

### 4.2 最优频率不能从单篇文献直接选定

文献给出了 1 Hz、50 Hz、125 Hz、300-400 Hz 和 10 kHz 等不同频率，原因包括：

- 电池化学体系和 SOC 不同；
- 单电芯、模组、整车和电驱母线观察量不同；
- 正弦 AC、低频脉冲和 PWM 换流波形不同；
- 有些文献以温升最大为目标，有些以电压或析锂边界为约束。

**决定：**M5 不冻结一个通用最优加热频率。控制开关频率、加热电流包络频率和电池阻抗评价频率分别记录；用 `Z(T,SOC,f)` 和安全判据共同扫描频率，文献的 50/125/300-400 Hz 只作为候选点。

### 4.3 频域热公式不能无条件套用于 PWM 脉冲

Zhang 2015 和 Jian 2023 的产热公式主要基于正弦交流；NPBH 给出了驱动低频和加热频率双频叠加的工程近似，但其二阶 ECM 是全电池阻抗，不能直接产生负极电位。

**决定：**首版电热模型必须标出波形假设。至少提供以下两条审计路径之一：

1. 对 PWM/脉冲电流做基波及主要谐波分解，按各频率的 `Re(Z)` 累加产热；或
2. 使用时域电流和已定义的阻抗/等效热源接口校核频域近似。

若没有波形等效校核，频域热结果只能标为平台估算，不能作为安全或设备温升结论。

### 4.4 析锂判据与全电池 ECM 不是同一个模型

Jian 2023 的 NEPCS 需要负极阻抗或负极电位信息；NPBH 的二阶 ECM 主要用于全电池阻抗和热源计算。不能由一个二阶全电池 ECM 自动推导负极电位并宣称安全。

**决定：**v2 安全层分为三个状态：

- `UNKNOWN`：没有合格的负极参数或化学体系映射，禁止形成安全结论；
- `CANDIDATE`：已运行负极电位/端电压组合判据，但仍依赖文献比例或平台假设；
- `EVALUABLE`：参数来源、温度/SOC 适用范围、波形等效和独立验证条件均满足，才允许 `DeviceProtection_Evaluable=1`。

M5 首版应完成 `CANDIDATE` 路径和拒止逻辑，不能因为脚本输出了负极电位代理就自动进入 `EVALUABLE`。

## 5. M5 分层路线选择

### 5.0 正式路线命名

- **M4**：CMI（Combined Module Invert，文献原称）架构下的开环低频 d 轴脉冲电流自加热路线。
- **M5 首版**：dSVPWM（dead-zone Space Vector PWM，死区空间矢量脉宽调制）换流型脉冲自加热路线。
- M4 的“开环”仅限定低频 `Id_ref` 加热参考层，不表示底层电流跟踪环全部开环；M5 的 `d` 表示 `dead-zone`，不是 M4 的 d 轴。
- 详细机理、控制差异和 `bn/d0/a0` 建模边界以 `M5_文献调研与差异分析_v01.md` 第 8 节为准。

### 5.1 层 0：硬件边界

**选择：首版保持 M4 的单包三相三线标准三相两电平逆变器 + PMSM 边界，不新增电池中点、N 线或开关。**

理由：

- 能直接复用 M4 已验证的物理接口、官方 PWM、FOC、门极互补检查和账本基础设施。
- 可以把性能变化归因于控制机制和电池模型，而不是同时引入硬件架构变化。
- 符合用户当前授权的通用系统集成范围，也避免把具体设备电池包拓扑当作默认事实。

NPBH、BYD 四线制和 DMSI 作为“硬件替代层”保留，不进入首版默认路径。它们只有在首版控制改进无法满足目标，或用户明确需要架构比较时进入后续同一 v2 模型的参数化拓扑研究。

### 5.2 层 1：控制和调制

**选择：dSVPWM（死区空间矢量脉宽调制）为首版主控制机制。**

首版控制结构按快慢层次组织：

1. FOC/机械边界层：保持转矩和速度控制接口，零速锁止作为基线工况，行驶工况作为后续扩展。
2. dSVPWM 调制层：实现标准 SVPWM 的非零矢量、零矢量、死区矢量和补偿矢量时序，产生实际六路门极。
3. 电压约束层：使用电池端电压峰值/谷值快速回撤，作为快保护/限制，不把它当作析锂证明。
4. 温度监督层：根据电池温度和温升趋势调节 `bn` 或加热强度，形成温度锁定或温度目标控制。
5. RMS 审计层：独立计算电池支路 RMS、DC-link RMS、相电流 RMS 和窗口有效性，不用审计量直接替代控制量。

CN679 进入“控制对照层”：可在独立 `SimulationInput` 工况中测试母线 RMS 闭环和虚拟前馈表，但不在同一工况与 dSVPWM 主控制器叠加。

### 5.3 层 2：电池频域电热模型

**选择：以 Zhang 2015 的频域热生成和集总热平衡为结构基础，以 NPBH 的二阶 ECM 形式为首版候选实现，保留参数资格和波形适用性标志。**

模型至少需要区分：

- 全电池 `Z(T,SOC,f)` 和 `Re(Z)`；
- 电池支路电流、电驱有功电流和加热交流电流；
- 电池欧姆/极化热、逆变器损耗、电机铜耗/铁耗、DC-link 损耗和热散失；
- 频域近似产热与时域账本结果；
- 温度状态、环境/冷却边界和 SOC 状态。

Zhang 2015 的 18650 参数、NPBH 的 NPBH 台架参数、templock 的电池包参数都只能作为 `LITERATURE` 参考或初始扫描点，不直接变成平台默认设备配置。

### 5.4 层 3：析锂安全判据

**选择：建立 Jian 2023 的 NEPCS/TVCS 组合判据候选，但把可评估资格作为独立门控。**

首版安全输出建议包括：

- `U_neg_est` 或等效负极电位代理；
- `U_terminal_peak`、`U_terminal_valley`；
- `f_boundary(T,SOC)`；
- `I_limit_li_plating`、`I_limit_voltage` 和最终允许电流；
- `ProtectionStatus`、`ProtectionReason`、`DeviceProtection_Evaluable`。

在化学体系映射、D1/D2/D3 假设登记、目标电芯数据和 PWM 波形适用性检查完成前，`DeviceProtection_Evaluable` 保持 0。Jian 2023 原件视觉核验已完成，但这不等于 v2 数据契约和安全资格已满足。这样可以先把判据路径和未知项传播建起来，又不把平台代理误报成产品安全结论。

### 5.5 层 4：拓扑替代和工程场景

后续以单一 v2 模型的参数化扩展方式评估：

- **NPBH**：新增电池中点-电机中性点开关，重点审计 `i0`、驱动/加热解耦、开关故障和电池包拆分接口。
- **BYD 四线制**：重点审计 N 线、扩展 Clarke-Park、行驶/充电/停车多场景和控制器接口。
- **DMSI**：重点审计电池包分组、继电器状态和不支持途中加热的模式边界。

这些不是首版控制层的“另一套 PI 参数”，而是硬件拓扑和接口变化，必须单独定义验收条件。

### M5-01：M4 物理基线迁移和控制接口准备

### M5-02：dSVPWM 控制最小闭环

- 先实现 `bn=0` 时退化为标准 SVPWM，`bn>0` 时形成可审计的死区/补偿矢量换流。
- 以零速锁止、固定温度和短窗口作为诊断工况，再扩展温度监督。
- 验证实际门极配对、矢量时序、母线电流正负方向和电机转矩/速度边界。

## 7. 验收门槛和拒止条件

### 7.1 首版控制验收

- 模型结构检查和编译通过。
- 实际六路门极无重叠、上下桥臂严格互补，矢量时序与控制命令一致。
- `bn=0` 与标准 SVPWM 的退化关系可读回；`bn>0` 的电池支路电流变化可由独立 RMS 和波形审计复算。
- 电机转矩、转速和驱动跟踪不被误写成电池加热能力。

### 7.2 电热验收

- `Re(Z)` 的温度/SOC/频率适用范围、来源身份和替换影响均有记录。
- 频域热源与时域电气账本能够在固定窗口内复算；完整窗口账本残差按项目规范控制在 `1e-12` 量级。
- 热模型未通过波形等效校核时，输出标记为 `PlatformEstimate`，不形成产品温升结论。

### 7.3 安全验收

- 能输出负极电位/电压约束候选量、边界频率、限流值和触发原因。
- 参数不合格、化学体系不匹配、波形不适用或数据缺失时，未知项传播并保持 `DeviceProtection_Evaluable=0`。
- `DeviceProtection_Evaluable=1` 只能在数据契约、假设审计、模型验证和适用范围均满足后使用，不能由仿真运行成功自动置 1。

### 7.4 拒止条件

以下任一条件成立时，停止向后扩展并先修正：

1. 把 NPBH/BYD/DMSI 的硬件改动静默写入无硬件改动首版模型。
2. 把电机相电流、DC-link 电流、电池支路电流或热等效电流混为同一指标。
3. 用文献的 50 Hz、125 Hz、10 kHz 或 300-400 Hz 单值直接宣布“最优频率”。
4. 用全电池二阶 ECM 直接生成负极电位并宣布析锂安全。
5. 未完成 PWM 波形与频域热公式的适用性检查，却用频域公式给出设备级温升或安全结论。
6. 用专利定性效果、ATIC PPT 或 Shell 工程统计替代可复现实验/模型证据。

## 8. 结论边界

### 已验证事实

- 10 份 PDF 已有逐篇分析文件；其中 7 篇已完成原件视觉核验，另 3 篇仍为文本初析待核验。
- `2024templock` 和 `2026bit NPBH` 已完成高质量 Claude 深度分析并完成原件核验。
- M4 的 CMI 架构下开环低频 d 轴脉冲电流自加热路线存在电池侧有效电流不足的 Reference / Platform 卡点。
- 文献中的无硬件 dSVPWM、硬件重构 NPBH/四线制/DMSI、频域电热和析锂判据属于不同模型层。

### 基于证据的规划决定

- M5 首版选择无新增硬件的 dSVPWM 换流型脉冲自加热主线。
- 频域电热模型和析锂判据作为独立的模型层和安全层接入，不作为控制路线的附属参数。
- NPBH、BYD 四线制和 DMSI 保留为重要拓扑替代，不进入首版默认硬件。
- CN679 作为 RMS 闭环对照，CN173/ATIC 作为辅助材料。

### 尚未验证和禁止外推

- 当前没有具体设备电池化学体系、EIS 数据、电池包拓扑、平台电压等级或控制硬件配置。
- CN679、CN173、ATIC 的原件图表/页码仍未完成视觉核验；Jian 2023 和 Zhang 2015 已完成原件核验，但其参数仍不得越过适用性和数据资格门槛。
- 任何文献中的电流、频率、温升速率、SOH、效率和保护阈值都不是 v2 默认设备参数。
- 本报告不证明具体设备的析锂安全、寿命、量产可行性或法规合规。
