### 1.2 CMI 控制核心精确定位（v2 必须排除）

`blk_2` 内 M4 CMI 控制链路：
- `blk_532 Id_Pulse_Stall`（DiscretePulseGenerator）：低频脉冲源
- `blk_533 Scale_To_Bipolar_A`（Gain=2*pulse_id_amplitude_A）：缩放为双极
- `blk_534 Offset_To_Negative_A`（Bias）：偏置为负
- 经 `blk_536 Mux_Idq_ref` 输出 Idq_ref

这是"CMI 架构下开环低频 d 轴脉冲电流自加热"的控制核心，v2 必须删除并用常量占位。

### 1.3 实际门极路径

`blk_856 PWMGenerator(Three-phase,Two-level)`（载波比较型，输入 Vabc+Vdc），不是命令侧 duty。符合 v2 接口契约 PWM_Gates[1:6] 是实际六路门极的要求。

### 1.4 测量骨架

`blk_761 Battery_Current_Sensor` / `blk_762 Battery_Voltage_Sensor` -> PS-Simulink 转换 -> Goto 发布 I_batt_meas/V_batt_meas；Connection Label 建立 DC_Pos/Neg_Node、HV_Pos/Neg_Batt 节点骨架。

## 4. 结论边界

1. v2 模型已建立物理路径和测量/账本接口，但未实现任何加热控制（Idq_ref=[0;0] 占位）。
2. 0.04s 占位工况的账本残差非零是预期行为（DC-link 电容启动暂态 + PWM 纹波），不是账本框架错误；账本框架有效性已通过 Energy_Ledger_Valid=1 和 Electrical_Ledger_Valid=1 确认。
3. v2 当前状态为 Reference/Platform 接口基线，不构成加热能力、设备性能或安全结论。
4. IGBT_Vf_ref=1.2V（base）与 AGENTS.md 更正的 0.8V 存在冲突，属 M4 冻结基线遗留事项，v2 不在 M5-01 阶段修改；后续如需更正应在明确授权下进行并记录来源。
5. 下一步进入 M5-02：dSVPWM 控制最小闭环（bn=0 退化为标准 SVPWM，bn>0 死区/补偿矢量换流）。
