## 2. 优先级与状态定义

| 标记 | 含义 |
|---|---|
| `P0` | 阻断首轮主路径或账本语义；M4-01 必须先处理或形成经确认的阻断说明 |
| `P1` | M4-01 前置或首轮账本必须明确的影响 |
| `P2` | 不阻断首轮主路径，可在 M4-02/M4-03 或后续保真度提升处理 |
| `待确认` | 有结构证据，但尚未获得参数、权限或工程身份确认 |
| `建议采用` | 适合作为 M4-01 最小修改候选，不代表已授权 |
| `阻断` | 在关键数据/接口确认前不得进入 Active 或工程结论 |
| `暂缓` | 保留为参考，不进入当前修改范围 |

## 3. 基线事实和主路径

### 3.1 活动 V4 事实

| 事实 | 当前读回结果 | 影响 |
|---|---|---|
| 顶层六模块 | 六个模块均存在并有 root 级连接 | 暂保留顶层骨架 |
| 物理电源连接 | root 层 `Battery_pos <-> DC_plus`、`Battery_neg <-> DC_minus` | 只有端口连接事实，不能替代 DC-link 物理模型 |
| DC-link 模块 | `blk_20` 仅对 `BatteryDC` 和 `InverterDCFeedback` 做 Selector/信号桥 | DC-link 电容、R/L、接触器/预充、储能和高压损耗尚未明确 |
| 电池负载入口 | `BatteryThermalManagement_BatteryPack` 的 `BatteryLoad` 输入在内部接至 Terminator | 电池物理端口和信号负载路径存在语义脱节风险 |
| 驱动反馈 | `InverterDCFeedback=[V_dc,I_dc,P_dc,status]` 的 Mux 存在；V4 注释仍标明 `I_dc/P_dc` 为 NaN/部分状态 | 不能把 DC 功率和电池负载当作已验证测量 |
| 控制输入 | `Idq_ref` 进入 SI 单位 FOC，输出 `Duty_Cycles` | 参考语义、零速位置、采样和限幅仍需闭合 |
| 脉冲命令 | `Id_Pulse_Stall`、`Iq_ref_zero_A`、频率/占空比/相位/模式/状态字段 | V4 自有候选路径接口应保留，不被快充或速度控制替代 |
| KPI | `LimitStatus` 10 字段、`SystemKPI` 24 字段；相关 Mux 存在未连接目的端记录 | 需处理输出接收、记录和唯一字段语义 |
| RMS | `Compute_I_rms_V4G` 使用 `abs(I_instant)` | 不满足窗口 RMS 契约，不能作为工程 RMS |

### 3.2 首轮主路径

```text
电池包物理端
  -> 高压路径/DC-link（待物理闭合）
  -> 三相逆变器
  -> PMSM 电气端
  -> Park/Lock 机械边界（待明确）
```

推荐的官方候选组合为：`OF-BAT-01` 电池与冷却结构、`OF-HV-01` 官方物理电气元件、`OF-DRV-01` 三相 PMSM 热驱动、`OF-MCB-01` 的 `Control_System` 子系统，外加 V4 自有脉冲命令和审计层。

## 5. M4-01 最小修改候选集

在用户明确授权后，建议 M4-01 按以下顺序进一步细化，不在本文件阶段执行：

1. 先处理 `HV-001`、`HV-002`、`BAT-001` 和 `INV-001`，明确唯一物理能量路径和三个电流层级。
2. 再处理 `HV-003`、`KPI-002`、`TH-001`，建立端口功率、储能、损耗、热源和残差字段。
3. 同步处理 `CTRL-001`、`CTRL-002`、`CTRL-003`、`CTRL-004` 和 `MECH-001`，把脉冲参考、执行链和 Park/Lock 边界对齐。
4. 处理 `KPI-001`、`KPI-003`、`KPI-004`，使 RMS、输出记录和状态字段具备唯一审计语义。
5. 建立 `WARN-001` 至 `WARN-004` 台账，并在首个可运行物理工况前逐项修复或形成确认后的保留理由。

上述顺序是 M4-01 的候选执行顺序，不是本轮已授权的模型修改顺序。

## 6. 重构触发条件复核

M4-00 当前结论为：**暂不提出顶层六模块重构**。

保留六模块的理由：

- 电池、驱动、控制、脉冲监督和 KPI 责任边界仍可分别归入现有六模块。
- DC-link 缺口集中在现有 `DC_Link_And_Inverter_Interface` 内部，尚未证明必须改变顶层边界。
- 官方资产复用优势主要是子系统和物理块级，不要求新建并行顶层模型。

如果 M4-01 发现现有 `DC_Link_And_Inverter_Interface` 无法同时容纳物理母线、端口测量、储能、损耗和账本接口，必须另行提交重构提案，至少包含：具体优势、模块/接口/责任域变化、对 M4-01/M4-02/M4-03 的影响、增加的验证与追溯成本，以及不重构的明确缺陷。

## 7. M4-00 结束判定

本清单与 `M4-00_官方资产复用矩阵_v01.md` 共同满足以下 M4-00 输出要求：

- 六个 V4 顶层模块均有官方资料或“V4 自有薄层/缺口”结论。
