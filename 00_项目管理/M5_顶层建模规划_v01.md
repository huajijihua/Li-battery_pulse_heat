## 0. 路线命名与连续性

- **M4**：CMI（Combined Module Invert，文献原称）架构下的开环低频 d 轴脉冲电流自加热路线。
- **M5 首版**：dSVPWM（dead-zone Space Vector PWM，死区空间矢量脉宽调制）换流型脉冲自加热路线。
- M4 的"开环"仅限定低频 `Id_ref` 加热参考层；M5 的 `d` 表示 `dead-zone`（死区），不是 d 轴。
- M4 和 M5 保持同一首版硬件边界（单包、三相三线、标准三相两电平逆变器 + PMSM），M5 通过死区换流重构电池侧电流形成机制。
- 详细机理、控制差异和 `bn/d0/a0` 建模边界以 `M5_文献调研与差异分析_v01.md` 第 8 节为准。

## 1. M5 分层路线总览

M5 采用分层路线，控制、热模型、安全判据和拓扑替代不写成互相并列的首版路线，而是按模型层依次接入：

| 层 | 阶段 | 职责 | 主参考 | 状态 |
|---|---|---|---|---|
| 层 0 硬件边界 | M5-00 | 文献证据、接口契约、路线选择 | 2024templock + 2024 IEEE TTE | 完成 |
| 层 1a 控制接口 | M5-01 | M4 物理基线迁移、CMI 控制剥离 | M4 冻结基线 | 完成 |
| 层 1b 调制闭环 | M5-02 | dSVPWM 门极生成、bn=0 退化、bn>0 换流 | 2024templock + 2024 IEEE TTE | 完成 |
| 层 2 电热模型 | M5-03 | 频域电热、热账本交叉复算 | Zhang 2015 + NPBH 2026 | 完成 |
| 层 3 安全判据 | M5-04 | 析锂安全候选、拒止逻辑 | Jian 2023 | 完成 |
| 层 4 对照与拓扑 | M5-05 | 控制对照、拓扑替代评估 | CN679/CN173/NPBH/BYD/DMSI | 未启动 |

### 1.1 阶段依赖关系

```
M5-00（文献证据+接口契约）[完成]
  -> M5-01（物理迁移+CMI剥离）[完成]
     -> M5-02（dSVPWM调制闭环）[完成]
        -> M5-03（频域电热+热账本）
           -> M5-04（安全判据+拒止逻辑）
              -> M5-05（控制对照+拓扑替代）
```

阶段顺序固定，不允许跳过。每个阶段的准入条件包含前一阶段完成判据全部通过。

### 1.2 首版系统边界

| 项目 | 首版约束 |
|---|---|
| 硬件 | 单包、三相三线、标准三相两电平逆变器、PMSM、DC-link |
| 机械工况 | 零速锁止作为首个诊断边界；行驶工况在控制接口验证后扩展 |
| 主控制 | dSVPWM 换流型调制族，电压快速约束与温度慢速监督分层 |
| 不纳入默认拓扑 | 电池中点、N 线、NPBH 开关、DMSI 电池分组 |
| 不纳入默认结论 | 具体设备额定值、析锂安全、寿命、量产或法规结论 |

### 2.4 M5-03：频域电热与热账本

**状态**：已完成（2026-07-24）

**准入条件**：
- M5-02 完成判据全部通过。
- dSVPWM 电流波形已定义（bn>0 工况的 I_batt 正负交变波形可读回）。

**技术方向（依据文献评估报告第 5.3 节）**：
- 以 Zhang 2015 的频域热生成和集总热平衡为结构基础。
- 以 NPBH 2026 的二阶 ECM 形式为首版候选实现。
- 保留 `LITERATURE`/`ENGINEERING_DEFAULT`/`UNKNOWN` 来源标志。
- 对 PWM 电流执行谐波等效或时域交叉检查。
- 将固定 `R_batt` 代理和频域模型结果并列审计，避免未经验证的静默替换。

**工作内容**：
1. 建立电池频域阻抗接口 `Z_batt(T,SOC,f)` 和 `Re(Z)`，记录波形假设和来源。

6. 区分电池欧姆/极化热、逆变器损耗、电机铜耗/铁耗、DC-link 损耗和热散失。

### 2.5 M5-04：安全判据候选和拒止逻辑

**状态**：已完成（2026-07-24）

**阶段目标**：
接入 Jian 2023 的负极电位、端电压和边界频率组合逻辑，建立 `UNKNOWN/CANDIDATE/EVALUABLE` 三状态安全候选路径，实现拒止逻辑（参数不合格时保持 `DeviceProtection_Evaluable=0`）。

**准入条件**：
- M5-03 完成判据全部通过。
- M5-03 输出具备来源/适用范围标记。

**技术方向（依据文献评估报告第 5.4 节）**：
- 采用 Jian 2023 的 NEPCS/TVCS 组合判据候选。
- 把可评估资格作为独立门控，与判据计算分离。

**工作内容**：
1. 建立安全候选输出接口：`U_neg_est`、`U_terminal_peak`、`U_terminal_valley`、`f_boundary(T,SOC)`、`I_limit_li_plating`、`I_limit_voltage`、`ProtectionStatus`、`ProtectionReason`、`DeviceProtection_Evaluable`。
2. 实现 NEPCS（负极电位）判据路径候选。
3. 实现 TVCS（端电压）判据路径候选。
4. 实现拒止逻辑：参数不合格、化学体系不匹配、波形不适用或数据缺失时，未知项传播并保持 `DeviceProtection_Evaluable=0`。
5. 先完成单电池 Reference/Platform 验证，再讨论电池包聚合和不一致性。

**安全状态契约**：

| 状态 | 条件 | `DeviceProtection_Evaluable` |
|---|---|---|
| `UNKNOWN` | 缺少目标化学体系、EIS、负极映射、波形校核或适用范围 | `0` |
| `CANDIDATE` | 已运行 NEPCS/TVCS 数学路径，但依赖文献比例或平台假设 | `0` |
| `EVALUABLE` | 来源、单位、温度/SOC 范围、波形等效和独立验证全部满足 | `1` |

**完成判据**：

| 判据 | 标准 |
|---|---|
| 安全候选输出接口建立 | 能输出负极电位/电压约束候选量、边界频率、限流值和触发原因 |
| 拒止逻辑有效 | 参数不合格时未知项传播并保持 `DeviceProtection_Evaluable=0` |
| `EVALUABLE` 门控严格 | 只能在数据契约、假设审计、模型验证和适用范围均满足后置 1，不能由仿真运行成功自动置 1 |
| 单电池验证优先 | 先完成单电池 Reference/Platform 验证，再讨论电池包聚合 |

**收口条件**：
- 安全候选路径和拒止逻辑已实现，`DeviceProtection_Evaluable` 在资格不满足时稳定保持 `0`。
- M5-05 准入条件满足（首版控制、电热和安全接口稳定）。

- 不得用 ATIC PPT 或 Shell 工程统计替代可复现实验/模型证据。

### 2.6 M5-05：控制对照与拓扑替代

**状态**：未启动

**阶段目标**：
在首版控制、电热和安全接口稳定后，独立比较 CN679 RMS 闭环和必要的 CN173 开环 Ud 脉冲（控制对照），评估 NPBH、BYD 四线制和 DMSI 拓扑替代（硬件替代）。

**准入条件**：
- M5-04 完成判据全部通过。
- 首版控制、电热和安全接口稳定。

**技术方向（依据文献评估报告第 5.5 节）**：
- 控制对照：CN679 RMS 闭环和 CN173 开环 Ud 脉冲作为独立 `SimulationInput` 工况，不与 dSVPWM 主控制器叠加。
- 拓扑替代：NPBH、BYD 四线制、DMSI 作为单一 v2 模型的参数化扩展方式评估，不作为"另一套 PI 参数"。

**工作内容**：
1. CN679 控制对照：在独立 `SimulationInput` 工况中测试母线 RMS 闭环和虚拟前馈表，不覆盖主路线结果。
2. CN173 开环对照：在独立工况中测试开环 Ud 脉冲，作为信号流反例。
3. NPBH 拓扑评估：新增电池中点-电机中性点开关（参数化扩展），审计 `i0`、驱动/加热解耦、开关故障和电池包拆分接口。
4. BYD 四线制评估：审计 N 线、扩展 Clarke-Park、行驶/充电/停车多场景和控制器接口。
5. DMSI 评估：审计电池包分组、继电器状态和不支持途中加热的模式边界。

**完成判据**：

| 判据 | 标准 |
|---|---|
| 控制对照独立 | CN679/CN173 在独立 `SimulationInput` 工况中测试，不与 dSVPWM 主控制器叠加 |
| 拓扑替代参数化 | NPBH/BYD/DMSI 以单一 v2 模型参数化扩展方式评估，不创建并行模型副本 |
| 对照不覆盖主路线 | 控制对照和拓扑替代结果独立记录，不覆盖首版主路线结论 |
| 验收条件单独定义 | 每个拓扑替代单独定义验收条件，不混为"另一套 PI 参数" |

**收口条件**：
- 控制对照和拓扑替代评估完成，结果独立记录。
- M5 阶段全部完成。

**禁止外推**：

- ECM 参数为 ENGINEERING_DEFAULT 占位，所有频域热结果标 PlatformEstimate，不形成产品温升结论。

## 4. 拒止条件

以下任一条件成立时，停止向后扩展并先修正：

1. 把 NPBH/BYD/DMSI 的硬件改动静默写入无硬件改动首版模型。
2. 把电机相电流、DC-link 电流、电池支路电流或热等效电流混为同一指标。
3. 用文献的 50 Hz、125 Hz、10 kHz 或 300-400 Hz 单值直接宣布"最优频率"。
4. 用全电池二阶 ECM 直接生成负极电位并宣布析锂安全。
5. 未完成 PWM 波形与频域热公式的适用性检查，却用频域公式给出设备级温升或安全结论。
6. 用专利定性效果、ATIC PPT 或 Shell 工程统计替代可复现实验/模型证据。

## 5. 参数与证据纪律

1. 只允许 `OFFICIAL`、`LITERATURE`、`ENGINEERING_DEFAULT`、`PLATFORM_DEFAULT` 和明确的 `PLACEHOLDER` 进入 v2 默认配置；所有参数必须有来源标签。
2. 文献参数进入 v2 须标 `LITERATURE` 来源，记录原文页码/表格/公式编号。
3. IEEE TTE 的 `Kp` 在正文 `0.01` 与 Table II `0.1` 冲突，登记为 `LITERATURE_CONFLICT`，不得写入 v2 默认控制器。
4. Zhang 2015 的 18650 参数、NPBH 台架参数、templock 电池包参数只作为 `LITERATURE` 参考或初始扫描点，不直接变成平台默认设备配置。
5. 峰值、平均值、窗口 RMS、电池支路电流、DC-link 电流、电机相电流和热等效量必须分别定义，不能互相替代。
6. 每项结论必须区分已验证事实、基于证据的推断和待确认假设。
7. 真实设备数据接入前，不得将平台默认值写成设备参数或安全边界。
8. `DeviceProtection_Evaluable=0` 在 M5-04 资格满足前保持；只有来源、适用范围、波形等效和独立验证均满足后，才允许置为 `1`。
