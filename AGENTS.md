## 3. M3 建模定位

M3 是通用、模块化的低温脉冲加热**系统集成设计与参数匹配模型**。它应在系统边界内表达电池、高压配电与 DC-link、逆变器、PMSM、控制器、热管理、保护逻辑和必要的简化机械约束，并以工程兼容性筛查作为验证方法。

M3 服务于：

1. 通用低温脉冲加热系统集成设计和参数匹配。
2. 跨子系统电压、电流、功率、热、频率、时序和保护约束的兼容性判断。
3. 能量路径、热源分配、参数敏感性和控制策略的可审计分析。

M3 不服务于：

1. 未经数据验证的具体产品定型、析锂安全、寿命、NVH、退磁或量产许可结论。
2. 完整 CAD、壳体、支架、线束三维布局或结构有限元建模。
3. 直接继承历史车型、L0.5 或 M2 的默认参数和架构。

## 5. 参数与证据纪律

1. 只允许 `OFFICIAL`、`LITERATURE`、`ENGINEERING_DEFAULT`、`PLATFORM_DEFAULT` 和明确的 `PLACEHOLDER` 进入 M3 默认配置；所有参数必须有来源标签。
2. 历史案例和 L0.5 参数只可标记为 `HISTORICAL_CASE` 或 `L0.5_SCREENING`，不得进入默认模型、默认结论或默认工况。
3. 峰值、平均值、时间窗口 RMS、母线电流、电池支路电流、电机相电流和热等效量必须分别定义，不能互相替代。
4. 每项结论必须区分已验证事实、基于证据的推断和待确认假设；脚本通过或仿真完成不等于物理正确或工程可用。
5. 真实设备数据接入前，不得将平台默认值写成设备参数或安全边界。

## 7. MATLAB/Simulink 纪律

1. 读取 `.slx` 优先使用 `model_overview -> model_read(depth=0/1) -> model_query_params/model_resolve_params`，不按 XML 或文本修改结构化模型。
2. 每次结构修改遵循：范围定义 -> 保存/记录检查点 -> 单一 scope 最小修改 -> `model_check` -> read-back -> 视觉画布复核 -> 保存。
3. 任何结构 warning 必须修复或形成可审计的保留理由；任何能量、RMS、控制或收敛结论必须有明确计算口径。
4. 禁止使用未限定范围的物理连接清除；若必须重接，只对已解析的实际端口做定向断开，并在 `partial` 或失败后立即 read-back/check。
5. 视觉复核必须检查 root 和受影响 scope 的断线、悬空线、端口错接、节点短接和明显重叠；临时图片只放在系统临时目录，不进入 Active 资产。
6. AMESim、COMSOL、统计模型或其他工具不是预设主线。只有 M3 的系统边界或验证需求明确需要时，才建立相应计划和接口契约。
