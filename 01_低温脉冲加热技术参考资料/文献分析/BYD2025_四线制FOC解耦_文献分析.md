## 2. 核心方法

### 2.1 电池包拆分 + 中性点引出（四线制拓扑）

将电池包拆分为上下两个半包，从中性点引出 N 线连接至电机定子中性点，构成三相四线制系统。相比传统三相三线制（IA+IB+IC=0），引入 IN 自由度：

```
IA + IB + IC + IN = 0
```

加热过程一个周期包含四个换流模式：上半包放电→下半包充电→下半包放电→上半包充电，通过上下半包交替充放电产生焦耳热。

### 2.2 扩展 Clarke-Park 变换（3×3 矩阵）

在传统 FOC 的 2×3 Clarke-Park 变换矩阵基础上，增加 IN 维度，扩展为 3×3 矩阵：

```
[Id, Iq, IN]^T = T' · [IA, IB, IC]^T
```

实现加热电流 IN 与驱动电流 Id/Iq 在 dq0 坐标系下的完全解耦控制。

### 2.3 复合控制框架

| 模块 | 功能 |
|---|---|
| 相电流采集与变换 | Hall 传感器采集三相电流，经扩展 Clarke-Park 分解为 Id, Iq, IN |
| 电机转矩控制（黄色） | 查表获得 Id/Iq 目标值，PI 调节生成 UA, UB, UC |
| 电池自加热控制（绿色） | 根据电池状态确定 IN 目标值，PI 调节生成 UN：UN = Kp·eN + Ki·∫eN dt |
| 复合电压处理（蓝色） | 将 UA/UB/UC 与 UN 合成最终逆变器电压：[UA*, UB*, UC*]^T = f([UA, UB, UC, UN]^T) |

### 2.4 全场景覆盖

- **充电场景**：停车+连接充电桩，IN 独立控制加热
- **停车启动场景**：-38°C 极低温冷启动
- **行驶场景**：IN 加热与电机转矩并行控制（-30°C，IN=100A 同时维持驱动性能）

## 7. 结论边界（不可外推项）

1. **需要电池包中性点引出 + N 线（硬件改动）**：该方法要求将电池包拆分为上下半包并引出中性点，属于电池包内部结构级改动，非纯软件方案。v2 平台若采用此方法，需评估电池包结构变更的可行性和成本。
2. **加热频率 50 Hz 基于实验优化，缺乏频域阻抗模型支撑**：文中未建立电池包的频域阻抗模型（EIS）来解析 50 Hz 最优的理论依据，仅通过实验扫描确定。频率选择的可迁移性取决于电池类型和电池包电感/电容特性。
3. **未讨论析锂判据**：虽然 300 次循环后 SOH>97%，但未建立析锂阈值模型（如负极电位监测、阳极过电位估算），低温大电流脉冲的析锂安全边界未量化。
4. **未讨论 NVH 的定量分析**：仅定性说明 50 Hz 振动弱于更低频率，未给出电机振动、噪声的定量测量数据或 NVH 模型。
5. **控制参数未公开**：IN 的 PI 控制器 Kp/Ki 参数未公开，控制器的载波频率/开关频率未说明，这些参数直接影响控制性能的工程复现。
6. **加热电流上限受限于逆变器电流容量**：IN 最大 RMS 300 A、相电流瞬时最大值 600 A 是基于比亚迪特定逆变器硬件能力，不可直接推广到其他平台。
7. **仅验证 LFP 电池**：150 Ah LFP 电芯的内阻特性、频响特性和热特性与 NCM 不同，该方法在 NCM 上的加热速率和寿命影响需要额外验证。

## 8. 对 v2 建模的参考价值

### 8.1 可迁移的方法

1. **四线制扩展 FOC 解耦架构**：3×3 扩展 Clarke-Park 变换实现 Id/Iq/IN 解耦，是 v2 从 CMI 开环架构转向闭环解耦控制的核心参考路径，可直接作为 v2 控制子系统的一种候选架构。
2. **双 PI 独立控制框架**：转矩控制（Id/Iq 双 PI）+ 加热控制（IN 单 PI）的独立控制架构，分工明确，可迁移到 v2 的控制层设计。
3. **复合电压合成策略**：将电机控制电压（UA/UB/UC）与加热控制电压（UN）在最终级合成的方案，为 v2 的 PMW 调制前端提供参考。

### 8.2 可迁移的参数与判据

1. **50 Hz 最优频率**：可作为 v2 平台加热频率的初始参考值（标记为 `LITERATURE` 来源），但需结合 v2 电池包频域特性重新验证。
2. **1.5C 倍率加热电流**：IRMS=225 A（1.5C）在 300 次循环内 SOH>97%，可作为 v2 加热电流幅值的参考上限（标记为 `LITERATURE` 来源）。
3. **SOC 无显著影响**：10%-90% SOC 范围内加热速率稳定，可简化 v2 加热策略的 SOC 依赖设计。

### 8.3 需谨慎处

1. **电池包结构差异**：v2 平台若不分半包、不引出 N 线，则无法直接采用四线制 FOC 解耦；需改用其他方案（如 CMI、dSVPWM、电压反馈式等）实现电池侧交流电流。
2. **参数未公开**：PI 增益、载波频率等关键控制参数未公开，v2 需自行整定，不可默认使用比亚迪参数。
3. **析锂判据缺失**：v2 应建立析锂阈值模型，弥补该文献的安全评估空白。
4. **NVH 需定量评估**：v2 平台应建立电机振动/噪声的定量评估模型，不可仅依赖 50 Hz 的定性结论。

### 8.4 与 M4 及其他文献的关系

- **与 M4 CMI 架构的关系**：该文献的四线制 FOC 解耦代表了闭环解耦控制路径，与 M4 的 CMI 开环 d 轴脉冲注入在架构上存在本质差异。v2 平台可选择二者之一作为控制架构候选，也可混合使用。
- **与 2024 IEEE TTE 电压反馈式加热的关系**：该文献（比亚迪，整车级验证）与 2024 IEEE TTE 电压反馈式加热（浙大，台架级验证）在 DC-link 电容约束上立场不同——比亚迪方案不依赖 DC-link 电容边界，通过 N 线直接注入 IN；而电压反馈式要求 Cp < 1 mF。两者可互为 v2 不同硬件拓扑下的控制方法候选。
- **与 M4 冻结基线的关系**：该文献的参数应标记为 `LITERATURE` 来源，不自动升级为 v2 默认值。M4 的 RMS 窗口审计、账本校验和参数纪律应继承到 v2。
