## 2. 核心方法

### 2.1 控制架构总览

提出一种**全开环 Ud 电压源脉冲控制**架构，由电机控制单元（MCU）执行。**全程无三相电流反馈量**，仅依赖转子位置信号和标定表。每个控制周期依次执行：

1. 查档位-频率-电压表得 f 和 Ud（加热档位请求值由 BMS 根据电池温度确定）
2. 对 Ud 和 Uq=0 做 Park⁻¹(θ) → Uα, Uβ
3. 生成周期为 2/f 的脉冲信号（前 1/f 高电平，后 1/f 低电平）
4. Uα, Uβ + f → SVPWM → 初始 PWM（周期 1/f）
5. 初始 PWM 与脉冲信号做**与运算** → 实际 PWM（周期 2/f）
6. 实际 PWM 驱动六路功率开关

### 2.2 储能与续流状态

专利明确描述了两种工作状态（说明书 [0057]–[0059]）：

**储能状态**（图 2）：S1(上 U)、S2(上 V)、S6(下 W) 导通，其余断开。电流从电池正极 → S1/S2 → U/V 相绕组 → W 相绕组 → S6 → 电池负极。电池放电→电机电感储能，电流方向为电池正极流出、负极流入。

**续流状态**（图 3）：六管全断。由于电感续流特性，电流从 W 相绕组流出 → S3 体二极管 → 电池正极 → 电池负极 → S4/S5 体二极管 → U/V 相绕组。电池被充电，电流方向与储能状态相反。

储能与续流交替形成脉冲电流，电池内阻在两种状态下均产热。

### 2.3 档位-频率-电压表

该表是核心工程标定环节，通过标定方式获得并预存在 MCU 中。每个加热档位请求值对应一组 (f, Ud)：

| 加热档位请求值 | 含义 |
|---|---|
| 0 | 无需加热（初始值） |
| 1 | 低档 |
| 2 | 中档 |
| 3 | 高档 |

档位请求值由 BMS 根据电池温度确定，通过 CAN 发送给 MCU。专利未公开具体 (f, Ud) 数值。

### 2.4 进入与退出条件

**进入条件**（条件 1a~1d，需同时满足）：
- 1a：加热允许指令 ≠ 0（VCU 下发）
- 1b：加热档位请求值 ≠ 0（BMS 下发）
- 1c：电机系统处于高压待机模式
- 1d：电机转速 ≤ n1（预设不会导致车辆行驶/抖动的转速阈值）

**退出条件**（条件 2a~2d，任意一条满足即退出）：
- 2a：加热允许指令 = 0
- 2b：加热档位请求值 = 0
- 2c：电机转速 > n1
- 2d：脉冲加热持续时间 ≥ Tmax（单次最大允许加热时间阈值）

### 2.5 与 M4 冻结基线（CMI 架构）的差异

| 维度 | 专利 173 | M4（CMI 架构） |
|---|---|---|
| 控制方式 | 开环电压源（Ud 参考） | 开环电流源（Id_ref 参考） |
| 换流路径 | 标准 SVPWM + 脉冲信号与运算 | 非零矢量 + 零矢量 + 死区时间 |
| 电流反馈 | 无 | 无 |
| 调制周期 | 2/f（实际 PWM） | 由 CMI 时序决定 |
| 转子位置依赖 | 是（Park⁻¹ 需要 θ） | 不依赖（固定 d 轴注入） |
| 桥臂导通模式 | 储能态 S1/S2/S6 导通 | 由 CMI 时序决定 |

## 3. 关键步骤

```
加热允许指令(VCU) + 加热档位请求值(BMS) + 电机转速 + 转子位置 θ
                          │
                          ▼
              加热条件处理模块（判断进入/退出）
                          │
                          ▼
                   档位分析模块
                          │
              查档位-频率-电压表 → f, Ud
                          │
          ┌─────────────────┼─────────────────┐
          ▼                 ▼                  ▼
   Park⁻¹(Ud, Uq=0)   脉冲信号生成          SVPWM 模块
          │             (周期 2/f,           │
          ▼              前 1/f=1)            ▼
     Uα, Uβ ◄───────────────┤         初始 PWM (周期 1/f)
          │                                   │
          └─────────────────┬─────────────────┘
                            ▼
                  与运算（初始 PWM & 脉冲信号）
                            │
                            ▼
                  实际 PWM (周期 2/f)
                            │
                            ▼
               六路功率开关 → 储能/续流交替
                            │
                            ▼
                电池内阻产热 → 脉冲加热
```

## 6. 关键结果

专利文本未提供定量结果，仅声明以下定性效果（说明书 [0033]–[0036]）：

| 效果 | 声明依据 |
|---|---|
| 无需增加硬件成本 | 复用电机系统实现脉冲加热 |
| 避免车辆非预期行驶/抖动 | Uq=0 确保零转矩 |
| 开环控制响应更快 | 无三相电流反馈，无闭环延迟 |
| 脉冲电流稳定、波动小 | 开环方式避免反馈量不稳定导致的波动 |
| 加热速度更快 | 脉冲电流生成速度更快 |
| 无电流闭环也能实现加热 | 通过 Ud 直接调节脉冲电流幅值 |

**无定量数据**：无加热速率、电流幅值、温升曲线、效率或电池老化数据。

## 7. 结论边界（不可外推项）

1. **无定量实验结果**：专利仅描述方法架构和标定框架，未提供任何台架或整车实验结果，无法评估实际加热速率、电流精度、温升效率或稳定性指标。

2. **档位-频率-电压表数值未公开**：f 和 Ud 的具体数值未知，该表与特定电机（Ld/Lq/ψf）、特定电池电压平台强相关，不具备跨平台通用性，v2 无法直接移植。

3. **无电流闭环导致参数漂移无补偿**：Ud 开环注入方式下，电机电阻随温度变化、IGBT 导通压降随电流变化、电池电压随 SOC/温度变化，这些漂移均无法自动补偿，输出电流幅值会随温度/工况偏移。专利声明"脉冲电流稳定、波动小"未提供证据支持。

4. **DC-link 电容影响未讨论**：未提及母线电容 C 的取值边界或对脉冲电流形成的影响。IEEE TTE 2024 等文献已证明 DC-link 电容是电池侧交流电流形成的决定性边界条件。

5. **无安全/析锂判据**：未讨论低温大电流脉冲下的析锂风险、负极电位监测或保护策略。"脉冲加热效果"的改善无定量佐证。

6. **标定工作量未知**：档位-频率-电压表需在特定电驱硬件上逐点标定，专利未说明标定方法、工况点数或标定周期，工程实现难度未知。

7. **SVPWM 具体实现参数未公开**：载波频率、死区时间、PWM 更新周期、最小脉宽限制等均未说明，这些参数直接影响实际脉冲电流波形。

## 8. 对 v2 建模的参考价值

### 8.1 可迁移的方法

1. **开环 Ud 电压源脉冲架构**：v2 可将此架构作为电压源方案的参考实现，与专利 679 的电流闭环方案形成对比基线。在 v2 中构建 Ud 开环注入 + SVPWM + 脉冲信号与运算的备选控制策略，便于评估两种架构的加热效果差异。

2. **储能/续流双状态描述**：专利对储能状态（S1/S2/S6 导通）和续流状态（六管全断）的电流路径描述清晰，可直接用于 v2 的物理模型验证——检查逆变器输出是否与专利描述的导通模式一致。该模式与 ATIC 等文献描述一致。

3. **档位-频率-电压表结构**：虽然数值不可用，但"档位→(f, Ud)"的表驱动架构可在 v2 中复用为 BMS 接口层的参考生成逻辑。v2 初期可先以仿真生成虚拟标定表，再视需要替换为台架数据。

4. **Uq=0 零转矩约束**：作为 v2 控制策略的硬约束，与专利 679 一致，确保加热过程不产生净转矩输出。

### 8.2 需谨慎处

1. **无电流反馈**：开环 Ud 注入在变温工况下输出电流会漂移，不能替代有电流闭环的方案。v2 若采用此架构，需额外评估温度漂移对加热效果的影响。

2. **无定量数据**：无法从中提取任何可验证的数值参考（加热速率、电流幅值、温升斜率等），v2 的参数整定需依赖仿真或文献对比。

3. **标定工作量大但不透明**：档位-频率-电压表需逐点标定，且专利未说明标定方法，v2 初期应先用仿真生成虚拟标定表验证控制架构可行性。

4. **转子位置依赖**：Park⁻¹ 需要 θ，该架构在零速/极低速下的转子位置观测精度会影响控制效果。v2 需评估位置传感器精度或实现无传感器位置观测算法。

### 8.3 与 M4 / 其他文献的关系

- **与 M4 CMI 架构的关系**：该专利不依赖 CMI 换流模式，而是通过标准 SVPWM + 脉冲信号与运算实现储能/续流切换。与 M4 的 CMI 架构在换流路径上正交。该专利的"与运算"实现方式比 CMI 的死区时间控制更简洁，但 SVPWM 本身依赖转子位置，CMI 不依赖。

- **与专利 679 的关系**：专利 173 是开环电压源方案，专利 679 是闭环电流源方案（含 RMS 反馈 + 转子角度前馈）。两者都是长安系专利，方法互补：
  - 173 的优势：响应快、实现简单、无电流传感器依赖
  - 679 的优势：闭环抑制漂移、控制精度高
  - 两者共同点：Uq=0 零转矩、储能/续流双状态、档位表的工程化思路

- **与 M4 冻结基线的关系**：M4 采用 Id 参考开环注入（电流源风格），专利 173 采用 Ud 参考开环注入（电压源风格）。两者都无电流反馈，但参考量类型不同。v2 应将两种开环方案均纳入对比，评估电压源 vs 电流源开环脉冲在加热效果、参数敏感性和实现复杂度上的差异。

- **参数标记**：该专利的参数应标记为 `LITERATURE` 来源，不自动升级为 v2 默认值。其 Ud 控制方式可作为 v2 对比方案之一，但主要参考价值在于架构思路而非具体数值。
