## 2. 核心方法

### 2.1 控制架构总览

提出一种**母线电流有效值闭环 + 转子角度前馈**的双闭环控制架构，由电机控制器（MCU）执行。每个控制周期依次执行：

1. 读取旋变角度 θ 和 BMS 下发的期望母线电流有效值 I_rms_ref
2. 查直轴前馈电流表得 Id_ini(θ, I_rms_ref)
3. 相电流 → Clark → Park → 阈值筛选 RMS → Id_fb / Iq_fb
4. d 轴：Id_ini + Id_fb → PI → Ud_req
5. q 轴：Iq_tag(=0) + Iq_fb → PI → Uq_req
6. Ud_req / Uq_req → Park⁻¹(θ) → Uα/Uβ → SVPWM → 6 路 PWM

**核心创新**：控制目标是**母线电流有效值**（而非电池电压），且通过转子角度前馈查表补偿定子电感随角度变化引起的电流偏差。

### 2.2 直轴前馈电流表标定

该表是专利的核心工程环节，通过**电驱总成测功机台架**逐点标定获得：

| 步骤 | 操作 |
|---|---|
| 1 | 选取 n 个转子角度（如 0°, 10°, 20°, ...）和 m 个母线电流有效值目标（如 100 A, 110 A, 120 A, ...） |
| 2 | 电驱总成上测功机台架，电流钳接直流母线监控 RMS |
| 3 | 测功机带动转子转动 1 min 后锁止在第一个角度 |
| 4 | 在电池额定电压 UN 下，Uq=0，调节 Ud，使母线电流 RMS 依次达到 m 个目标值，记录各目标下的相电流 |
| 5 | 对每组相电流做 Clark→Park→RMS（阈值筛选），得 Id_ini |
| 6 | 遍历 n 个角度，形成 n×m 二维表 |

**RMS 计算方式**：选取第二预设时间（如 1 s）内所有大于预设阈值的直轴实际电流值进行 RMS 计算。

### 2.3 PI 闭环控制

- **d 轴 PI**：参考 = Id_ini(θ, I_rms_ref) + Id_fb，输出 Ud_req
- **q 轴 PI**：参考 = 0 + Iq_fb，输出 Uq_req，确保 Iq_tag=0 → 零转矩
- 电流反馈量 Id_fb / Iq_fb 均为**经过阈值筛选的 RMS 值**，而非瞬时值

### 2.4 期望电流来源

BMS 根据电池温度查询预设的**温度-电流表**得到 I_rms_ref，再经 CAN 发送给 MCU。温度-电流表同样通过标定获得。

## 5. 实验设计

### 5.1 前馈表标定方法

采用**电驱总成测功机台架**离线标定：

- 锁止电机转子在各角度，消除机械运动
- Uq=0 确保零转矩输出
- 手动调节 Ud 使母线电流 RMS 达到目标值
- 通过相电流后处理构建 Id_ini(θ, I_rms_ref) 二维表

### 5.2 控制验证

专利未提供实验数据或仿真结果，仅描述方法本身。验证方法为**权利要求 9–10** 中定义的系统级和整车级权利要求，未提供具体测试结果。

## 6. 关键结果

专利文本未提供定量结果（加热速率、电流精度、稳定性指标等），仅声明以下定性效果：

| 效果 | 声明依据 |
|---|---|
| 降低转子角度引起的电流偏差 | 前馈查表补偿电感随角度变化 |

**无定量数据**：无加热速率、电流精度、温升曲线、效率或电池老化数据。

## 7. 结论边界（不可外推项）

1. **无定量实验结果**：专利仅描述方法架构和标定流程，未提供任何台架或整车实验结果，无法评估实际控制精度、加热速率或稳定性指标。
2. **PI 参数未公开**：d/q 轴 PI 增益、积分时间常数未知，无法直接移植到 v2 平台；需重新整定。
3. **开关频率/死区时间未公开**：SVPWM 的载波频率、死区配置、PWM 周期均未说明，这些参数直接影响电池侧电流纹波幅值和加热效果。
4. **DC-link 参数未讨论**：未提及 DC-link 电容值、母线寄生电阻/电感，而 IEEE TTE 2024 文献已证明这些参数是电池侧交流电流形成的决定性边界条件。
5. **标定条件依赖特定硬件**：前馈表在特定电驱总成（特定电机 Ld/Lq/ψf、特定控制器、特定电池额定电压 UN）下标定，更换硬件后需重新标定，不具备跨平台通用性。
6. **未涉及安全/析锂判据**：未讨论低温大电流脉冲下的析锂风险、负极电位监测或保护策略。
7. **未讨论温度耦合效应**：标定在额定电压 UN 下进行，实际加热过程中电池电压、电机电阻、IGBT 导通压降均随温度变化，前馈表在变温工况下的有效性未验证。

## 8. 对 v2 建模的参考价值

### 8.1 可迁移的方法

1. **母线电流有效值闭环架构**：v2 可采用类似的 d 轴 RMS 闭环 + q 轴零转矩闭环结构，替代 M4 的开环 Id 脉冲注入方式。该架构的 PI 闭环天然具备抗干扰和抗参数漂移能力。
2. **转子角度前馈查表**：用于补偿定子电感随角度变化引起的电流偏差，可移植到 v2 的 d 轴电流参考生成环节。在 v2 中可先用仿真生成虚拟前馈表（基于电机 Ld(θ)/Lq(θ) 模型），再视需要替换为台架标定数据。
3. **RMS 反馈计算方式**：阈值筛选 + 固定窗口 RMS 的计算方法，与 M4 已有的 RMS 窗口审计模块在概念上一致，可复用 M4 的 RMS 计算基础设施。
4. **温度-电流双层表架构**：BMS 温度→I_rms_ref→Id_ini 的表驱动链路，可移植到 v2 的 BMS 接口和控制器参考生成逻辑。

### 8.2 可迁移的安全代理

1. **Iq_tag=0 零转矩约束**：作为 v2 控制策略的硬约束，确保加热过程不产生净转矩输出。
2. **电流阈值筛选 RMS**：避免零电流区间的噪声/死区分量污染 RMS 计算，可继承 M4 的 RMS 窗口设计经验。

### 8.3 需谨慎处

1. **无定量实验数据**：无法从中提取任何可验证的数值参考（加热速率、电流精度、温升斜率等），v2 的控制参数整定需依赖仿真或文献对比。
2. **标定工作量大**：n×m 二维表需逐点台架标定，v2 初期应先用仿真生成虚拟标定表，验证控制架构可行性后再考虑实物标定路径。
3. **硬件依赖性强**：前馈表在特定电驱硬件上标定，v2 若更换电机或控制器参数，前馈表需重新生成或引入自适应补偿。
4. **未涉及 DC-link 电容约束**：需结合 IEEE TTE 2024（Cp < 1 mF）的结论，在 v2 中增加 DC-link 参数对母线电流 RMS 控制可行性的影响分析。

### 8.4 与 M4 / 其他文献的关系

- **与 M4 CMI 架构的关系**：该专利不依赖 CMI 换流模式（非零矢量+零矢量+死区），而是通过标准 SVPWM + d/q 轴 PI 闭环实现母线电流 RMS 控制。与 M4 的 CMI 架构在换流路径上正交，可作为 v2 从 CMI 转向标准 SVPWM 控制的重要参考。
- **与 2024 IEEE TTE 电压反馈策略的关系**：该专利控制目标为**母线电流有效值**，IEEE TTE 控制目标为**电池峰值/谷值电压**，两者在控制架构上互补。该专利的优势在于直接控制电流（更接近加热源），IEEE TTE 的优势在于已有实验数据和 DC-link 边界分析。
- **与 M4 冻结基线的关系**：该专利的 RMS 窗口计算方式与 M4 的 RMS 窗口审计模块在概念上一致，M4 的 RMS 计算基础设施（`Compute_I_rms_V4G`）可直接为 v2 复用。该专利的参数应标记为 `LITERATURE` 来源，不自动升级为 v2 默认值。
