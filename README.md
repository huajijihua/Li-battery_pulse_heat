# 车载低温脉冲加热系统工程建模

日期：2026-07-15
当前阶段：M3 阶段 4 已冻结；下一步 M4-00 资料检索与官方案例复用复核

本仓库用于建立面向工程集成设计的通用低温脉冲加热系统模型。研究对象不是某一具体车型或产品，而是电池、高压配电与 DC-link、逆变器、PMSM、控制器、热管理和保护逻辑之间的多域兼容性。

## 当前目标

M3 的目标是形成通用低温脉冲加热系统集成设计与参数匹配框架，用于回答：

1. 系统凭什么能发热。
2. 产生的热是否合理、能量和热源分配是否可审计。
3. 发热过程如何被控制，控制策略如何受电气、热、机械和保护约束限制。
4. 电压、电流、功率、热、频率、采样、保护与持续时间是否具有共同参数匹配包线。

M3 不直接证明产品安全、寿命、量产可行性或具体设备定型。

## 核心物理与工程问题

电池加热的一阶近似为：

```text
Q_bat ~= I_rms^2 * R_bat(T, SOC, f)
```

有频域阻抗数据时，应以 `Re(Z_bat(T, SOC, f))` 替代简单电阻近似。工程问题不只是温升，还包括实际电流路径、PWM/采样可实现性、电池充放电与热约束、电驱损耗、保护策略和参数身份一致性。
