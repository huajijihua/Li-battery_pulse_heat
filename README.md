## 当前核心判断

低温脉冲加热的第一性原理是利用电驱系统形成高频或准高频脉冲电流，通过电池内阻或阻抗实部产生热量：

```text
Q_bat ~= I_rms^2 * R_bat(T, SOC, f)
```

若取得目标电芯低温 EIS 或频域阻抗数据，应升级为：

```text
Q_bat ~= I_rms^2 * Re(Z_bat(T, SOC, f))
```

本项目的核心矛盾不是热源公式，而是快速升温所需的大 RMS 电流，同时受到电池安全、电驱可靠性、整车高压架构、控制权限和经济价值约束。只回答温升不够，必须同时回答电流路径、逆变器/电机可实现性、电池低温回充安全、电驱热与可靠性、整车控制使能和应用价值。


单包独立接入、双电机集中作用单支路、单支路独立加热等只能作为理论对照或历史方案，不应写成当前实车工程可行方案。

## 建模分层

1. L0/L0.5 零维电热筛选：判断温升量级、能耗、电流压力和方案排序。
2. L1 电流可实现性：验证逆变器、电流环、PWM、母线电压、最小脉宽、死区和采样是否能实现目标脉冲电流。
3. L2 电池安全边界：评估低温回充半周、析锂、BMS 限流、端电压和温度/SOC 窗口。
4. L3 电机与逆变器风险：评估绕组温升、退磁、逆变器结温、NVH、绝缘和寿命。
5. L4 整车控制与应用价值：评估使能、退出、降级、场景、经济性和替代方案价值。

下一阶段不宜直接跳到“复杂模型越大越好”。优先路线是：

```text
资料盘点
  -> 参数注册表
  -> L0.5 结果复核
  -> L1 电流可实现性
  -> AMESim/Simulink 系统模型
  -> KPI 审计
  -> 台架/供应商问题清单
```

## AMESim/Simulink方向

OpenCode 接手后，建议把 `零维脉冲加热仿真模型-AMEsim/` 作为 AMESim 主线骨架目录，但先建立注册表和接口契约：

- architecture：电池包、高压母线、逆变器、电机绕组、热管理、控制边界；
- module：电池等效电路、热容/散热、电机 R-L、逆变器损耗、母线/DC-link、控制器；
- connection：三包并联整体输出、双电机支路、母线与支路电流语义；
- parameter：参数名、单位、来源、适用范围、可信等级；
- variable：RMS/峰值/平均电流、支路电流、母线电流、发热功率、温度、SOC、保护标志；
- study：默认工况、敏感性工况、边界工况、失败工况。

MATLAB/Simulink 更适合表达控制策略、参数扫描、数据处理、优化和系统级自由建模；AMESim 更适合表达一维物理网络、专业组件库、部件连接、电热网络和工程系统骨架。自动生成的 `.ame` 只能称为结构骨架或工作副本，不能直接称为已标定模型。

## 当前不能声称

当前 L0.5 结果不能证明：

- 电池高频脉冲充放电安全；
- 不会析锂；
- BMS 允许低温回充半周；
- MCU 一定支持驻车堵转高频电流控制；
- 550 Arms 可持续用于整个预热过程；
- 电机不会退磁、不会产生不可接受 NVH、不会造成绝缘或热老化风险；
- 逆变器结温和器件寿命满足要求；
- AMESim/Simulink/COMSOL 任一骨架模型已经完成物理标定。

## 结论表达约束

项目结论必须区分：

- 已验证事实；
- 基于证据的推断；
- 待确认假设。

零维模型只能用于原理筛选和方向判断，不能直接证明整车安全可行、控制可实现或硬件寿命满足。阶段输出应优先服务工程决策：下一步该问谁、补什么参数、做哪个最小试验、哪个模型边界会改变结论。
