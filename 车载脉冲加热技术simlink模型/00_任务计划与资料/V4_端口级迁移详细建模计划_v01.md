## 3. V4 建模总策略

V4 不从 v03 的手写控制链继续堆功能。V4 采用三层结构：

```text
官方核心子系统层
  BatteryThermalManagementModel / mcb_pmsm_foc_qep_f28379d_SIUnit / PMSMDriveThermal

项目适配层
  端口适配、总线适配、单位适配、低温堵转脉冲命令、DC-link 对接、缺失限制标志

审计输出层
  KPI、日志、最小回归、UNKNOWN/NaN 限制输出、验证记录
```

V4-A 的占位骨架只保留为接口地图。每替换一个占位块，都必须满足：官方来源可追溯、端口读回可验证、`model_check(all)` 无 error、缺失数据不默认为通过。

## 4. 官方示例依赖准备

### 4.1 目标

消除直接按 `.slx` 路径加载官方示例时出现的初始化脚本/Simscape 组件找不到问题，建立可重复的官方示例读取环境。

### 4.2 操作步骤

1. 在 MATLAB 中对三个官方示例分别确认所在目录：
   - `C:\Users\ADMIN\Documents\MATLAB\Examples\R2025b\simscapeelectrical\BatteryThermalManagementExample`
   - `C:\Users\ADMIN\Documents\MATLAB\Examples\R2025b\mcb\SIUnitsExample`
   - `C:\Users\ADMIN\Documents\MATLAB\Examples\R2025b\simscapeelectrical\PMSMDriveThermalExample`
2. 用 `addpath` 临时加入对应示例目录，只在当前 MATLAB 会话中生效；不把这些路径写入项目永久启动脚本。
3. 重新执行 `model_overview(interfaces)`、`model_read(depth=1)`、`model_check(all)`。
4. 对 `BatteryThermalManagementModel.slx` 额外确认 `BatteryThermalManagementInitialization` 和 `BatteryModule` 可解析。
5. 对 `PMSMDriveThermal.slx` 额外确认 `PMSMDriveThermalData` 可解析。

### 4.3 验收

- 三个官方示例均可无初始化缺失警告读取。
- 记录官方示例的必要依赖文件和模型工作区变量。
- 仍不把官方示例整包复制进 Git，除非后续明确决定建立受控工作副本目录。

## 5. V4-B1 电驱热模型端口迁移

### 5.1 目标

优先让 `PMSMDriveThermal_Inverter_And_Motor` 不再输出 `NaN`，而是真实复用 `PMSMDriveThermal.slx` 的逆变器、电机、编码器、热模型和损耗日志。

### 5.2 操作步骤

1. 读取 `PMSMDriveThermal.slx` 顶层以下关键块：`Three-phase inverter`、`Thermal model`、`Encoder`、`Scopes/losses`、`PMSM controller`。
2. 确认 `Three-phase inverter` 输入 `G,+,-` 和输出 `~` 的物理域语义。
3. 确认官方模型内电池/DC 源与逆变器 DC 端连接方式，决定 V4 是否先保留官方 DC 源，还是先接项目 `DC_Link_And_Inverter_Interface`。
4. 先迁移电驱植物和热/损耗日志，不迁移官方 `PMSM controller` 为主控制器；其仅作为对照。
5. 建立 `Duty_Cycles -> gate signals G` 适配器，若官方逆变器需要六路门极而 MCB 输出三路 duty，则显式加入 PWM/gate adapter，并标注调制语义。
6. 暴露 `Feedbacks_sim`，至少包含相电流、转子位置/速度、可供 MCB 控制器消费的反馈总线。
7. 暴露 `DriveKPI`，至少包含电机相电流、逆变器损耗、电机铜损/铁损、定转子温度。

### 5.3 验收

- `PMSMDriveThermal_Inverter_And_Motor` 内不再以 `NaN` 作为默认输出。
- 零 duty 或零门极工况下无非物理电流漂移。
- 电驱热模型和损耗日志可读回。
- 若 gating/PWM 语义尚未完成，必须输出 `UNKNOWN` 限制，不允许解释为电驱可实现。

## 6. V4-B2 MCB SI Units FOC 控制器迁移

### 6.1 目标

把 `MCB_SIUnits_FOC_Controller` 从占位输出替换为官方 `Current Control` 子系统及必要的输入缩放、闭环电流控制和 `SVPWM` 链路。

### 6.2 操作步骤

1. 读取 `mcb_pmsm_foc_qep_f28379d_SIUnit.slx` 的 `Current Control`、`Control_System`、`Closed Loop Control`、`Current_Controllers`、`SVPWM`、`Input Scaling`、`Open Loop Start-Up`。
2. 隔离硬件部署相关块：C2000 I/O、SCI、LED、hardware init 只保留为非主线参考。
3. 明确 `Idq_ref` 的真实数据结构：标量、向量还是 bus；统一单位为 A。
4. 明确 `Feedbacks_sim` 的真实数据结构：相电流、位置、速度、使能信号等。
5. 建立 `PulseHeating_Command_And_Limits -> Idq_ref` 适配器。
6. 建立 `PMSMDriveThermal_Inverter_And_Motor -> Feedbacks_sim` 适配器。
7. 输出 `Duty_Cycles` 和 `ControlKPI`，其中 `ControlKPI` 至少包括 `tracking_error_rms_A`、调制指数、饱和标志、闭环使能状态。

### 6.3 验收

- `0 A` 命令下 duty 输出稳定在安全默认状态。
- 小电流低频命令下 `Id/Iq` 跟踪方向正确。
- `SVPWM` 链路保留，不能退化为 v03 手写最小 PI 控制链。
- 硬件部署相关块不进入主仿真路径。

## 7. V4-B3 电池热管理基线迁移

### 7.1 目标

把 `BatteryThermalManagement_BatteryPack` 从 `NaN` 占位替换为 `BatteryThermalManagementModel.slx` 的电池包、模块热网络、冷却控制、SOC/温度限制接口。

### 7.2 操作步骤

1. 读取 `Battery pack`、`Module 1-4`、`Coolant control`、`SOC limits`、`Drive profile`。
2. 确认 `Drive profile` 输出电流方向、单位和与电池正负端的对应关系。
3. 将 `Drive profile` 替换为项目 `BatteryLoad` 输入，但先保留官方电池包和冷却控制结构。
4. 暴露 `BatteryDC`，至少包含电池端电压、电流、SOC 和包温。
5. 暴露 `BatteryStatus`，至少包含包最高/最低温、SOC min/max、冷却流量、充电/放电限制缺口标志。
6. 低温初始条件只使用官方示例已有变量或明确 `ENGINEERING_DEFAULT`/`PLACEHOLDER` 标签，不引入历史车型参数。

### 7.3 验收

- `BatteryModule` 和初始化脚本可解析。
- 零负载下 SOC、温度、电压无非物理漂移。
- `BatteryStatus` 中缺失的 BMS/析锂/低温回充限制输出 `NaN` 或 `UNKNOWN`。

## 8. V4-B4 DC-link 与能量路径闭合

### 8.1 目标

让电池端、DC-link、逆变器、电机三相之间的电流和功率方向可审计。

### 8.2 操作步骤

1. 明确电池正负端到逆变器 DC `+/-` 的连接方式。
2. 明确 DC-link 电容是否复用官方电驱模型默认值，或作为项目适配层参数。
3. 加入电池端电流、DC-link 电流、DC-link 电压测量。
4. 建立电池负载反馈 `BatteryLoad`，使电池包输入和逆变器能量消耗一致。
5. 对回充半周方向单独标注，不能把双向脉冲平均值当作单向放电。

### 8.3 验收

- 小电流工况下电池端电流、DC-link 电流、电机相电流方向一致且可解释。
- `P_battery_terminal + P_inverter_loss + P_motor_loss` 的符号和量级可审计。
- 若能量闭合未通过，不进入 V4-C 限制接口结论。

## 9. V4-C 限制接口

### 9.1 必须输出

- `voltage_margin`
- `modulation_index`
- `motor_phase_peak_margin`
- `motor_phase_rms_margin`
- `battery_charge_limit_margin`
- `battery_temperature_margin`
- `coolant_flow_margin`
- `unknown_safety_margin`
- `limiting_factor`

### 9.2 规则

所有缺失供应商、BMS、MCU、逆变器、电机退磁和寿命边界的数据，统一输出 `NaN` 或 `UNKNOWN`。不得用经验默认值把限制判为通过。

## 10. V4-D KPI 与最小回归

### 10.1 KPI 字段

至少输出：

- `I_battery_terminal_rms_A`
- `I_dc_link_rms_A`
- `I_motor_phase_rms_A`
- `I_heat_equiv_rms_A`
- `V_battery_terminal_V`
- `V_dc_link_V`
- `P_battery_heat_W`
- `P_motor_copper_W`
- `P_motor_iron_W`
- `P_inverter_loss_W`
- `T_battery_C`
- `T_motor_stator_C`
- `T_motor_rotor_C`
- `SOC_cell`
- `tracking_error_rms_A`
- `limiting_factor`

### 10.2 最小回归矩阵

1. 零命令：所有主电流接近 0，温度/SOC 无非物理漂移。
2. 低频小电流：方向正确，控制链闭环可跟踪。
3. 诊断频率小电流：记录跟踪误差，不要求通过产品指标。
4. 诊断频率中等电流：检查调制、电压、电机相电流和热模型响应。
5. 缺失数据边界工况：限制接口输出 `UNKNOWN`/`NaN`，不输出 `PASS`。

## 12. 停止条件

出现以下情况时停止建模并先记录问题：

1. 官方示例初始化脚本或 Simscape 组件无法解析。
2. MCB `Duty_Cycles` 无法明确映射到 PMSMDriveThermal 逆变器门极或调制接口。
3. 电池端和 DC-link 能量方向无法闭合。
4. 为了跑通模型需要引入历史产品参数作为默认值。
5. `model_check(all)` 出现 error 级结构问题且无法在当前子阶段闭环修复。
