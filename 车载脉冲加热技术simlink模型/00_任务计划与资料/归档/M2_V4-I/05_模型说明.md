## 1. 项目定位

本项目是**通用车载脉冲加热系统仿真平台**，不是具体产品仿真或特定车型仿真。模型服务于结构规范、能量路径、控制链、限制接口、KPI 语义和验证闭环。

默认参数来源：`OFFICIAL`（MathWorks 官方）、`PLATFORM_DEFAULT`（平台自洽）、`ENGINEERING_DEFAULT`（工程经验）、`PLACEHOLDER`（明确缺口）。公司历史参数和 L0.5 粗筛参数不进入默认模型。详见 `00_任务计划与资料/剥离原则与参数分层规则_v01.md`。

## 3. V4-D2 KPI 接口（历史阶段快照）

V4-D1 新增 `SystemKPI` 24 维接口；V4-D2 保持字段顺序不变，只替换部分字段来源。以下 UNKNOWN 说明保留为 V4-D2 阶段快照，当前 V4-I 的字段来源和限制以第 11、12、13、14、15 节及 `V4_能力审计矩阵_v01.md` 为准：

```text
SystemKPI = [
  I_battery_terminal_rms_A;
  I_dc_link_rms_A;
  I_motor_phase_rms_A;
  I_heat_equiv_rms_A;
  V_battery_terminal_V;
  V_dc_link_V;
  modulation_index;
  voltage_margin;
  P_battery_heat_W;
  P_motor_copper_W;
  P_motor_iron_W;
  P_inverter_loss_W;
  T_battery_C;
  T_motor_stator_C;
  T_motor_rotor_C;
  tracking_error_rms_A;
  control_saturation_flag;
  closed_loop_enable;
  current_limit_margin;
  thermal_margin;
  battery_charge_limit_margin;
  unknown_safety_margin;
  limiting_factor_code;
  kpi_status_code
]
```

当前字段来源：
- `I_motor_phase_rms_A = DriveKPI[1] = sqrt((ia^2 + ib^2 + ic^2)/3)`，来自 `From_i_abc -> Compute_Current_KPI_V4D2`；当前是单个仿真样本的三相幅值代理，不是时间窗口 RMS
- `V_dc_link_V = DCBus`
- `modulation_index = ControlKPI[2] = max(abs(2*duty_abc - 1))`，来自 `Control_System/Duty_abc -> Compute_Duty_KPI_V4D2`
- `voltage_margin = V_dc*(1-mod_index)`，来自 `Compute_Voltage_Margin_V4G`；该公式仍是平台裕度代理
- `P_motor_copper_W = DriveKPI[3]`
- `P_motor_iron_W = DriveKPI[4]`
- `P_inverter_loss_W = DriveKPI[5]`
- `T_battery_C`（历史字段名，实际单位为 K）`= BatteryStatus[1]`
- `T_motor_stator_C = DriveKPI[6]`
- `T_motor_rotor_C = DriveKPI[7]`
- `tracking_error_rms_A = ControlKPI[1]`
- `control_saturation_flag = ControlKPI[3] = any(duty_abc <= 0 | duty_abc >= 1)`
- `closed_loop_enable = ControlKPI[4]`
- `battery_charge_limit_margin = BatteryStatus[6]`
- `unknown_safety_margin = safety_margin_V4F`，来自电池限流/风险代理
- `limiting_factor_code = Compute_Limiting_Factor_V4H`，当前 SAFE 工况为 0
- `kpi_status_code` 仍沿用 V4-D2 状态码语义；24/24 numeric 不等于物理通过
- 电池端电流、DC-link RMS 和热等效电流当前由 `Compute_I_rms_V4G` 的瞬时绝对值代理提供，不是时间窗口 RMS

`SystemKPI` 顶层输出当前接到 `Terminate_SystemKPI`。`03_脚本/run_v4d_minimal_regression.m` 检查接口契约、模型工作区参数和 0.001 s 烟测；`03_脚本/run_v4d2_case_matrix_regression.m` 通过临时 `To Workspace` 块读取 `SystemKPI/LimitStatus`，脚本结束后删除临时块，不导出文件。

### 8.4 Simscape 传感器端口布局

在 R2025b 中，`fl_lib` 的 Current Sensor 和 Voltage Sensor 的物理端口布局为：

| 块 | LConn1 | RConn1 | RConn2 |
|---|---|---|---|
| Current Sensor | p (+电气) | I (物理信号输出) | n (-电气) |
| Voltage Sensor | p (+电气) | V (物理信号输出) | n (-电气) |

**RConn1 是信号端口，不是电气端口。** 连接电气 n 端时必须用 RConn2，否则会报"物理建模域连接规则"错误。

## 9. 信号验证规范（防止 Goto/From scope 类问题）

### 9.1 跨子系统 Goto/From 规则

- 新增 Goto/From 跨子系统信号时，**必须**检查 `TagVisibility`：
  - `local`（默认）：仅同一子系统内可见
  - `global`：全模型可见
- 如果 Goto 和 From 不在同一个子系统内，**必须**设为 `global`
- 每次 `model_edit` 或脚本新增 Goto 块后，用以下命令审计：
  ```matlab
  gotoBlocks = find_system(modelName, 'BlockType', 'Goto');
  for i = 1:numel(gotoBlocks)
      tag = get_param(gotoBlocks{i}, 'GotoTag');
      scope = get_param(gotoBlocks{i}, 'TagVisibility');
      fprintf('  %s: tag="%s" scope=%s\n', gotoBlocks{i}, tag, scope);
  end
  ```

### 9.2 信号验证双重确认

读取仿真结果时，**不能只依赖一种捕获方式**。以下两种方式之一为正时仍需交叉验证：

| 方式 | 优点 | 缺点 |
|---|---|---|
| Simscape log (`simlog.XXX.Y.v.series.values`) | 直接读取物理量，不经过信号链路 | 需要 `SimscapeLogType=all` |
| To Workspace / Outport | 读取 Simulink 信号链路末端 | 受 Goto/From scope、PS-Simulink Converter 连接等影响 |

**规则：当 Simscape log 和 To Workspace 结果不一致时，以 Simscape log 为准，并排查信号链路断点。**

### 9.3 零值检查

仿真后如果任何 KPI 字段或传感器读数为 0，**必须**用 Simscape log 交叉验证，不能直接假设"物理上确实是 0"。0 值的常见原因：

1. Goto/From tag scope 不匹配（local vs global）
2. PS-Simulink Converter 连接到错误的传感器端口（如 RConn1 信号端口而非 RConn2 电气端口）
3. From 块找不到匹配的 Goto 时，Simulink 静默返回 0，不报错
4. To Workspace `SaveFormat=Array` 在单步仿真时只保存最终值

### 12.3 Charge_limit / Discharge_limit 公式

```text
Charge_limit = I_base * f(T) * g(SOC)
  I_base = cellAH * 1C；当前限流块 Constant 使用 27A
  f(T): T >= 273.15K (0C) -> 1.0; T < 273.15K -> max(0.1, (T-253.15)/20)
  g(SOC): SOC < 0.8 -> 1.0; SOC >= 0.8 -> max(0.1, (1-SOC)/0.2)

Discharge_limit = I_base * h(T)
  h(T): T < 263.15K (-10C) -> 0.5; T >= 263.15K -> 1.0
```

来源：LITERATURE（文献趋势）+ ENGINEERING_DEFAULT（通用公式参数）。注意：当前电池模块 `AH=20`，而限流块 `cellAH_param=27`，容量输入尚未统一；27A 不能视为由 active 电池模块容量自动推导。
文献依据：
- 析锂判据：负极固液相电位差 < 0 V vs Li/Li+（4 篇文献一致）
- 低温充电安全温度阈值：0°C（文献 1/3/4）
- 安全电流限制：文献 1 Table A1（50% SOC, 1000Hz, -20~5°C 范围）

  0 = Safe (T >= 0C or I_pack < 0.8*charge_limit)
  1 = Warning (T < 0C and safety_margin < 20% of charge_limit)

### 15.4 V4-I I3/I4/I5 已知限制

1. **残差未闭合**：公式约 86.4%，simlog 损耗口径约 92.4%；需要分离有功/无功、储能项和符号约定，或改用统一时间窗口的能量积分审计
2. **P_batt_heat 口径未闭合**：当前块使用瞬时 `I_pack² × R_pack`，I4 报告的是该信号的时间均值；不能把它称为 RMS 发热。需先定义 `I_rms=sqrt(mean(I_pack²))` 和阻抗频率适用范围，再计算热功率
3. **PMSM.power_dissipated 不含铜损**：PMSM 的 power_dissipated 只含铁损和机械损耗，铜损由外部公式 `1.5*Rs*(Id²+Iq²)` 计算
4. **protection_action 瞬态触发**：启动时 I_pack 超调导致 safety_margin < 0，触发停脉冲。这是 PI 控制器启动瞬态，不是稳态问题
