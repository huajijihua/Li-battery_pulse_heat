## 3. 复用边界

### 3.1 电池基线

固定复用 `BatteryThermalManagementModel.slx` 的电池包和热管理骨架。

必须保留：

1. `Battery pack` 作为电池系统骨架。
2. `Module 1-4` 的模块化组织方式。
3. `Coolant control` 中基于温差、包温和 SOC 限制的控制结构。
4. 电芯/模块/包温度、SOC 和热端口日志口径。

允许适配：

1. 将 `Drive profile` 替换为脉冲加热电流/功率工况输入。
2. 将默认温度、SOC、环境和冷却边界改为低温脉冲工况。
3. 增加 `battery_pulse_current_margin`、`battery_charge_limit_margin`、`unknown_safety_margin` 等缺口标志。

不得修改：

1. 不把 v03 单个 BEC 缩放模型作为 V4 默认电池基线。
2. 不把公司历史电池参数写入默认模型。
3. 不把 `BatteryDCFastCharging.slx` 作为主电池模型；它只可补充低温充电限流和保护逻辑。

### 3.2 控制器基线

固定复用 `mcb_pmsm_foc_qep_f28379d_SIUnit.slx` 的 SI 单位 FOC 控制结构。

必须保留：

1. `Current Control` 下的 `Control_System`。
2. `Closed Loop Control`、`Current_Controllers`、`PI_Controller_Id`、`PI_Controller_Iq`。
3. `SVPWM` 调制链。
4. `Open Loop Start-Up` 作为零速/低速启动风险参考。
5. `Input Scaling` 和电流/位置/速度计算的单位处理思路。

允许适配：

1. 将速度/扭矩外环替换或旁路为堵转脉冲电流命令。
2. 增加 `Id/Iq` 脉冲幅值、频率、占空比和相位补偿输入。
3. 把硬件 C2000 I/O、SCI、LED、host 通信等部署相关部分隔离为非主线。
4. 将控制输出对接到 `PMSMDriveThermal.slx` 的三相逆变器或平均值逆变器接口。

不得修改：

1. 不再手写 v03 最小 Clarke/Park/PI/SVPWM 作为主控制器。
2. 不删除官方 FOC 结构后只保留几个 Gain 和 Saturation。
3. 不把控制器跑通等同于 MCU 量产权限已开放。

### 3.3 电机/电驱基线

固定复用 `PMSMDriveThermal.slx` 的电机、电驱和热模型骨架。

必须保留：

1. `Three-phase inverter`。
2. `Thermal model`。
3. `Encoder`。
4. `Scopes/losses` 中损耗日志组织方式。
5. PMSM 与逆变器、电机热端口、损耗输出之间的系统级连接。

允许适配：

1. 将原牵引速度/扭矩工况改为驻车堵转或近零速脉冲电流工况。
2. 将示例控制器接口改接 MCB SI Units FOC 控制器。
3. 增加电池端、DC-link、电机相、电机热和逆变器限制日志。
4. 用 `PMSMDriveThermalDQ.slx` 只做 DQ frame/步长/高频数值稳定性对照。

不得修改：

1. 不把 `PMSMDrive.slx` 非热版作为主电机基线；它只作无热对照。
2. 不把 v03 工程默认 PMSM 参数作为 V4 默认电机参数。
3. 不跳过热模型和损耗日志，直接用理想 PMSM 块替代。

## 4. V4 模型结构

V4 工作副本建议命名：

```text
车载脉冲加热技术simlink模型/02_模型/pulse_heating_official_spine_v04.slx
```

V4 顶层结构固定为：

```text
BatteryThermalManagement_BatteryPack
    -> DC_Link_And_Inverter_Interface
    -> PMSMDriveThermal_Inverter_And_Motor
    -> MCB_SIUnits_FOC_Controller
    -> PulseHeating_Command_And_Limits
    -> KPI_And_Logging
```

其中：

- 电池包和热边界来自 `BatteryThermalManagementModel.slx`。
- 逆变器、电机、编码器、热模型和损耗日志来自 `PMSMDriveThermal.slx`。
- FOC 电流环、SVPWM 和启动结构来自 `mcb_pmsm_foc_qep_f28379d_SIUnit.slx`。
- 脉冲加热命令、限制接口和 KPI 是本项目新增薄层，不替代官方核心子系统。

## 5. V4-A 到 V4-D 执行计划

### V4-A 官方基线工作副本

目标：建立工作副本，不做行为结论。

步骤：

1. 复制三个官方示例到任务工作区或通过 Simulink Project 引用，记录来源路径。
2. 建立 `pulse_heating_official_spine_v04.slx`。
3. 按官方子系统边界导入或引用三大基线。
4. 执行 `model_overview(detail=interfaces)`、`model_read(depth=1)`、`model_check(all)`。

验收：

- 三大基线来源和子系统边界可追溯。
- `model_check(all)` 无关键错误。
- 未导入公司历史参数。

### V4-B 堵转脉冲工况适配

目标：把牵引/充电工况改为低温驻车堵转脉冲加热工况。

步骤：

1. 控制器输入改为 `Id_ref/Iq_ref` 或脉冲电流幅值、频率、占空比。
2. 电机机械端改为堵转或近零速约束。
3. 电池驱动输入改为与 DC-link/逆变器一致的脉冲负载。
4. 保留官方采样、单位缩放和限幅结构。

验收：

- `0 A` 工况无非物理漂移。
- 小电流工况电机相电流、电池端电流和 DC-link 电流方向一致。
- 所有脉冲输入都有单位、来源标签和默认值说明。

### V4-C 限制接口

目标：输出 L1 电流可实现性所需限制来源。

必须实现：

- `voltage_margin`
- `modulation_index`
- `motor_phase_peak_margin`
- `motor_phase_rms_margin`
- `battery_charge_limit_margin`
- `battery_temperature_margin`
- `coolant_flow_margin`
- `unknown_safety_margin`
- `limiting_factor`

缺失供应商/BMS/安全数据时输出 `NaN` 或 `UNKNOWN`，不允许默认为通过。

### V4-D KPI 与最小回归

目标：形成可复用、可审计的 V4 输出。

KPI 必须至少包含：

- `I_battery_terminal_rms_A`
- `I_dc_link_rms_A`
- `I_motor_phase_rms_A`
- `I_heat_equiv_rms_A`
- `V_battery_terminal_V`
- `V_dc_link_V`
- `P_battery_heat_W`
- `P_motor_copper_W`
- `P_motor_iron_W`
- `P_inverter_loss_W`
- `T_battery_C`
- `T_motor_stator_C`
- `T_motor_rotor_C`
- `SOC_cell`
- `tracking_error_rms_A`
- `limiting_factor`

最小回归：

1. 零命令。
2. 低频小电流。
3. 诊断频率小电流。
4. 诊断频率中等电流。
5. 缺失数据边界工况。

## 7. 完成定义

只有同时满足以下条件，才能称为 V4 基线完成：

1. 三大官方示例基线均有来源路径、模型名和子系统边界记录。
2. V4 工作副本完成 `model_overview`、`model_read(depth=1)` 和 `model_check(all)`。
3. 控制器不再使用 v03 手工最小 FOC 作为主线。
4. 电池不再使用单 BEC 缩放模型作为主线默认电池包。
5. 电机/电驱不再使用工程默认 PMSM 参数作为主线默认参数。
6. KPI 区分电池端、DC-link、电机相和热等效电流。
7. 所有缺失的安全、BMS、MCU、逆变器和电机边界均以 `UNKNOWN`/`NaN` 输出，不被解释为通过。
