日期：2026-07-07
范围：单电池包-电机控制器-单电机-车载脉冲加热技术系统电路的通用规范化 Simscape 模型
基于：当前模型审计 + 官方库规范案例搜索 + Battery Pack Builder 验证

本文件识别当前模型在模块内容上的规范化缺陷，给出基于官方成熟案例的修复路线。核心原则：**有模块必有参考，不盲目、想当然地用 MATLAB Function 粗制滥造**。

## 1. 当前模型规范化审计

### 1.1 审计方法

逐块检查 ReferenceBlock（来源库）、BlockType、参数配置和物理语义，判断每个块是否有官方规范参考、是否正确使用、是否表达了的物理语义。

### 1.2 块来源审计结果

| 块名 | 来源 | 规范性 | 问题 |

### 1.3 关键缺陷总结

**缺陷 1：电池包建模不规范**
- 当前：用单个 BEC 块（单电芯）充当整个 252S 电池包
- 问题：OCV=3.3V（单体）不是 811V（电池包），R0=8.5mΩ（单体）不是 0.58Ω（252S），热容不匹配
- 后果：发热功率计算错误，温升不正确，无法表达电池包串并联拓扑

**缺陷 2：逆变器建模不规范**
- 当前：用 Controlled Voltage Source（理想受控源）替代逆变器
- 问题：无调制策略、无死区、无最小脉宽、无开关损耗、无 DC 母线接口
- 后果：无法判断电流可实现性，P_inverter=0 不反映实际损耗

**缺陷 3：控制链完全缺失**
- 当前：用 Sine Wave 直接生成 Vabc 正弦波
- 问题：无 FOC 电流闭环（Clarke/Park/PI/逆Park）、无 PWM/SVPWM 调制、无脉冲命令发生器
- 后果：I_target 与 I_actual 无闭环关系，无法判断电流可实现性

**缺陷 4：无脉冲加热特征**
- 当前：施加连续正弦波，不是脉冲加热特有的高频双向脉冲
- 问题：脉冲加热需要参数化的高频双向脉冲（频率、占空比、幅值可切换），不是简单正弦波

## 2. 官方规范案例识别

### 2.1 电池包规范建模：Battery Pack Builder

**工具链**：`Cell → ParallelAssembly → Module → ModuleAssembly → Pack → buildBattery`

**已验证可用**（2026-07-07 MATLAB R2025b 实测）：

```matlab
% 创建 Cell（使用 Battery Equivalent Circuit 块，暴露热端口）
cmo = simscape.battery.builder.CellModelBlock(...
    CellModelBlockPath='batt_lib/Cells/Battery Equivalent Circuit');
bp = cmo.BlockParameters;
bp.ThermalModel = 3;  % LumpedThermalMass（枚举索引：1=TempIndependent, 2=ConstantTemperature, 3=LumpedThermalMass, 4=HeightDistributedMass）
cmo.BlockParameters = bp;

cellObj = simscape.battery.builder.Cell;
cellObj.CellModelOptions = cmo;

% 创建 252S1P 电池包
paObj = simscape.battery.builder.ParallelAssembly(Cell=cellObj, NumParallelCells=1);
modObj = simscape.battery.builder.Module(ParallelAssembly=paObj, NumSeriesAssemblies=252, ModelResolution="Lumped");
maObj = simscape.battery.builder.ModuleAssembly(Module=modObj);
packObj = simscape.battery.builder.Pack(ModuleAssembly=maObj);
packObj.AmbientThermalPath = "CellBasedThermalResistance";

% 生成 Simulink 库
simscape.battery.builder.buildBattery(packObj, LibraryName='battery_pack_252S1P', Directory='...');
```

**生成结果**：`battery_pack_252S1P.slx` + `battery_pack_252S1P_lib.slx`

**规范电池包端口**：
- `+`（电气正）、`-`（电气负）、`AmbH`（环境热端口）
- 输出信号：batteryCurrent、batteryTemperature、batteryVoltage、numCycles、socCell、socParallelAssembly、vParallelAssembly
- 内部层次：Pack1 → ModuleAssembly1 → Module1（252S Lumped）

**同一性识别**：电池包 = Pack Builder 生成的规范模型，不是单个 BEC 块

### 2.2 逆变器规范建模

**官方库可选块**：

| 块名 | 库路径 | 类型 | 适用场景 |
|---|---|---|---|
| Average-Value Inverter (Three-Phase) | ee_lib/Semiconductors & Converters/Converters/ | SimscapeBlock | 平均值，无信号控制端口 |
| Average-Value Voltage Source Converter (Three-Phase) | ee_lib/Semiconductors & Converters/Converters/ | SimscapeBlock | 平均值，有 input_option=ModWave/MagAngle |
| Converter (Three-Phase) | ee_lib/Semiconductors & Converters/Converters/ | SimscapeBlock | 两电平变换器 |
| H-Bridge | ee_lib/Semiconductors & Converters/Converters/ | SimscapeBlock | H 桥 |
| IGBT (Ideal, Switching) | ee_lib/Semiconductors & Converters/ | SimscapeBlock | 开关级 IGBT |
| MOSFET (Ideal, Switching) | ee_lib/Semiconductors & Converters/ | SimscapeBlock | 开关级 MOSFET |
| Universal Bridge | sps*Lib (SPS 域) | — | 通用桥（SPS 域，非 Simscape 物理） |

**GitHub 官方参考**：
- Design-motor-controllers-with-Simscape-Electrical（B13，275星）：FOC/SVPWM/电流环完整实现，控制层主参考
- Simscape-Battery-Electric-Vehicle-Model（B11，151星）：BEV 完整电驱链路

**同一性识别**：逆变器 = Average-Value Voltage Source Converter（平均值+控制信号输入）或开关级 IGBT/MOSFET 桥，不是理想受控电压源

**修复方案**：
- 第一版修复：用 `Average-Value Voltage Source Converter (Three-Phase)`，设 `input_option=ModWave`，接受调制波控制信号
- 后续升级：用 IGBT (Ideal, Switching) 组建开关级三相桥

### 2.3 电机控制器（FOC）规范建模

**官方库可选块**：

| 块名 | 库路径 | 类型 | 说明 |
|---|---|---|---|
| spsFieldOrientedControllerLib | sps*Lib (SPS 域) | 空库 | FOC 控制器（SPS 域，空库不可直接用） |
| elec_conv_Field_OrientedControlInd | elec_conv_* (SPS 域) | SubSystem | 完整 FOC 驱动系统（含电机+逆变器+控制一体） |
| mcbfoclib | mcb/mcbblocks | 存在 | MCB FOC 块（接口偏代码生成导向） |
| mcbdeadtimecomplib | mcb/mcbblocks | 存在 | 死区补偿块 |
| spsabctodq0Lib / spsdq0toabcLib | sps*Lib (SPS 域) | 空库 | Clarke/Park 变换 |

**关键发现**：Simscape 物理网络库中没有独立的规范 FOC 控制器块。FOC 控制链需要用 Simulink 信号流标准块自建，参考 GitHub B13 的实现。

**同一性识别**：电机控制器 = 信号流 FOC（Clarke/Park/PI/逆Park/PWM），参考 GitHub B13 Design-motor-controllers-with-Simscape-Electrical 的实现

**修复方案**：
- 用 Simulink 信号流标准块（Discrete PI Controller、Trigonometric Function、Math Operations）自建 FOC
- 参考《建模依据清单》第3.1节的方程链（来自 NXP/TI/Microchip 官方文档）
- 参考《逆变器调幅口径》第3.3节的推荐分层结构

### 2.4 PMSM 规范建模

**当前状态**：✓ 已规范
- 块来源：`ee_lib/Electromechanical/Permanent Magnet/PMSM`
- 端口：三相电气 + 机械 R/S
- 参数：Ld/Lq/Rs/nPolePairs/pm_flux_linkage/J/stator_thermal_mass/rotor_thermal_mass

**待改进**：
- 参数多为 PLACEHOLDER，需用官方示例参数或文献值填充
- pm_flux_linkage=0.05 Wb 是占位值
- J=0.01 是占位值

### 2.5 脉冲加热特征规范建模

**当前状态**：❌ 缺失
- 当前用连续正弦波，不是脉冲加热特有的高频双向脉冲

**文献依据**（来自《建模依据清单》）：
- CN114789679A：Id/Iq 闭环调幅，PI 输出 Ud/Uq，SVPWM 驱动
- Voltage-Feedback：零矢量占空比 d0 调幅，双 PI 电压反馈
- Drive circuitry：CMI/DMSI 驱动电路加热模式

**同一性识别**：脉冲命令发生器 = 参数化高频双向脉冲（频率、占空比、幅值可切换），生成 Id_ref/Iq_ref，不是简单 Sine Wave

**修复方案**：
- 用 Signal Builder 或 Signal Generator + Switch 组建参数化脉冲命令发生器
- 频率、占空比、幅值通过变量参数化，可切换不同工况
- 参考《逆变器调幅口径》第5节统一输出字段

## 3. 缺陷修复路线

### 3.1 修复优先级

| 优先级 | 缺陷 | 修复方式 | 影响 |
|---|---|---|---|
| P0 | 电池包建模不规范 | 用 Battery Pack Builder 生成 252S1P 规范电池包 | 发热/温升/电压全部依赖此修复 |
| P1 | 控制链完全缺失 | 用信号流自建 FOC（Clarke/Park/PI/逆Park/PWM） | 电流闭环/可实现性判断依赖此修复 |
| P1 | 逆变器建模不规范 | 用 Average-Value Voltage Source Converter 替换 Controlled VS | 调制/损耗/DC母线依赖此修复 |
| P2 | 脉冲加热特征缺失 | 组建参数化脉冲命令发生器 | 工况切换/KPI 对齐依赖此修复 |
| P3 | PMSM 参数 PLACEHOLDER | 用官方示例参数或文献值填充 | 精度提升 |

### 3.2 修复步骤

**步骤 1：替换电池包（P0）**
1. 将 `battery_pack_252S1P.slx` 中的 Pack1 子系统拖入主模型
2. 删除当前的 Battery_ECM 块
3. 连接 Pack1 的 +、-、AmbH 端口
4. 设置 Pack1 的 Cell 参数（OCV/R0/热容）用官方默认值
5. 验证：电池包电压 ≈ 252 × 3.3V ≈ 830V

**步骤 2：替换逆变器（P1）**
1. 删除当前的 Controlled_VS 块
2. 添加 `Average-Value Voltage Source Converter (Three-Phase)` 块
3. 设置 `input_option=ModWave`
4. 连接 DC 母线（+、-）和三相输出
5. 连接调制波控制信号输入

**步骤 3：添加 FOC 控制链（P1）**
1. 添加 Clarke 变换（Iabc → Ialpha/Ibeta）
2. 添加 Park 变换（Ialpha/Ibeta → Id/Iq）
3. 添加 Discrete PI Controller（Id_ref vs Id_fb → Vd_ref）
4. 添加 Discrete PI Controller（Iq_ref vs Iq_fb → Vq_ref）
5. 添加逆 Park 变换（Vd/Vq → Valpha/Vbeta）
6. 添加 PWM Generator（Valpha/Vbeta → 调制波）
7. 连接到逆变器控制输入

**步骤 4：添加脉冲命令发生器（P2）**
1. 创建参数化脉冲命令发生器（频率、占空比、幅值可切换）
2. 生成 Id_ref（高频双向脉冲）和 Iq_ref（=0，零转矩）
3. 连接到 FOC 控制链

**步骤 5：填充 PMSM 参数（P3）**
1. 用 ee_lib PMSM 默认参数或文献典型值替换 PLACEHOLDER
2. 标注参数来源和可信等级

## 4. 规范化建模原则

### 4.1 有模块必有参考

每个 Simscape 物理块必须有明确的官方库来源（ReferenceBlock）。自建块仅限于：
- 信号流控制逻辑（FOC/PI/变换）——Simulink 标准库块组合
- 脉冲命令发生器——Signal Builder/Generator 组合
- 数据记录——To Workspace/Scope

### 4.2 设备同一性识别

| 设备 | 规范建模方式 | 官方参考 | 禁止方式 |
|---|---|---|---|
| 电池包 | Battery Pack Builder (Cell→PA→Module→MA→Pack) | batt_lib + buildBattery | 用单个 BEC 块充当电池包 |
| 逆变器 | Average-Value VSC 或 IGBT/MOSFET 桥 | ee_lib/Semiconductors & Converters | 用理想受控电压源替代 |
| PMSM | ee_lib PMSM 块 | ee_lib/Electromechanical | 用 MATLAB Function 实现 |
| FOC 控制器 | Simulink 信号流标准块组合 | GitHub B13 + 文献方程 | 用 MATLAB Function 粗制滥造 |
| 电流传感器 | ee_lib Current Sensor (Three-Phase) | ee_lib/Sensors | 自建电流测量 |
| 热模型 | BEC 自带 LumpedThermalMass + Convective Heat Transfer | batt_lib + fl_lib | 自建热模型 |
| 脉冲命令 | Signal Builder/Generator 参数化 | 文献专利 | 用固定 Sine Wave |

### 4.3 参数来源标注

每个参数必须标注来源：
- `[OFFICIAL]`：官方库默认值或官方示例参数
- `[LITERATURE]`：文献典型值
- `[L0.5]`：从 L0.5 零维模型继承
- `[PLACEHOLDER]`：占位假设，待实验数据替换

## 7. 2026-07-07 GPT审计后的强制收敛口径

### 7.1 当前不能再使用的表述

- 不再称 `pulse_heating_single_pack_v01.slx` 为“当前规范模型”或“闭环模型”。它只是开环物质流历史基线。
- 不再称 `pulse_heating_v02.slx` 为“已完成电池包替换模型”。它是未连通草稿。
- 不再把相电流 RMS 直接等同为电池包电流 RMS 来计算电池发热，除非在结果表中明确标注为占位 sanity check。
- 不再把 Controlled Voltage Source 或 Controlled Current Source 称为逆变器规范模型。它们只能作为中间受控源或测试夹具。

### 7.2 下一轮建模门槛

下一轮 `.slx` 修改完成后必须满足以下最低门槛，才允许写入“模型已走通”：

1. `model_check('all')` 无关键物理端口未连接、无 dangling line。
2. 电池包 `+/-/AmbH` 与 DC-link、热边界正确连接，电池电压、电流、温度、SOC 输出可记录。
3. 逆变器阶段至少选择一种明确口径：平均值 VSC、开关级桥，或临时受控源测试夹具；文档与模型块必须一致。
4. 控制链至少实现 `Id_ref/Iq_ref -> PI限幅 -> Vd/Vq -> abc电压或调制信号`，且有饱和/限流占位接口。
5. KPI 中必须区分电池端电流、母线电流、电机相电流和热等效电流，不允许单一 `I_actual_rms_A` 复用所有物理含义。
6. 首次仿真只要求短时走通和变量非空，不输出工程可行结论。
