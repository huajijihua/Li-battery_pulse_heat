# V4 建模主计划 v01

日期：2026-07-10

本文件合并了原 `V4_官方案例复用建模规划_v01.md`、`V4_建模前点检审计与实施步骤计划_v01.md` 和 `V4_端口级迁移详细建模计划_v01.md`，是 V4 建模的唯一主计划入口。

## 1. V4 定位

V4 是本项目从 v03 诊断沙箱转向 MathWorks 官方案例骨架复用的主线规划。V4 不再从空白模型或手工拼接库块开始，而是固定复用三个已经在本机打开、经 MCP 只读确认的官方示例：

| 设备 | V4 固定复用基线 | 本地模型路径 | 用途 |
|---|---|---|---|
| 电池 | Battery Thermal Management | `...\BatteryThermalManagementExample\BatteryThermalManagementModel.slx` | 电池包、模块热网络、冷却控制和 SOC/温度限制接口 |
| 控制器 | Motor Control Blockset SI Units PMSM FOC | `...\SIUnitsExample\mcb_pmsm_foc_qep_f28379d_SIUnit.slx` | SI 单位 FOC、电流环、SVPWM、开环启动和仿真植物接口 |
| 电机/电驱 | PMSM Drive with Thermal Model | `...\PMSMDriveThermalExample\PMSMDriveThermal.slx` | PMSM、三相逆变器、编码器、热模型、损耗日志和电驱系统骨架 |

这三个基线在 V4 中不再作为候选项讨论，除非本机许可证或模型文件损坏导致不可用。其他官方示例只作为补充参考，不替代这三条主线。

## 3. 复用边界

### 3.1 电池基线

固定复用 `BatteryThermalManagementModel.slx` 的电池包和热管理骨架。

必须保留：`Battery pack`、`Module 1-4`、`Coolant control`、电芯/模块/包温度/SOC/热端口日志口径。

允许适配：将 `Drive profile` 替换为脉冲加热电流/功率工况输入；将默认温度/SOC/环境/冷却边界改为低温脉冲工况；增加缺口标志。

不得修改：不把 v03 单个 BEC 缩放模型作为 V4 默认电池基线；不把公司历史电池参数写入默认模型；不把 `BatteryDCFastCharging.slx` 作为主电池模型。

### 3.2 控制器基线

固定复用 `mcb_pmsm_foc_qep_f28379d_SIUnit.slx` 的 SI 单位 FOC 控制结构。

必须保留：`Current Control` 下的 `Control_System`、`Closed Loop Control`、`Current_Controllers`、`PI_Controller_Id/Iq`、`SVPWM`、`Open Loop Start-Up`、`Input Scaling`。

允许适配：将速度/扭矩外环替换或旁路为堵转脉冲电流命令；增加 `Id/Iq` 脉冲幅值/频率/占空比/相位补偿输入；隔离硬件 C2000 I/O/SCI/LED/通信；将控制输出对接到 `PMSMDriveThermal.slx` 的三相逆变器。

不得修改：不再手写 v03 最小 Clarke/Park/PI/SVPWM 作为主控制器；不删除官方 FOC 结构后只保留几个 Gain 和 Saturation；不把控制器跑通等同于 MCU 量产权限已开放。

### 3.3 电机/电驱基线

固定复用 `PMSMDriveThermal.slx` 的电机、电驱和热模型骨架。

必须保留：`Three-phase inverter`、`Thermal model`、`Encoder`、`Scopes/losses`、PMSM 与逆变器/热端口/损耗输出之间的系统级连接。

允许适配：将原牵引速度/扭矩工况改为驻车堵转或近零速脉冲电流工况；将示例控制器接口改接 MCB FOC；增加电池端/DC-link/电机相/电机热/逆变器限制日志；用 `PMSMDriveThermalDQ.slx` 只做 DQ frame/步长/高频数值稳定性对照。

不得修改：不把 `PMSMDrive.slx` 非热版作为主电机基线；不把 v03 工程默认 PMSM 参数作为 V4 默认电机参数；不跳过热模型和损耗日志。

## 4. V4 模型结构

V4 工作副本：`车载脉冲加热技术simlink模型/02_模型/pulse_heating_official_spine_v04.slx`

V4 顶层结构固定为：

```text
PulseHeating_Command_And_Limits -> MCB_SIUnits_FOC_Controller -> PMSMDriveThermal_Inverter_And_Motor
BatteryThermalManagement_BatteryPack -> DC_Link_And_Inverter_Interface -> PMSMDriveThermal_Inverter_And_Motor
PMSMDriveThermal_Inverter_And_Motor -> KPI_And_Logging
```

- 电池包和热边界来自 `BatteryThermalManagementModel.slx`
- 逆变器、电机、编码器、热模型和损耗日志来自 `PMSMDriveThermal.slx`
- FOC 电流环、SVPWM 和启动结构来自 `mcb_pmsm_foc_qep_f28379d_SIUnit.slx`
- 脉冲加热命令、限制接口和 KPI 是本项目新增薄层

## 6. V4 建模总策略

V4 采用三层结构：

```text
官方核心子系统层
  BatteryThermalManagementModel / mcb_pmsm_foc_qep_f28379d_SIUnit / PMSMDriveThermal

项目适配层
  端口适配、总线适配、单位适配、低温堵转脉冲命令、DC-link 对接、缺失限制标志

审计输出层
  KPI、日志、最小回归、UNKNOWN/NaN 限制输出、验证记录
```

每替换一个占位块，都必须满足：官方来源可追溯、端口读回可验证、`model_check(all)` 无 error、最小仿真可复现、缺失数据不默认为通过。

## 9. 后续实施步骤

### 阶段 1：MCB FOC 控制器迁移（下一步）

目标：用官方 MCB FOC 替换 V4 控制器占位，不回退到 v03 手写控制链。

操作：
1. 读取 `mcb_pmsm_foc_qep_f28379d_SIUnit.slx` 的 `Current Control`、`Control_System`、`Closed Loop Control`、`Current_Controllers`、`SVPWM`、`Input Scaling`、`Open Loop Start-Up`。
2. 隔离硬件部署相关块：C2000 I/O、SCI、LED、hardware init 只保留为非主线参考。
3. 明确 `Idq_ref` 的真实数据结构（2x1，A）和 `Feedbacks_sim` 的真实数据结构。
4. 建立 `PulseHeating_Command_And_Limits -> Idq_ref` 适配。
5. 建立 `PMSMDriveThermal_Inverter_And_Motor -> Feedbacks_sim` 适配。
6. 建立 `Duty_Cycles`（MCB 3x1 duty_abc）-> 6 gate 适配器，显式说明调制语义和死区。
7. 输出 `ControlKPI`：`tracking_error_rms_A`、`modulation_index`、饱和标志、闭环使能状态。
8. 解决 MCB 参数（Teknic-2310P, 24V）与 V4 电驱参数（N=6, 48V）的不匹配。

验收：
- `0 A` 命令下 duty 输出稳定在安全默认状态。
- 小电流低频工况 `Id/Iq` 方向正确。
- `SVPWM` 链路保留，控制器不是 v03 简化 PI 的复制。
- 硬件部署相关块不进入主仿真路径。

停止条件：采样时间、单位缩放或反馈总线无法闭合。

### 阶段 2：电池热管理模型接入

目标：把 `BatteryThermalManagement_BatteryPack` 从 `NaN` 占位替换为官方电池包、模块热网络、冷却控制、SOC/温度限制接口。

前置门槛：必须先解释 28 个 module 端口 warning。

操作：
1. 读取 `Battery pack`、`Module 1-4`、`Coolant control`、`SOC limits`、`Drive profile`。
2. 将 `Drive profile` 替换为 V4 `BatteryLoad` 输入。
3. 暴露 `BatteryDC`（电压、电流、SOC、包温）和 `BatteryStatus`（温度/SOC 限制缺口标志）。
4. 低温初始条件只使用官方变量或 `ENGINEERING_DEFAULT`/`PLACEHOLDER` 标签。

验收：零负载下 SOC/温度/电压无非物理漂移；缺失 BMS/析锂/低温回充限制输出 `NaN`。

停止条件：module 端口 warning 无法解释，或 `BatteryLoad` 与电池正负端方向无法定义。

### 阶段 3：DC-link 与能量路径闭合

目标：让电池端、DC-link、逆变器、电机三相之间的电流和功率方向可审计。

操作：
1. 连接电池正负端到逆变器 DC `+/-`（替换临时 Battery_DC_Source）。
2. 明确 DC-link 电容来源。
3. 加入电池端电流、DC-link 电流、DC-link 电压测量，让 `I_dc`/`P_dc` 脱离 NaN。
4. 建立 `InverterDCFeedback -> BatteryLoad` 反馈。
5. 对回充半周方向单独标注。

验收：小电流工况下方向一致且可解释；`P_battery_terminal + P_inverter_loss + P_motor_loss` 符号和量级可审计。

### 阶段 4：堵转脉冲工况层

目标：在官方骨架上建立低温驻车堵转脉冲工况。

操作：
1. 在 `PulseHeating_Command_And_Limits` 中生成 `Id_ref/Iq_ref`、幅值、频率、占空比和相位补偿。
2. 机械端设置堵转或近零速边界。
3. 将 `diagnostic_frequency_Hz` 作为诊断变量，不写成产品默认频率。
4. 设置零命令、低频小电流、诊断频率小电流、诊断频率中等电流四类基础工况。

### 阶段 5：限制接口（V4-C）

必须输出：`voltage_margin`、`modulation_index`、`motor_phase_peak_margin`、`motor_phase_rms_margin`、`battery_charge_limit_margin`、`battery_temperature_margin`、`coolant_flow_margin`、`unknown_safety_margin`、`limiting_factor`。

缺失数据输出 `NaN` 或 `UNKNOWN`，不允许默认为通过。

### 阶段 6：KPI 与最小回归（V4-D）

KPI 字段至少覆盖：

| 类别 | 字段 |
|---|---|
| 电流 | `I_battery_terminal_rms_A`、`I_dc_link_rms_A`、`I_motor_phase_rms_A`、`I_heat_equiv_rms_A` |
| 电压/调制 | `V_battery_terminal_V`、`V_dc_link_V`、`modulation_index`、`voltage_margin` |
| 损耗/温度 | `P_battery_heat_W`、`P_motor_copper_W`、`P_motor_iron_W`、`P_inverter_loss_W`、`T_battery_C`、`T_motor_stator_C`、`T_motor_rotor_C` |
| 控制 | `tracking_error_rms_A`、饱和标志、闭环使能状态 |
| 保护 | `current_limit_margin`、`thermal_margin`、`battery_charge_limit_margin`、`unknown_safety_margin`、`limiting_factor` |

最小回归矩阵：
1. 零命令：所有主电流接近 0，温度/SOC 无非物理漂移。
2. 低频小电流：方向正确，控制链闭环可跟踪。
3. 诊断频率小电流：记录跟踪误差，不要求通过产品指标。
4. 诊断频率中等电流：检查调制、电压、电机相电流和热模型响应。
5. 缺失数据边界工况：限制接口输出 `UNKNOWN`/`NaN`，不输出 `PASS`。

每轮必须执行：`model_overview(interfaces)`、`model_read(depth=1)`、`model_check(all)`、最小仿真、文档同步。

## 11. 完成定义

只有同时满足以下条件，才能称为 V4 基线完成：

1. 三大官方示例基线均有来源路径、模型名和子系统边界记录。
2. V4 工作副本完成 `model_overview`、`model_read(depth=1)` 和 `model_check(all)`。
3. 控制器不再使用 v03 手工最小 FOC 作为主线。
4. 电池不再使用单 BEC 缩放模型作为主线默认电池包。
5. 电机/电驱不再使用工程默认 PMSM 参数作为主线默认参数。
6. KPI 区分电池端、DC-link、电机相和热等效电流。
7. 所有缺失的安全、BMS、MCU、逆变器和电机边界均以 `UNKNOWN`/`NaN` 输出，不被解释为通过。

## 13. 停止条件

出现以下情况时停止建模并先记录问题：

1. 官方示例初始化脚本或 Simscape 组件无法解析。
2. MCB `Duty_Cycles` 无法明确映射到 PMSMDriveThermal 逆变器门极或调制接口。
3. 电池端和 DC-link 能量方向无法闭合。
4. 为了跑通模型需要引入历史产品参数作为默认值。
5. `model_check(all)` 出现 error 级结构问题且无法在当前子阶段闭环修复。
