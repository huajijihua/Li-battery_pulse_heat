# V4 建模主计划 v01

日期：2026-07-10

本文件合并了原 `V4_官方案例复用建模规划_v01.md`、`V4_建模前点检审计与实施步骤计划_v01.md` 和 `V4_端口级迁移详细建模计划_v01.md`，是 V4 建模的唯一主计划入口。

## 1. V4 定位

V4 是本项目从 v03 诊断沙箱转向 MathWorks 官方案例骨架复用的主线规划。V4 不再从空白模型或手工拼接库块开始，而是固定复用三个已经在本机打开、经 MCP 只读确认的官方示例：

| 设备 | V4 固定复用基线 | 本地模型路径 | 用途 |
|---|---|---|---|
| 电池 | Battery Thermal Management | `...\BatteryThermalManagementExample\BatteryThermalManagementModel.slx` | 电池包、模块热网络、冷却控制和 SOC/温度限制接口 |
| 控制器 | Motor Control Blockset SI Units PMSM FOC | `...\SIUnitsExample\mcb_pmsm_foc_qep_f28379d_SIUnit.slx` | SI 单位 FOC、电流环、SVPWM、开环启动和仿真植物接口 |
| 电机/电驱 | PMSM Drive with Thermal Model | `...\PMSMDriveThermalExample\PMSMDriveThermal.slx` | PMSM、三相逆变器、编码器、热模型、损耗日志和电驱系统骨架 |

这三个基线在 V4 中不再作为候选项讨论，除非本机许可证或模型文件损坏导致不可用。其他官方示例只作为补充参考，不替代这三条主线。

## 3. 复用边界

### 3.1 电池基线

固定复用 `BatteryThermalManagementModel.slx` 的电池包和热管理骨架。

必须保留：`Battery pack`、`Module 1-4`、`Coolant control`、电芯/模块/包温度/SOC/热端口日志口径。

允许适配：将 `Drive profile` 替换为脉冲加热电流/功率工况输入；将默认温度/SOC/环境/冷却边界改为低温脉冲工况；增加缺口标志。

不得修改：不把 v03 单个 BEC 缩放模型作为 V4 默认电池基线；不把公司历史电池参数写入默认模型；不把 `BatteryDCFastCharging.slx` 作为主电池模型。

### 3.2 控制器基线

固定复用 `mcb_pmsm_foc_qep_f28379d_SIUnit.slx` 的 SI 单位 FOC 控制结构。

必须保留：`Current Control` 下的 `Control_System`、`Closed Loop Control`、`Current_Controllers`、`PI_Controller_Id/Iq`、`SVPWM`、`Open Loop Start-Up`、`Input Scaling`。

允许适配：将速度/扭矩外环替换或旁路为堵转脉冲电流命令；增加 `Id/Iq` 脉冲幅值/频率/占空比/相位补偿输入；隔离硬件 C2000 I/O/SCI/LED/通信；将控制输出对接到 `PMSMDriveThermal.slx` 的三相逆变器。

不得修改：不再手写 v03 最小 Clarke/Park/PI/SVPWM 作为主控制器；不删除官方 FOC 结构后只保留几个 Gain 和 Saturation；不把控制器跑通等同于 MCU 量产权限已开放。

### 3.3 电机/电驱基线

固定复用 `PMSMDriveThermal.slx` 的电机、电驱和热模型骨架。

必须保留：`Three-phase inverter`、`Thermal model`、`Encoder`、`Scopes/losses`、PMSM 与逆变器/热端口/损耗输出之间的系统级连接。

允许适配：将原牵引速度/扭矩工况改为驻车堵转或近零速脉冲电流工况；将示例控制器接口改接 MCB FOC；增加电池端/DC-link/电机相/电机热/逆变器限制日志；用 `PMSMDriveThermalDQ.slx` 只做 DQ frame/步长/高频数值稳定性对照。

不得修改：不把 `PMSMDrive.slx` 非热版作为主电机基线；不把 v03 工程默认 PMSM 参数作为 V4 默认电机参数；不跳过热模型和损耗日志。

## 4. V4 模型结构

V4 工作副本：`车载脉冲加热技术simlink模型/02_模型/pulse_heating_official_spine_v04.slx`

V4 顶层结构固定为：

```text
PulseHeating_Command_And_Limits -> MCB_SIUnits_FOC_Controller -> PMSMDriveThermal_Inverter_And_Motor
BatteryThermalManagement_BatteryPack -> DC_Link_And_Inverter_Interface -> PMSMDriveThermal_Inverter_And_Motor
PMSMDriveThermal_Inverter_And_Motor -> KPI_And_Logging
```

- 电池包和热边界来自 `BatteryThermalManagementModel.slx`
- 逆变器、电机、编码器、热模型和损耗日志来自 `PMSMDriveThermal.slx`
- FOC 电流环、SVPWM 和启动结构来自 `mcb_pmsm_foc_qep_f28379d_SIUnit.slx`
- 脉冲加热命令、限制接口和 KPI 是本项目新增薄层

## 6. V4 建模总策略

V4 采用三层结构：

```text
官方核心子系统层
  BatteryThermalManagementModel / mcb_pmsm_foc_qep_f28379d_SIUnit / PMSMDriveThermal

项目适配层
  端口适配、总线适配、单位适配、低温堵转脉冲命令、DC-link 对接、缺失限制标志

审计输出层
  KPI、日志、最小回归、UNKNOWN/NaN 限制输出、验证记录
```

每替换一个占位块，都必须满足：官方来源可追溯、端口读回可验证、`model_check(all)` 无 error、最小仿真可复现、缺失数据不默认为通过。

### 阶段 3：DC-link 与能量路径闭合（V4-D3 已完成，物理 DC-link 已闭合）

目标：让电池端、DC-link、逆变器、电机三相之间的电流和功率方向可审计。

操作：
1. 连接电池正负端到逆变器 DC `+/-`（替换临时 Battery_DC_Source）。
2. 明确 DC-link 电容来源。
3. 加入电池端电流、DC-link 电流、DC-link 电压测量，让 `I_dc`/`P_dc` 脱离 NaN。
4. 建立 `InverterDCFeedback -> BatteryLoad` 反馈。
5. 对回充半周方向单独标注。

验收：小电流工况下方向一致且可解释；`P_battery_terminal + P_inverter_loss + P_motor_loss` 符号和量级可审计。

阶段 3 完成范围（V4-D3，2026-07-14）：物理 DC-link 已闭合，临时 Battery_DC_Source 已删除，电池正负端经 PMIOPort 连接到电驱 DC+/-；DC_Current_Sensor 和 DC_Voltage_Sensor 已接入；Goto/From tag scope 已修复为 global；`I_dc_from_sensor`/`V_dc_for_invDC`/`P_dc_calc` 信号链路已建立；Simscape log 验证 V_pack=70.3V、I_dc peak=11.2A。剩余缺口：DC-link 电容尚未建模；`inverter.V_dc=66` 待用仿真实际 V_pack 校准；回充半周方向标注待 V4-G 补充。

### 阶段 4：堵转脉冲工况层（已完成）

目标：在官方骨架上建立低温驻车堵转脉冲工况。

操作：
1. 在 `PulseHeating_Command_And_Limits` 中生成 `Id_ref/Iq_ref`、幅值、频率、占空比和相位补偿。
2. 机械端设置堵转或近零速边界。
3. 将 `diagnostic_frequency_Hz` 作为诊断变量，不写成产品默认频率。
4. 设置零命令、低频小电流、诊断频率小电流、诊断频率中等电流四类基础工况。

阶段 4 当前完成范围：已完成堵转脉冲 `Idq_ref` 薄层和 `LimitConfig` 输出；四类基础工况的批量回归矩阵留到阶段 6 KPI 最小回归中执行。

### 阶段 5：限制接口（V4-C）（已完成）

必须输出：`voltage_margin`、`modulation_index`、`motor_phase_peak_margin`、`motor_phase_rms_margin`、`battery_charge_limit_margin`、`battery_temperature_margin`、`coolant_flow_margin`、`unknown_safety_margin`、`limiting_factor`。

缺失数据输出 `NaN` 或 `UNKNOWN`，不允许默认为通过。

阶段 5 当前完成范围：已完成 `LimitStatus` 10 维接口、字段顺序、UNKNOWN 状态码和顶层终止连接。真实电压/电流/温度/流量裕度待阶段 6 或 DC 传感器修复后接入。

### 阶段 6：KPI 与最小回归（V4-D，第二版已完成）

KPI 字段至少覆盖：

| 类别 | 字段 |
|---|---|
| 电流 | `I_battery_terminal_rms_A`、`I_dc_link_rms_A`、`I_motor_phase_rms_A`、`I_heat_equiv_rms_A` |

| 损耗/温度 | `P_battery_heat_W`、`P_motor_copper_W`、`P_motor_iron_W`、`P_inverter_loss_W`、`T_battery_C`、`T_motor_stator_C`、`T_motor_rotor_C` |
| 控制 | `tracking_error_rms_A`、饱和标志、闭环使能状态 |
| 保护 | `current_limit_margin`、`thermal_margin`、`battery_charge_limit_margin`、`unknown_safety_margin`、`limiting_factor` |

最小回归矩阵：
1. 零命令：所有主电流接近 0，温度/SOC 无非物理漂移。
2. 低频小电流：方向正确，控制链闭环可跟踪。
3. 诊断频率小电流：记录跟踪误差，不要求通过产品指标。
4. 诊断频率中等电流：检查调制、电压、电机相电流和热模型响应。
5. 缺失数据边界工况：限制接口输出 `UNKNOWN`/`NaN`，不输出 `PASS`。

阶段 6 第一版完成范围：已完成 `SystemKPI` 24 维接口、V4-D1 UNKNOWN 状态码、`run_v4d_minimal_regression.m` 最小回归脚本、`model_check(all)=healthy`、关闭重载后 `sim(StopTime=0.001)=ReachedStopTime`。

阶段 6 第二版完成范围：已完成相电流瞬时 RMS/峰值、duty 调制指数代理和 duty 饱和标志的部分数值 KPI；新增 `run_v4d2_case_matrix_regression.m` 并完成 4 个烟测 case。V4-D2.1 已修复逆变器 DC 母线拓扑，V4-D2.2 已修复 PI/PWM/设备元数据适配，控制/门极/电驱链可形成有效脉冲相电流；但物理 DC-link、电池端 V/I、DC 电流/功率和损耗计算仍缺失，后续应从 V4-D3 的物理 DC-link 闭合、DC 传感器和损耗审计继续。

每轮必须执行：`model_overview(interfaces)`、`model_read(depth=1)`、`model_check(all)`、最小仿真、文档同步。

## 9.2 后续建模计划（V4-E 至 V4-I）

V4-A 至 V4-D3 完成了"官方基线迁移 + 物理拓扑闭合 + 接口骨架"，但模型仍不是行为模型。后续按"先打通能量守恒闭环 -> 再补电化学限制 -> 最后行为验证"的顺序推进。

### V4-E：损耗与发热计算（电热核心，最高优先级）

目标：让 4 个损耗功率脱离 NaN，形成 `P_battery = P_inv_loss + P_motor_loss + P_mech + P_battery_heat` 能量守恒等式。

前置条件：开启 `SimscapeLogType=all`。

| 子步 | 内容 | 来源标签 |
|---|---|---|
| E1 | 电机铜损 `P_cu = 1.5*Rs*(Id^2+Iq^2)`，从 Feedbacks_sim 取 Idq、从 pmsm.Rs 取电阻；替换 blk_193(P_cu_UNKNOWN) | OFFICIAL 公式 |
| E2 | 电机铁损 `P_iron`，接入官方 losses 子系统(blk_171)输出或用 Steinmetz 简化；替换 blk_194(P_iron_UNKNOWN) | OFFICIAL 结构 |
| E3 | 逆变器损耗 `P_inv`，从 IGBT 导通+开关损耗估算（需 V_dc/I_dc/duty）；替换 blk_195(P_inv_UNKNOWN) | ENGINEERING_DEFAULT |
| E4 | 电池发热 `P_batt_heat = I_pack^2 * R_pack`，R_pack 从 Battery_pack 内阻聚合；替换 blk_570(KPI_P_batt_heat_UNKNOWN) | OFFICIAL 结构 |
| E5 | 能量守恒审计块：`P_battery_terminal = P_inv_loss + P_motor_loss + P_mech + P_battery_heat`；新增审计薄层 | 新增审计薄层 |

验收：4 个损耗字段有数值且量级可解释；能量守恒残差 < 工程容忍带；`model_check(all)` healthy；最小仿真通过。

停止条件：Simscape log 无法提取损耗分量，或损耗量级与物理预期差 2 个数量级以上。

### V4-F：电池电化学与限制边界

目标：让电池端 4 个 NaN 限制字段脱离 UNKNOWN，建立低温安全边界。

前置条件：V4-E 完成（需要 I_pack 和 P_batt_heat 有数值）。

| 子步 | 内容 | 来源标签 |
|---|---|---|
| F1 | 电池低温参数适配：ECM 内阻温度依赖、SOC-OCV 曲线（从官方 BatteryModule.ssc 参数或文献） | OFFICIAL/LITERATURE |
| F2 | `T_min` 从 Battery_pack 4 模块温度场取最小值；替换 blk_519(T_min_UNKNOWN) | OFFICIAL 结构 |
| F3 | `SOC_limit` 从 Coolant_control/SOC_limits 子系统接入；替换 blk_520(SOC_limit_UNKNOWN) | OFFICIAL 结构 |
| F4 | `Charge_limit` / `Discharge_limit` 从低温充电窗口计算（文献公式 + 工程默认）；替换 blk_521/blk_522 | LITERATURE + ENGINEERING_DEFAULT |
| F5 | 析锂风险标志（可选，标记为 PLACEHOLDER，需文献支撑才能升级） | PLACEHOLDER |

验收：限制接口不再由 UNKNOWN 主导；`limiting_factor_code` 从 99 降级；`limit_status_code` 更新。

停止条件：低温充电窗口无文献支撑，或 BatteryModule.ssc 参数无法适配低温边界。

### V4-G：DC-link 与能量路径完善

目标：让 DC_Link 子系统从"两个 Selector"升级为具备能量审计能力的物理桥接层。

前置条件：V4-E 完成（需要损耗数值做能量守恒审计）。

| 子步 | 内容 | 来源标签 |
|---|---|---|
| G1 | DC-link 电容建模（Simscape Capacitor，参数 ENGINEERING_DEFAULT） | ENGINEERING_DEFAULT |
| G2 | 能量方向审计：充放电半周分离，标记回充方向 | 新增审计薄层 |
| G3 | `inverter.V_dc=66` 用仿真实际 V_pack 校准 | 校准 |
| G4 | I_dc / P_dc / V_pack / I_pack 全部 KPI 字段从传感器填充（替换剩余 NaN 常量） | 传感器实测 |

验收：DC_Link 子系统不再是薄 Selector；4 个 DC 量有数值且方向一致；能量守恒残差可审计。

停止条件：DC-link 电容参数无工程依据，或能量方向符号无法统一。

### V4-H：控制策略完善

目标：让控制器具备限流闭环和真实裕度输出。

前置条件：V4-F 完成（需要电池限制边界做电流限幅）。

| 子步 | 内容 | 来源标签 |
|---|---|---|
| H1 | `tracking_error_rms` 从 Idq_ref vs Idq_fb 计算；替换 blk_337(KPI_tracking_NaN) | OFFICIAL 公式 |
| H2 | 电压裕度 `voltage_margin = V_dc - V_modulation_required`；替换 Limit_voltage_UNKNOWN | OFFICIAL 公式 |
| H3 | 电流限幅：基于电池 Charge_limit 和电机相电流限值做饱和 | ENGINEERING_DEFAULT |
| H4 | 脉冲加热指令与限制联动：LimitConfig 反馈到命令层，超限时降额或停脉冲 | 新增控制逻辑 |

验收：控制器不再是开环命令；限制裕度有数值；超限工况有保护动作。

停止条件：限流逻辑与 FOC 电流环冲突，或保护动作导致仿真发散。

### V4-I：KPI 补全与行为验证

目标：SystemKPI 全部脱离 NaN，完成长仿真行为验证。

前置条件：V4-E/V4-F/V4-G/V4-H 均完成。

| 子步 | 内容 |
|---|---|
| I1 | 剩余 UNKNOWN 字段填充（热等效电流 I_heat_equiv、电流限制裕度、热裕度） |
| I2 | 限制接口全部字段从真实裕度计算（替换 limit_margin_unknown_value） |
| I3 | 长仿真行为验证：10-30s 温升曲线、SOC 消耗、能量守恒残差 |
| I4 | 工况矩阵行为回归（零命令/低频/诊断频率/中等电流 4+工况） |
| I5 | 能量守恒闭环验证：`P_battery = P_inv + P_motor + P_mech + P_heat` 残差审计 |

验收：SystemKPI 24 维全有数值；能量守恒残差 < 容忍带；温升曲线物理合理；`kpi_status_code` 和 `limit_status_code` 脱离 UNKNOWN 主导。

停止条件：长仿真发散，或温升曲线与物理预期矛盾且无法定位原因。

### V4-E 至 V4-I 执行顺序

```text
V4-E (损耗与发热) -> V4-G (DC-link 完善) -> V4-F (电池电化学限制) -> V4-H (控制策略) -> V4-I (KPI 补全与行为验证)
```

V4-E 是解锁后续所有阶段的前提：没有损耗数值就无法做能量守恒、无法验证温升、无法判断限制裕度。

## 11. 完成定义

只有同时满足以下条件，才能称为 V4 基线完成：

1. 三大官方示例基线均有来源路径、模型名和子系统边界记录。
2. V4 工作副本完成 `model_overview`、`model_read(depth=1)` 和 `model_check(all)`。
3. 控制器不再使用 v03 手工最小 FOC 作为主线。
4. 电池不再使用单 BEC 缩放模型作为主线默认电池包。
5. 电机/电驱不再使用工程默认 PMSM 参数作为主线默认参数。
6. KPI 区分电池端、DC-link、电机相和热等效电流。
7. 所有缺失的安全、BMS、MCU、逆变器和电机边界均以 `UNKNOWN`/`NaN` 输出，不被解释为通过。

## 13. 停止条件

出现以下情况时停止建模并先记录问题：

1. 官方示例初始化脚本或 Simscape 组件无法解析。
2. MCB `Duty_Cycles` 无法明确映射到 PMSMDriveThermal 逆变器门极或调制接口。
3. 电池端和 DC-link 能量方向无法闭合。
4. 为了跑通模型需要引入历史产品参数作为默认值。
5. `model_check(all)` 出现 error 级结构问题且无法在当前子阶段闭环修复。
