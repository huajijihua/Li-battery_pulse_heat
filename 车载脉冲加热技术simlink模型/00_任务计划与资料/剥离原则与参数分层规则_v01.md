# 剥离原则与参数分层规则 v01

日期：2026-07-15
范围：M3 工程兼容性模型的参数、配置、脚本、模型和文档

## 1. 核心目标

M3 是通用低温脉冲加热工程兼容性模型，不是具体产品、特定车型或历史案例的复刻。参数治理的目标是让每个参数具有可追溯来源、明确物理身份、适用范围和受控替换关系。

参数剥离不等于剥离官方示例、公开文献或历史证据。应隔离的是把历史产品、L0.5 粗筛或 M2 未闭合参数写成 M3 默认配置和工程结论的行为。

## 2. 参数分层规则

所有进入 M3 配置的参数必须带来源标签。没有标签、设备身份或适用范围的参数不得作为默认值使用。

| 层级 | 标签 | 含义 | M3 使用规则 |
|---|---|---|---|
| 平台默认 | `PLATFORM_DEFAULT` | 为 M3 通用架构建立的自洽参数组合 | 可作为默认配置，须记录组合逻辑和适用范围 |
| 官方 | `OFFICIAL` | 官方模型、库、数据手册或标准默认值 | 可作为候选来源，须记录版本和迁移差异 |
| 文献 | `LITERATURE` | 公开论文、专利或标准中的量级与机理依据 | 可作为候选来源，须记录对象和适用条件 |
| 工程经验 | `ENGINEERING_DEFAULT` | 非产品化的工程量级或简化假设 | 仅在缺少更高可信来源时使用，须记录理由和替换条件 |
| 占位 | `PLACEHOLDER` | 明确的数据缺口或未建模机理 | 只能表达未知或待确认，不得用于通过判定 |
| 历史案例 | `HISTORICAL_CASE` | 历史车型、公司资料或冻结模型中的数据 | 不得进入默认配置，仅作为背景、风险或对照 |
| 粗筛 | `L0.5_SCREENING` | L0.5 计算或敏感性扫描假设 | 不得进入默认配置，仅作为量级和问题来源参考 |

## 3. 最小参数元数据

每一条 M3 参数至少记录：

1. 参数名、物理含义、单位和数值或占位状态。
2. 来源标签、来源文件/页码或模型版本。
3. 设备身份或“平台默认”身份。
4. 温度、SOC、电压、频率、时间窗口和冷却条件等适用范围。
5. 影响的系统模块、接口和下游重算项。
6. 可信等级、拟合状态、验证状态和待确认项。

峰值、平均值、窗口 RMS、母线电流、电池支路电流、电机相电流和热等效量必须独立建模和登记，禁止用同一个参数名或数值互相替代。

## 4. 历史与冻结资产的隔离

以下内容不得直接进入 M3 默认参数、默认架构或现行结论：

1. 历史车型、电池包、电机、逆变器和控制器的具体参数。
2. L0.5 中为粗筛建立的拓扑、频率、幅值、持续时间或安全假设。
3. M2/V4-I 中未完成身份确认、拓扑推导、RMS 定义或能量闭环的参数。
4. 没有来源、对象、单位或适用范围的常数。

这些内容只允许在历史说明、风险登记、对照工况或明确标记的可选案例中出现。M3 引用它们时，必须标明 `HISTORICAL_CASE` 或 `L0.5_SCREENING`，并说明为何不能作为默认值。

## 5. 配置与替换规则

1. M3 的电池、DC-link、逆变器、电机和控制器配置应具有独立身份，并通过兼容性矩阵形成受控组合。
2. 替换任一设备配置时，必须重新检查相关电压、电流、功率、热、频率、采样、保护和持续时间约束。
3. 拓扑、电阻、热容量、损耗和限制值必须从同一配置来源推导；不得用不同设备或不同拓扑的值拼接为默认模型。
4. `PLACEHOLDER` 可以保留接口和数据缺口，但不得被限流、保护、热或安全逻辑默认为通过。
5. 具体产品数据进入模型前，必须先登记来源、设备身份、版本、适用范围和替换影响；产品数据不自动覆盖平台默认配置。

## 6. 污染检查门槛

每次创建或修改 M3 配置、接口、脚本或模型前，检查是否存在：

| 检查项 | M3 要求 |
|---|---|
| 历史产品参数 | 只允许作为有标签的历史参考，不能成为默认值 |
| 固定车型或旧拓扑 | 只允许在可选案例中出现，不能定义 M3 默认架构 |
| 高频或脉冲频率 | 必须标明工况角色、适用范围和控制时序，不得仅因历史使用而成为默认 |
| 官方示例参数 | 必须区分“官方来源”与“已适配 M3”的状态 |
| RMS 与热量 | 必须具有时间窗口、采样和积分口径，不得使用瞬时代理 |
| 安全或保护阈值 | 必须有设备来源或明确的工程占位标记，不能由缺失数据推断通过 |

## 7. 进入模型前的验证要求

M3 参数配置在进入 `.slx` 前至少应满足：

1. 默认参数都有来源标签、单位和设备/平台身份。
2. 关键接口参数已定义适用范围和替换影响。
3. 电池拓扑、限流、电阻、热源和电流口径没有未声明冲突。
4. 任何未知机理或数据缺口均以 `PLACEHOLDER` 或风险项表达。
5. 未将历史或冻结模型的参数、判据和结论写成 M3 已验证事实。

实现后的结构、仿真和工程验收要求由 `00_项目管理/M3_工程兼容性模型规划_v01.md` 的 G0-G3 门槛和后续验证计划定义。
