## 1. 架构决策更新

> 2026-07-07 GPT审计更新：本文件原先把 Controlled Voltage Source 调整描述为实际方案收敛。该表述过强。Controlled Voltage Source/Controlled Current Source 只能作为临时受控源测试夹具，不是规范逆变器。规范主线仍应收敛到 Average-Value Voltage Source Converter、两电平变换器或开关级桥，并在文档中明确当前采用哪一种口径。

### 1.1 逆变器方案调整

原计划用 `Average-Value Inverter (Three-Phase)`，但验证发现该块**无信号控制端口**（LConn:2 RConn:1 全为电气端口），无法接受 FOC 调制波信号输入。

临时调整为：用 `Controlled Voltage Source (Three-Phase)` 作为受控电压源，表达"FOC → Vabc 电压指令 → 受控源 → PMSM"链条。

理由：
- Controlled VS 支持 `input_option=instantaneous`，接受三相瞬时电压 PS 信号输入
- FOC 电流环输出 Vd/Vq → 逆 Park → Valpha/Vbeta → SVPWM/PWM → Vabc → Simulink-PS Converter → Controlled VS
- 物理上表达了"控制器输出电压驱动电机绕组"的语义，符合《逆变器调幅口径》第3.3节
- 标注为"受控电压源近似控制器输出"，不等于实际逆变器开关级实现

### 1.2 控制链信号流（目标结构，当前模型尚未实现）

```text
脉冲命令发生器 → Id_ref, Iq_ref
    ↓
Current Sensor (Three-Phase) → Iabc → PS-Simulink Converter → Clarke → Park → Id_fb, Iq_fb
    ↓
PI 电流环 (Discrete PI) → Vd_ref, Vq_ref
    ↓
逆 Park (dq→alpha/beta) → Valpha, Vbeta
    ↓
PWM/SVPWM → Vabc (三相调制波)
    ↓
Simulink-PS Converter → Controlled Voltage Source (Three-Phase) → PMSM
```

### 1.3 堵转 theta 处理

堵转时转子不动（omega=0），theta 保持初始值。第一版设 theta=0，d 轴与 a 相重合。逆 Park 变换中 theta=0 时 Valpha=Vd, Vbeta=Vq。

端口语义（待 model_read 实际确认）：

## 3. 信号流与物理网络接口

### 3.1 物理网络（Simscape 域）

```text
电池 BEC (+) ─── DC-link 电容 ─── Controlled VS (+)
                                        │
                                   三相 Vabc
                                        │
                                   Current Sensor
                                        │
                                    PMSM (a,b,c)
                                        │
电池 BEC (-) ─────────────────── Controlled VS (-)
```

### 3.2 信号流（Simulink 域）

```text
脉冲命令 → Id_ref ──┐
                     ├──> PI(d) ──> Vd ──┐
Current Sensor → Iabc → Clarke → Park → Id_fb ──┘        ├──> 逆Park → Valpha,Vbeta → PWM → Vabc
                     │                                   │
                     ├──> PI(q) ──> Vq ─────────────────┘
脉冲命令 → Iq_ref ──┘
                     │
                     └──> Iq_fb
theta = 0 (堵转固定角)
```

### 3.3 桥接接口

| 接口点 | 物理域 → 信号域 | 转换块 |
|---|---|---|
| 电流反馈 | Simscape electrical → Simulink signal | PS-Simulink Converter |
| 电压指令 | Simulink signal → Simscape physical signal | Simulink-PS Converter |

### 3.4 采样时间

- 物理网络：连续时间（Simscape 求解器）
- 信号流控制：离散采样 Ts=1e-4s（100us）
- 桥接处用 Rate Transition 或统一采样时间避免冲突

## 4. 已确认与待确认项

以下信息已通过 v03-B2 搭建和诊断确认：

1. `Battery_Pack` 使用 `batt_lib/Cells/Battery Equivalent Circuit`，按 `216S1P` 等效包缩放，DC-link 约 801 V。
2. `Average-Value Voltage Source Converter (Three-Phase)` 在理想 DC 供电隔离测试和主模型供电修复后均可驱动 PMSM 形成相电流。
3. `Current Sensor (Three-Phase)` 输出可经 `PS-Simulink Converter` 接入 `Iabc_to_IdIq_theta0`。
4. 堵转机械端口当前接 Mechanical Rotational Reference，作为 theta=0 最小验证口径。

以下信息仍需在 v03-C 或后续版本中确认：

1. PI 增益、采样周期和调制限幅的可解释整定范围。
2. 电压饱和、最小脉宽、死区和 MCU/逆变器限流接口。
3. KPI v02 的电池端、母线端、电机相和热等效电流字段。

## 5. 与计划文档的差异记录

| 项目 | 计划原方案 | 实际验证后调整 | 原因 |
|---|---|---|---|
| 逆变器 | Average-Value Inverter (Three-Phase) | Controlled Voltage Source (Three-Phase) | Average-Value Inverter 无信号控制端口，无法接受 FOC 调制波 |
| 控制接口 | PWM gating → 逆变器 | Vabc → Simulink-PS → Controlled VS | Controlled VS 接受瞬时电压 PS 信号，更直接表达 FOC 电压指令 |
| DC-link | ee_lib/Passive 电容 | ee_lib/Passive/Capacitor（已确认） | 确认具体块路径 |
| 调制策略 | 先 PWM 后 SVPWM | 不变 | Vabc 生成方式第一版用简化 PWM，第二版升级 SVPWM |
