## 1. 架构决策更新

> 2026-07-08 官方案例复用纠偏：本文件当前描述的是 v03 诊断模型接口契约。v03 只做到官方库块复用和手工信号链集成，不等同于复用 MathWorks 成熟 PMSM drive/FOC/热管理案例。后续 V4 工作副本必须以 `V4_官方案例复用建模规划_v01.md` 为准，三大基线固定为电池 `BatteryThermalManagementModel.slx`、控制器 `mcb_pmsm_foc_qep_f28379d_SIUnit.slx`、电机/电驱 `PMSMDriveThermal.slx`；V4 接口必须通过官方示例 `model_overview/model_read` 的实际接口重新建立。

> 2026-07-07 GPT审计更新：本文件原先把 Controlled Voltage Source 调整描述为实际方案收敛。该表述过强。Controlled Voltage Source/Controlled Current Source 只能作为临时受控源测试夹具，不是规范逆变器。规范主线仍应收敛到 Average-Value Voltage Source Converter、两电平变换器或开关级桥，并在文档中明确当前采用哪一种口径。

### 1.1 逆变器方案调整

原计划用 `Average-Value Inverter (Three-Phase)`，但验证发现该块**无信号控制端口**（LConn:2 RConn:1 全为电气端口），无法接受 FOC 调制波信号输入。

当前 v03-B3 实际模型采用 `Average-Value Voltage Source Converter (Three-Phase)`，块名为 `VSC_Avg_Test`，输入口径为 `Modulation waveforms`。它表达"FOC → dq 调制命令 → 三相调制波 → 平均值 VSC → PMSM"链条。

理由：
- `Average-Value Voltage Source Converter (Three-Phase)` 支持 `ModWave` 物理信号输入，能够接受三相调制波
- FOC 电流环输出 `md/mq` → 逆 Park/Clarke 近似 → `mabc` → Simulink-PS Converter → Average-Value VSC
- 物理上表达了"调制命令驱动平均值变换器"的语义，比受控电压源更接近 v04 逆变器规范化方向
- 标注为"平均值测试执行器"，不等于开关级逆变器、死区模型或量产控制器实现

### 1.2 控制链信号流（v03-B3 当前结构）

```text
脉冲命令发生器 → Id_ref, Iq_ref
    ↓
Current Sensor (Three-Phase) → Iabc → PS-Simulink Converter → Clarke → Park → Id_fb, Iq_fb
    ↓
PI 电流环 (Discrete PI) → md, mq
    ↓
dq 到三相调制矩阵（theta=0）
    ↓
    PWM/SVPWM → mabc (三相调制波)
    ↓
    Simulink-PS Converter → Average-Value Voltage Source Converter (Three-Phase) → PMSM
```

### 1.3 堵转 theta 处理

堵转时转子不动（omega=0），theta 保持初始值。第一版设 theta=0，d 轴与 a 相重合。逆 Park 变换中 theta=0 时 Valpha=Vd, Vbeta=Vq。

### 2.2 平均值 VSC 测试执行器：Average-Value Voltage Source Converter (Three-Phase)

| 属性 | 值 |
|---|---|
| 库路径 | `ee_lib/Semiconductors & Converters/Converters/Average-Value Voltage Source Converter (Three-Phase)` |
| BlockType | SimscapeBlock |
| 端口 | LConn:2, RConn:2（ModWave + 三相端口 + DC 端口） |
| 物理域 | electrical + physical signal |

关键参数：

| 参数名 | 当前值 | 来源 | 说明 |
|---|---|---|---|
| input_option | Modulation waveforms | `OFFICIAL` | 接受三相调制波 PS 信号 |

端口语义（v03-B3 model_read 已确认）：
- `ModWave`：接收 `mabc` 三相调制波（通过 Simulink-PS Converter）
- `p`：连接 DC+ 母线
- `n`：连接 DC- 母线
- `N`：连接三相电流传感器和 PMSM 三相 composite 端口

对接关系：
- ModWave 输入 ← Simulink-PS Converter ← FOC/PWM 信号流 `mabc`
- 电气+ ← 电池 BEC 正极
- 电气- ← 电池 BEC 负极
- 三相输出 → PMSM a/b/c

标注：此块为"平均值测试执行器"，表达调制波驱动平均值 VSC 的物理语义，不等于实际逆变器开关级实现。后期升级开关级时替换为两电平桥、IGBT/MOSFET 逆变器或专用平均值逆变器模型。

## 3. 信号流与物理网络接口

### 3.1 物理网络（Simscape 域）

```text
电池 BEC (+) ─── DC-link 电容 ─── Average-Value VSC p
                                        │
                                  三相输出 N
                                        │
                                   Current Sensor
                                        │
    PMSM (a,b,c)
                                        │
电池 BEC (-) ─────────────────── Average-Value VSC n
```

### 3.2 信号流（Simulink 域）

```text
脉冲命令 → Id_ref ──┐
                      ├──> PI(d) ──> md ──┐
Current Sensor → Iabc → Clarke → Park → Id_fb ──┘        ├──> dq 到三相调制 → mabc
                      │                                   │
                      ├──> PI(q) ──> mq ─────────────────┘
脉冲命令 → Iq_ref ──┘
                      │
                      └──> Iq_fb
theta = 0 (堵转固定角)
```

### 3.3 桥接接口

| 接口点 | 物理域 → 信号域 | 转换块 |
|---|---|---|
| 电流反馈 | Simscape electrical → Simulink signal | PS-Simulink Converter |
| 调制指令 | Simulink signal → Simscape physical signal | Simulink-PS Converter |

### 3.4 采样时间

- 物理网络：连续时间（Simscape 求解器）
- 信号流控制：离散采样 Ts=1e-4s（100us）
- 桥接处用 Rate Transition 或统一采样时间避免冲突

## 5. 与计划文档的差异记录

| 项目 | 计划原方案 | 实际验证后调整 | 原因 |
|---|---|---|---|
| 逆变器 | Average-Value Inverter (Three-Phase) | Average-Value Voltage Source Converter (Three-Phase) | Average-Value Inverter 无信号控制端口；当前 VSC 可接受 `ModWave` 调制波 |

| 调制策略 | 先 PWM 后 SVPWM | v03-B3 先用 theta=0 dq 到三相调制矩阵 | 当前生成 `mabc`，后续再升级 SVPWM |
