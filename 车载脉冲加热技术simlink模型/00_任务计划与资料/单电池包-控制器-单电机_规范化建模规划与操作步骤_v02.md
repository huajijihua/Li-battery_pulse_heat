# 单电池包-控制器-单电机规范化建模规划与操作步骤 v02

日期：2026-07-07

定位：后续实际建模的权威执行计划。旧 `单电池包-电机控制器-单电机_Simscape第一版模型实施计划_v01.md` 保留为前置盘点记录，本文件作为后续建模操作入口。

## 1. 建模目标

本阶段目标不是用少量实测数据拟合出“看起来正确”的曲线，而是建立一个可复用、可扩展、可审计的单电池包-控制器-单电机低温脉冲加热模型骨架。

第一版模型必须回答：

1. 电池包、电驱、控制器、热边界之间的物理端口是否规范连通。
2. 目标脉冲电流是否经过控制链、调制链、电压/电流限制后形成实际电机相电流和电池端电流。
3. KPI 是否区分电池端电流、母线电流、电机相电流和热等效电流。
4. 参数来源是否可追溯，缺口是否标注为 `PLACEHOLDER`。

第一版不证明：电池低温高频脉冲安全、析锂边界、逆变器结温寿命、量产控制器可开放该模式或 4500-27 整车工程可行。

## 3. 模块划分

第一版模型按 6 个模块搭建，每个模块必须有明确参考来源。

| 模块 | 模型职责 | 首选复用资产 | 禁止方式 |
|---|---|---|---|
| 电池包 | 通用等效串并联电池包、电热耦合、电压/SOC/温度输出 | 当前采用官方默认 `Battery Equivalent Circuit` 数据按 `N_series/N_parallel` 缩放 | 将 252S1P 写成硬约束或继续使用零电压生成资产 |
| DC-link/母线 | 电池包与逆变器 DC 侧连接、母线电容、母线电压测量 | `ee_lib/Passive/Capacitor`、电压/电流传感器 | 隐式连接或只用信号线表示母线 |
| 逆变器/执行器 | 把控制器电压或调制命令施加到 PMSM | 第一闭环版允许 Controlled Voltage Source 测试夹具；规范版升级 Average-Value VSC 或两电平变换器 | 把受控源称为规范逆变器 |
| PMSM | 堵转三相电机、电感/电阻/反电势/热参数 | `ee_lib/Electromechanical/Permanent Magnet/PMSM` | MATLAB Function 自写电机方程作主物理块 |
| 控制器 | 脉冲命令、Clarke/Park、PI、限幅、逆 Park、调制 | Simulink 标准信号流块，参考官方 FOC/SVPWM 案例 | 用固定 Sine Wave 直接驱动物理网络 |
| KPI/日志 | 分离记录电池、电机、母线、控制器限制和热量 | To Workspace / logsout / 后处理脚本 | 单一电流变量复用所有物理含义 |

## 6. v03-A 操作步骤

v03-A 只解决物理网络规范连通，不输出工程可行结论。

1. 新建模型 `pulse_heating_single_pack_v03.slx`，设置求解器 `ode23t`，短时 StopTime 先用 `1 s`，MaxStep 先不小于 `1e-4 s`。
2. 放入 `Battery_Pack`，当前优先使用官方 `Battery Equivalent Circuit` 等效包；默认 `216S1P`，连接 `+`、`-`、`H`，并显式初始化 DC-link 电压。
3. 放入 DC-link 电容、电气参考、Solver Configuration，电容并联在电池包正负端。
4. 放入 PMSM，使用三相独立端口或 composite 端口时必须在接口契约中记录端口形式。
5. 放入临时执行器：优先 `Controlled Voltage Source (Three-Phase)` 测试夹具，文件内命名为 `Inverter_TestSource`，不得命名为规范逆变器。
6. 连接执行器 DC/参考端、三相输出与 PMSM，加入三相电流传感器。
7. 加入电池端电流、电池端电压、母线电压、电机相电流、PMSM 速度/转矩、Battery_Pack 温度/SOC 输出日志。
8. 机械端口先采用固定堵转口径：PMSM 机械端接 Mechanical Rotational Reference 或等效零速约束，并在模型说明中记录。
9. 运行 `model_read(depth=1)` 检查块来源和连接，再运行 `model_check(all)`。
10. 只在 `model_check` 无关键未连接后运行 0.1-1 s 最小仿真，确认关键输出非空和电池包端电压量级正确。

v03-A 验收输出：模型拓扑截图可不导出；只记录 `model_check` 结果、关键块清单、关键输出变量名和是否非空。

## 7. v03-B 操作步骤

v03-B 增加最小闭环控制，不要求最终控制器精度。

1. 建立脉冲命令子系统 `Pulse_Command`：输入 `pulse_frequency_Hz`、`pulse_duty`、`Id_amp_A`，输出 `Id_ref` 和 `Iq_ref=0`。
2. 建立电流反馈子系统 `Current_Feedback`：电机三相电流转换为 `Ialpha/Ibeta`，再用固定 `theta=0` 做 Park 变换得到 `Id_fb/Iq_fb`。
3. 建立 PI 子系统 `Current_PI`：分别计算 `Vd_cmd/Vq_cmd`，含限幅和抗积分饱和占位。
4. 建立逆 Park 子系统 `DQ_to_ABC`：输出 `Va_cmd/Vb_cmd/Vc_cmd` 或调制波。
5. 控制器采样时间先统一为 `ctrl_Ts_s=1e-4`，必要时在物理/信号桥接处加 Rate Transition。
6. 运行零命令测试：`Id_ref=0`、`Iq_ref=0`，检查电流不出现非物理漂移。
7. 运行小电流测试：`Id_amp_A=10-20 A`，确认闭环方向正确。
8. 再运行目标初始工况：`Id_amp_A=100 A`、`1250 Hz`、`duty=0.5`，记录跟踪误差，不做可行性结论。

v03-B 验收输出：`I_target_A`、`I_motor_phase_rms_A`、`I_d_rms_A`、`tracking_error_rms_A`、`voltage_cmd_peak_V`、饱和标志。

## 8. v03-C 操作步骤

v03-C 让模型具备 L1 电流可实现性最小接口。

1. 加入电压限幅：`V_cmd_limit = m_max * V_dc / sqrt(3)` 或等效调制上限，`m_max` 标注为 `PLACEHOLDER`。
2. 加入电机 RMS/峰值限幅：继承 L0.5 的 320/450/550 Arms 分级，但第一版默认只记录，不自动证明可持续。
3. 加入电池低温限流占位：输出 `battery_pulse_current_margin`，无数据时置 `NaN` 并标注缺口。
4. 加入最小脉宽/死区占位：输出 `min_pulse_width_margin`、`deadtime_margin`，无供应商数据时置 `NaN`。
5. 增加限制来源优先级：电压饱和、PI饱和、电机峰值、电机RMS、电池限流、频率越界、数据缺口。
6. KPI 脚本升级为 `compute_kpi_v02.m`，字段必须区分电池、电机、母线和热等效电流。

v03-C 验收输出：`limiting_factor` 可解释，所有 `NaN` 都代表明确数据缺口，不代表通过。

## 9. KPI 字段规范

第一版 KPI 至少包含：

| 字段 | 物理含义 |
|---|---|
| `I_battery_terminal_rms_A` | 电池包电气端口 RMS 电流 |
| `I_dc_link_rms_A` | DC-link/母线侧 RMS 电流 |
| `I_motor_phase_rms_A` | 三相电机相电流 RMS，可分别输出 a/b/c |
| `I_heat_equiv_rms_A` | 用于电池发热的热等效 RMS 电流，必须说明来源 |
| `V_battery_terminal_V` | 电池包端电压 |
| `V_dc_link_V` | DC-link 电压 |
| `V_cmd_peak_V` | 控制器命令电压峰值 |
| `P_battery_heat_W` | 电池内阻/阻抗实部发热功率 |
| `P_motor_copper_W` | 电机铜耗 |
| `P_inverter_loss_W` | 逆变器损耗，占位时为 `NaN` 或明确 `PLACEHOLDER` |
| `T_battery_C` | 电池包温度 |
| `SOC_cell` | 电芯 SOC |
| `tracking_error_rms_A` | 目标电流与实际反馈电流误差 |
| `limiting_factor` | 当前最先触发或最接近触发的限制来源 |

禁止把 `I_actual_rms_A` 同时解释为电池电流、母线电流、电机相电流和热等效电流。

## 17. 完成定义

只有同时满足以下条件，才允许称为“第一版规范化模型已完成”：

1. `model_check(all)` 无关键未连接端口/线。
2. 当前主电池资产是可追溯、可缩放的官方库电池包模型；若用单个 BEC，必须明确其为串并联缩放的等效包。
3. 控制器不再是固定 Sine Wave，至少有 `Id_ref/Iq_ref -> PI -> 电压命令` 链条。
4. 逆变器/执行器口径在模型、接口契约和说明文件中一致。
5. KPI 脚本区分电池端、母线、电机相和热等效电流。
6. 运行过短时最小仿真并完成 read-back verification。
7. 文档明确列出仍未验证的电池安全、控制器权限、逆变器热、PMSM 退磁/NVH 和参数缺口。
