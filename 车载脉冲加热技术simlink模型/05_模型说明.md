## 1. 项目定位

本项目是**通用车载脉冲加热系统仿真平台**，不是具体产品仿真或特定车型仿真。模型服务于结构规范、能量路径、控制链、限制接口、KPI 语义和验证闭环。

默认参数来源：`OFFICIAL`（MathWorks 官方）、`PLATFORM_DEFAULT`（平台自洽）、`ENGINEERING_DEFAULT`（工程经验）、`PLACEHOLDER`（明确缺口）。公司历史参数和 L0.5 粗筛参数不进入默认模型。详见 `00_任务计划与资料/剥离原则与参数分层规则_v01.md`。

### 8.4 Simscape 传感器端口布局

在 R2025b 中，`fl_lib` 的 Current Sensor 和 Voltage Sensor 的物理端口布局为：

| 块 | LConn1 | RConn1 | RConn2 |
|---|---|---|---|
| Current Sensor | p (+电气) | I (物理信号输出) | n (-电气) |
| Voltage Sensor | p (+电气) | V (物理信号输出) | n (-电气) |

**RConn1 是信号端口，不是电气端口。** 连接电气 n 端时必须用 RConn2，否则会报"物理建模域连接规则"错误。

## 9. 信号验证规范（防止 Goto/From scope 类问题）

### 9.1 跨子系统 Goto/From 规则

- 新增 Goto/From 跨子系统信号时，**必须**检查 `TagVisibility`：
  - `local`（默认）：仅同一子系统内可见
  - `global`：全模型可见
- 如果 Goto 和 From 不在同一个子系统内，**必须**设为 `global`
- 每次 `model_edit` 或脚本新增 Goto 块后，用以下命令审计：
  ```matlab
  gotoBlocks = find_system(modelName, 'BlockType', 'Goto');
  for i = 1:numel(gotoBlocks)
      tag = get_param(gotoBlocks{i}, 'GotoTag');
      scope = get_param(gotoBlocks{i}, 'TagVisibility');
      fprintf('  %s: tag="%s" scope=%s\n', gotoBlocks{i}, tag, scope);
  end
  ```

### 9.2 信号验证双重确认

读取仿真结果时，**不能只依赖一种捕获方式**。以下两种方式之一为正时仍需交叉验证：

| 方式 | 优点 | 缺点 |
|---|---|---|
| Simscape log (`simlog.XXX.Y.v.series.values`) | 直接读取物理量，不经过信号链路 | 需要 `SimscapeLogType=all` |
| To Workspace / Outport | 读取 Simulink 信号链路末端 | 受 Goto/From scope、PS-Simulink Converter 连接等影响 |

**规则：当 Simscape log 和 To Workspace 结果不一致时，以 Simscape log 为准，并排查信号链路断点。**

### 9.3 零值检查

仿真后如果任何 KPI 字段或传感器读数为 0，**必须**用 Simscape log 交叉验证，不能直接假设"物理上确实是 0"。0 值的常见原因：

1. Goto/From tag scope 不匹配（local vs global）
2. PS-Simulink Converter 连接到错误的传感器端口（如 RConn1 信号端口而非 RConn2 电气端口）
3. From 块找不到匹配的 Goto 时，Simulink 静默返回 0，不报错
4. To Workspace `SaveFormat=Array` 在单步仿真时只保存最终值

| `Compute_P_inv_V4E` | MATLAB Function | `P_inv = 2*(Vf*I_avg + Ron*I_rms²)`（导通损耗，不含开关损耗） | ENGINEERING_DEFAULT |
| `Compute_P_batt_heat_V4E` | MATLAB Function | `P_batt_heat = I_pack² * R_pack`，R_pack=0.1785Ω | ENGINEERING_DEFAULT |
| `Energy_Balance_Audit` | MATLAB Function | 残差 = P_batt_terminal - (P_inv+P_cu+P_iron+P_batt_heat+P_mech) | ENGINEERING_DEFAULT |

### 10.6 已知限制

1. **P_inv 只含导通损耗**：信号链公式 `2*(Vf*I_avg + Ron*I_rms²)` 不含开关损耗；simlog 精确值（含开关损耗）可通过后处理脚本提取
2. **P_batt_heat 只含电芯内阻发热**：不含电缆连接损耗（R12/R23/R34/neg_cable/pos_cable，simlog max=0.62W）和 RC 网络损耗
3. **能量守恒残差大**：主要因为脉冲电流的瞬时功率波动大、公式估算不完整、无功功率交换；后续 V4-G/V4-I 通过 simlog 后处理做精确审计
4. **P_iron 在堵转时为 0**：这是物理正确行为（w=0 时无铁损），不是计算错误
