## 1. 项目定位

本项目是**通用车载脉冲加热系统仿真平台**，不是具体产品仿真或特定车型仿真。模型服务于结构规范、能量路径、控制链、限制接口、KPI 语义和验证闭环。

默认参数来源：`OFFICIAL`（MathWorks 官方）、`PLATFORM_DEFAULT`（平台自洽）、`ENGINEERING_DEFAULT`（工程经验）、`PLACEHOLDER`（明确缺口）。公司历史参数和 L0.5 粗筛参数不进入默认模型。详见 `00_任务计划与资料/剥离原则与参数分层规则_v01.md`。

## 2. 当前状态

`pulse_heating_official_spine_v04.slx` 是 V4-B3 MCB FOC 控制器迁移工作副本。六个顶层边界已建立；电驱边界内从 `PMSMDriveThermal.slx` 复制了核心子系统；控制器边界内从 `mcb_pmsm_foc_qep_f28379d_SIUnit.slx` 复制了 `Control_System`（含 `Closed Loop Control`、`Open Loop Start-Up`、`SVPWM`），反馈适配、3 duty -> 6 gate 适配器和参数覆盖已连通。

**V4-B3 仍不是行为模型**：Battery 为官方 48V 临时 DC 源；`I_dc`/`P_dc`/`I_rms`/`I_peak`/`P_cu`/`P_iron`/`P_inv` 仍为 NaN；电池和能量路径尚未接入；PI 增益为 MCB 默认值（为 Teknic-2310P 整定），未重新整定。

## 4. V4-B3 接口

控制器边界：
- `Idq_ref`(2×1, A) -> UnitDelay -> Control_System
- `Feedbacks_sim`(6×1) -> UnitDelay -> Demux -> Iab_meas(2×1) + Pos + EnClosedLoop=1
- Control_System -> Duty_abc(3×1) -> RelOp(>0.5) -> DTC(double) -> complementary -> 6 gate -> UnitDelay -> Duty_Cycles(6×1)
- ControlKPI = [tracking_NaN; modindex_NaN; sat_NaN; enable=1; status=3]
- DataStores: Enable=1, Speed_ref=0（堵转）

参数覆盖（InitFcn）：`pmsm.Rs=Rs; pmsm.Ld=Ld; pmsm.Lq=Lq; pmsm.p=N; pmsm.FluxPM=PM; inverter.V_dc=48; dataType='double'`

电驱边界（V4-B2 不变）：
- `Duty_Cycles`(6×1 G1-G6)、`Feedbacks_sim`(6×1)、`InverterDCFeedback`(4×1)、`DriveKPI`(8×1)
