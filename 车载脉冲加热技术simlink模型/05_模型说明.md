## 1. 项目定位

本项目是**通用车载脉冲加热系统仿真平台**，不是具体产品仿真或特定车型仿真。模型服务于结构规范、能量路径、控制链、限制接口、KPI 语义和验证闭环，不服务于某个具体车型、电池包、电机或历史频率点。

默认模型参数来源：`OFFICIAL`（MathWorks 官方案例/官方库默认）、`LITERATURE`（公开文献）、`ENGINEERING_DEFAULT`（工程经验自洽匹配）。公司历史参数和 L0.5 粗筛参数仅作为背景参考和可选外部案例，不进入默认模型。参数剥离不包含剥离 MathWorks 官方案例；相反，后续应提高官方示例骨架复用比例。

## 2. 当前结论

`pulse_heating_single_pack_v03.slx` 是当前唯一保留的 Simulink/Simscape 诊断模型。模型使用官方默认 `Battery Equivalent Circuit` 数据缩放的 `216S1P` 通用等效包，DC-link 约 801 V，闭环和直接调制均可形成 PMSM 相电流。

`pulse_heating_official_spine_v04.slx` 是 V4-A 官方案例复用骨架工作副本。它已经建立六个顶层边界：`PulseHeating_Command_And_Limits`、`MCB_SIUnits_FOC_Controller`、`BatteryThermalManagement_BatteryPack`、`DC_Link_And_Inverter_Interface`、`PMSMDriveThermal_Inverter_And_Motor`、`KPI_And_Logging`。模型内注释记录三大官方示例来源路径，边界内部使用 `0` 或 `NaN` 占位信号表达待迁移接口和未知限制，不代表低温堵转脉冲行为已验证。当前最小仿真可到 `ReachedStopTime`，但 MATLAB 会提示全为虚拟/终止/占位支路并被优化删除，因此不能把该仿真视为行为验证。

但 v03 只实现了库块级复用：电池、VSC、PMSM、传感器来自官方库，控制器和系统集成仍是手工搭建。它不能等同于复用了 MathWorks 成熟 PMSM drive/FOC/热管理示例。后续主线改为 v04 官方案例骨架复用，详见 `00_任务计划与资料/V4_官方案例复用建模规划_v01.md` 和 `00_任务计划与资料/官方案例复用资产清单与重构路线_v01.md`。

Codex MATLAB MCP 只读验证已完成：`PMSMDriveThermal.slx`、`mcb_pmsm_foc_qep_f28379d_SIUnit.slx`、`BatteryThermalManagementModel.slx` 均可加载，`model_overview(tree)` 可读，`model_check(all)` 为 `healthy`。

v03-B2 已确认此前电流卡死不是控制器或 VSC/PMSM 本体问题，而是旧 Battery Pack Builder 生成资产参数化/初始化导致 DC-link 近零电压。旧 `battery_pack_252S1P` 生成资产、`pulse_heating_single_pack_v01.slx`、`pulse_heating_v02.slx` 和旧 KPI/参数脚本已从当前工作树删除；如需追溯，使用 Git 历史提交 `e23e0dc`。

v03-B3 已把控制链从 v03-B2 的饱和主导状态调整为非饱和审计基线：小信号方向为正向，`0 A` 工况正常，`5/20/100/200 A` 闭环工况未触及调制饱和；但高频诊断工况下跟踪误差仍高，当前不能声称控制器已完成整定。剩余主要问题是高频跟踪策略、限制接口、电池低温 EIS/HPPC 和 BMS 边界缺失；这些问题应迁移到 v04 官方案例骨架上解决。
