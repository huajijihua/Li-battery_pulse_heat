# 4500-27整车侧会前预读

日期：2026-06-15  
对象：VCU/整车控制、整车系统集成、高压架构、热管理、标定/试验  
用途：4500-27低温脉冲加热整车侧专题对接会前预读  

## 1. 当前方案对象

- 车型对象：`4500-27`
- 高压平台：`811.44 V`
- 电池侧口径：`3个252S/278Ah电池包并联整体输出，非独立输出`
- 电驱口径：`双电机脉冲`
- 当前场景：`驻车堵转预热`
- 当前模型级别：`L0.5零维粗筛`

## 2. 当前最好结果

在当前占位假设下，双电机主方案表现出值得继续验证的温升量级：

- 初始温升速率约 `1.43 C/min`
- 30 min平均温度约 `9.01 C`
- 到 `0 C` 时间约 `17.68 min`
- 到 `0 C` 总不可逆损耗等效SOC消耗约 `3.58%`

这组结果的含义只是：

- 当前方案在粗筛层面存在继续验证价值；
- 值得进一步问清是否存在整车功能接入路径。

这组结果不代表：

- 已证明整车一定允许该功能接入；
- 已证明现有 `VCU-BMS-MCU-高压-热管理` 执行链已经闭环；
- 已证明该方案已经值得装车或立项开发。

## 3. 当前最强约束

本次交流最需要整车侧帮助闭合的，不是原理，而是功能接入边界：

1. `VCU` 功能入口  
   当前项目最关键的整车问题，不是“理论上能不能把电池加热起来”，而是“整车是否愿意、也是否允许把它接成一个真正可执行的功能”。

2. 高压执行路径  
   当前零维模型没有显式建接触器、高压盒、驻车高压保持和执行链，当前拓扑不等于整车可执行方案。

3. 热管理协同  
   当前热边界仍是占位，驻车预热时泵阀、冷却液、散热抑制和附件功耗是否配合，直接影响整车收益是否还能成立。

4. 诊断与降级责任链  
   当前没有把故障报码、绝缘监测、退出条件和降级回退链条建进去，后续要不要进台架，取决于整车侧是否能给出最小安全闭环。

## 4. 当前明确不能声明的结论

为避免会议上把粗筛结果误读成整车功能已经成立，当前有三类话不能讲：

- 不能说整车一定允许该功能接入；
- 不能说当前结果已证明可直接装车；
- 不能说现有控制链、高压链和热管理链已经闭环。

## 5. 这次会议希望拿到什么

这次会议希望至少形成下面4类明确结论：

1. 哪些整车进入条件、退出条件和系统约束可直接引用到下一轮模型或评审。
2. 哪些整车控制口径允许先按保守假设占位推进。
3. 哪些问题必须补策略说明、补试验或补系统联调。
4. 是否存在一票否决项，例如 `VCU` 无功能入口、驻车高压路径不允许或整车收益不足不值得继续。

## 6. 我方当前立场

我们当前不是想把粗筛结果直接解释成整车功能已经成立，也不是想跳过整车控制和系统集成边界去推装车。  
当前目标只有一个：先判断在现有整车功能架构、高压执行路径、热管理协同和诊断降级边界内，4500-27利用现有三包并联和双电机硬件做低温脉冲加热，是否还存在值得继续验证的整车功能窗口。
