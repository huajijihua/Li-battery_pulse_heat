## 1. 使用说明

这张表不是系统资料清单，也不是泛泛的整车问题列表。它是整车侧会议对接表，目标只有三个：

1. 说明当前L0.5模型怎样映射到整车功能和整车约束；
2. 说明哪些整车边界现在只是占位，不能直接说成整车已闭环；
3. 逐项确认VCU/整车控制与整车集成/热管理需要给出什么判断、证据和后续动作，才能决定是否进入L1控制可实现性模型和后续验证。

会议记录时，每一行至少要填出以下4类结论之一：

- `已确认`：可直接引用到下一轮模型或评审；
- `可占位`：允许暂用保守口径推进，但必须显式标风险；
- `需补试验`：当前口头判断不够，必须补策略、补试验或补系统说明；
- `否决项`：若该项结论为否，当前主方案不能继续按既有主线推进。

## 2. 当前主线口径

- 当前模型级别：`L0.5零维粗筛`
- 当前主方案：`三包并联整体输出 + 双电机同步堵转脉冲`
- 当前整车映射口径：电池三包并联先汇入公共高压母线，双逆变器/双电机从同一母线取能，`VCU-BMS-MCU-热管理` 是否给出功能入口当前未显式建模
- 当前功能解释口径：当前结果只说明该方案在粗筛层面有温升收益，不等于该功能在整车上已有状态机入口和执行链
- 当前结论边界：可用于判断温升量级、同步价值和是否值得进入整车功能链验证；不能直接证明整车一定允许、现有高压执行链已闭环、可直接上车

## 4. 每个主题统一提问模板

每个主题块都按同一顺序问，避免讨论发散：

1. 我们现在怎么假设？
2. 这个假设最可能错在哪？
3. 你们是否认可先这样占位？
4. 如果不认可，最小替代证据是什么？
5. 如果短期拿不到精确结论，允许的保守口径是什么？
6. 在你们看来，这一项的红线是什么？

其中第4问和第5问必须同时保留，不能只问“有没有完整策略文档”。

## 5. 会中记录栏建议

建议会议时在每一行后面补记以下短字段，便于会后归档：

- `会议判断`：已确认 / 可占位 / 需补试验 / 否决项
- `口头结论摘要`：一句话写清对方本次判断
- `补证责任人`
- `补证形式`：数据 / 试验 / 规则说明 / 接口说明
- `补证截止时间`

## 6. 会后判定标准

会后这张表至少要形成4类明确结果：

### 6.1 可直接引用的整车边界

- 例如部分进入条件、退出条件、固定高压约束、固定场景限制

### 6.2 允许临时占位的整车口径

- 例如暂不建全状态机，但允许先按某些使能链和保守热管理口径做L1约束建模

### 6.3 必须补策略或补试验的整车问题

- 例如驻车高压执行链、故障降级逻辑、热管理协同、整车收益验证

### 6.4 一票否决项

- 例如 `VCU` 无功能入口、驻车高压路径不允许、整车收益不足不值得继续立项

## 7. 这张表完成后的自检

1. 每一行都对应当前4500-27模型、阶段结论或已有汇报口径中的一个具体假设，而不是泛泛整车问题。
2. 每一行都同时写了 `当前不能声明` 和 `不确认的影响`。
3. `VCU/整车控制` 与 `整车集成/热管理` 的责任必须分开写，不能把功能入口和热管理协同混成同一责任描述。
4. 会后每一行都能归入 `已确认 / 可占位 / 需补试验 / 否决项` 四类之一。
5. 即使对方只给口头判断不给完整文档，表里也能记录“是否接受占位口径”和“后续补证责任人”。
