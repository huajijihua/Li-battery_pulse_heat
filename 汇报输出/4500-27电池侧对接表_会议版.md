# 4500-27电池侧对接表

版本：v1  
日期：2026-06-15  
适用对象：电池系统研发、BMS/控制  
使用场景：4500-27低温脉冲加热电池侧专题对接会  
当前对象：三包并联整体输出、双电机脉冲、811.44 V平台  

## 1. 使用说明

这张表不是参数堆料表，而是会议对接表。用途只有三个：

1. 说明当前L0.5模型已经怎样使用电池侧信息；
2. 说明哪些边界现在只是占位，不能直接说成工程结论；
3. 逐项确认电池系统研发和BMS/控制需要给出什么判断、证据和后续动作，才能决定是否进入L1电流可实现性模型和后续验证。

会议记录时，每一行至少要填出以下4类结论之一：

- `已确认`：可直接引用到下一轮模型或评审；
- `可占位`：允许暂用保守口径推进，但必须标风险；
- `需补试验`：当前口头判断不够，必须补数据或补验证；
- `否决项`：若该项结论为否，当前方案不能继续按主线推进。

## 2. 当前主线口径

- 当前模型级别：`L0.5零维粗筛`
- 当前主方案：`三包并联整体输出 + 双电机脉冲`
- 当前电池发热口径：`I_rms^2 * R_bat(T,SOC)`，并用 `R_heat_factor` 对高频阻抗做敏感性占位
- 当前热边界口径：集总热容 + 弱对流，默认 `h = 5 W/m2/K`
- 当前结论边界：可用于判断温升量级、等效SOC消耗、电流压力和方案排序；不能直接证明高频安全、BMS允许、实车可装车

## 4. 每个主题统一提问模板

每个主题块都按同一顺序问，避免讨论发散：

1. 我们现在怎么假设？
2. 这个假设最可能错在哪？
3. 你们是否认可先这样占位？
4. 如果不认可，最小替代证据是什么？
5. 如果短期拿不到精确数据，允许的保守口径是什么？
6. 在你们看来，这一项的红线是什么？

其中第4问和第5问必须同时保留，不能只问“有没有数据”。

## 5. 会中记录栏建议

建议会议时在每一行后面补记以下短字段，便于会后归档：

- `会议判断`：已确认 / 可占位 / 需补试验 / 否决项
- `口头结论摘要`：一句话写清对方本次判断
- `补证责任人`
- `补证形式`：数据 / 试验 / 规则说明 / 接口说明
- `补证截止时间`

## 6. 会后判定标准

会后这张表至少要形成4类明确结果：

### 6.1 可直接引用的边界

- 例如部分低温窗口、部分热边界、部分BMS固定限值

### 6.2 允许临时占位的边界

- 例如暂用 DCIR、暂用保守热边界、暂不建动态保护但必须显式标风险

### 6.3 必须补数据或补试验的边界

- 例如 `Re(Z,T,SOC,f)`、低温高频回充接受能力、并联支路保护逻辑

### 6.4 一票否决项

- 例如电池安全侧或BMS侧明确禁止驻车高频双向脉冲

## 7. 这张表完成后的自检

1. 每一行都对应当前4500-27模型中的一个具体假设或输出，而不是泛泛问题。
2. 每一行都同时写了 `当前不能声明` 和 `不确认的影响`。
3. 每一行都区分了 `电池系统研发` 与 `BMS/控制` 的责任。
4. 会后每一行都能归入 `已确认 / 可占位 / 需补试验 / 否决项` 四类之一。
5. 即使对方只给口头判断不给数据，表里也能记录“是否接受占位口径”和“后续补证责任人”。
