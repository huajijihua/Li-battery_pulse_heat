## 1. 使用说明

这张表不是参数堆料表，也不是一般性的“要资料清单”。它是电驱/电机侧会议对接表，目标只有三个：

1. 说明当前L0.5模型已经如何使用电驱/电机侧信息；
2. 说明哪些边界现在只是占位，不能直接说成工程已闭环；
3. 逐项确认电驱/控制侧与电机/可靠性侧需要给出什么判断、证据和后续动作，才能决定是否进入L1电流可实现性模型和后续验证。

会议记录时，每一行至少要填出以下4类结论之一：

- `已确认`：可直接引用到下一轮模型或评审；
- `可占位`：允许暂用保守口径推进，但必须显式标风险；
- `需补试验`：当前口头判断不够，必须补数据、补试验或补控制逻辑说明；
- `否决项`：若该项结论为否，当前主方案不能继续按既有主线推进。

## 2. 当前主线口径

- 当前模型级别：`L0.5零维粗筛`
- 当前主方案：`三包并联整体输出 + 双电机同步堵转脉冲`
- 当前电流计算口径：母线电压、电机绕组电阻、电机d轴电感、频率和占空比形成 `R-L` 回路电流
- 当前控制解释口径：既要区分“原始电压驱动能力”，也要区分“目标电流是否可由逆变器实现”
- 当前结论边界：可用于判断温升量级、电流压力、同步风险和是否值得进入L1；不能直接证明 `MCU一定允许`、`550 Arms可长时持续`、`电机/逆变器不会过热或退化`

## 4. 每个主题统一提问模板

每个主题块都按同一顺序问，避免讨论发散：

1. 我们现在怎么假设？
2. 这个假设最可能错在哪？
3. 你们是否认可先这样占位？
4. 如果不认可，最小替代证据是什么？
5. 如果短期拿不到精确数据，允许的保守口径是什么？
6. 在你们看来，这一项的红线是什么？

其中第4问和第5问必须同时保留，不能只问“有没有数据”。

## 5. 会中记录栏建议

建议会议时在每一行后面补记以下短字段，便于会后归档：

- `会议判断`：已确认 / 可占位 / 需补试验 / 否决项
- `口头结论摘要`：一句话写清对方本次判断
- `补证责任人`
- `补证形式`：数据 / 试验 / 规则说明 / 接口说明
- `补证截止时间`

## 6. 会后判定标准

会后这张表至少要形成4类明确结果：

### 6.1 可直接引用的边界

- 例如控制使能条件、部分电流/频率边界、部分已确认的可靠性红线

### 6.2 允许临时占位的边界

- 例如暂用保守 `Ld/Rs` 口径、暂用控制降额系数、暂不建器件热动态但必须显式标风险

### 6.3 必须补数据或补试验的边界

- 例如最小脉宽/死区/采样边界、双电机同步精度、逆变器热边界、退磁/NVH约束

### 6.4 一票否决项

- 例如 `MCU` 明确禁止零速高频受控电流模式，或电机/逆变器侧明确判定该工况不可接受

## 7. 这张表完成后的自检

1. 每一行都对应当前4500-27模型中的一个具体假设、参数或敏感性结论，而不是泛泛问题。
2. 每一行都同时写了 `当前不能声明` 和 `不确认的影响`。
3. `电驱/控制侧` 与 `电机/可靠性侧` 的责任必须分开写，不能把控制权限和退磁/NVH风险混成同一责任描述。
4. 会后每一行都能归入 `已确认 / 可占位 / 需补试验 / 否决项` 四类之一。
5. 即使对方只给口头判断不给数据，表里也能记录“是否接受占位口径”和“后续补证责任人”。
