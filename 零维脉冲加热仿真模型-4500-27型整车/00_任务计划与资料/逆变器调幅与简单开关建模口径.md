# 逆变器调幅与简单开关建模口径

## 1. 结论

在脉冲加热模型中，不能把“简单开关”和“电机控制器/逆变器”完全等同。

简单开关模型的输入是电池电压、频率、占空比、回路电阻和电感；电流幅值是电路自然响应，不能直接指定。逆变器模型的输入可以是电流目标、d/q轴电流指令、调制深度或电压矢量；电流幅值由电流闭环调节，但必须受母线电压、调制范围、开关器件、电机、电池和控制带宽限制。

因此，本项目后续所有模型应同时保留两类结果：

- 原始电压驱动能力：在给定电压、频率、占空比和RL参数下，电流自然能达到多少。
- 受控电流结果：在给定目标电流下，逆变器能否通过调制实现；如果实现不了，由哪个边界限制。

## 2. 简单开关与逆变器的本质区别

| 项目 | 简单开关/开环方波 | 逆变器/电机控制器 |
|---|---|---|
| 控制对象 | 开关通断、频率、占空比 | 电压矢量、PWM、d/q轴电流、限流策略 |
| 电流幅值 | 由电池电压、R、L、频率、占空比自然决定 | 由电流目标和闭环控制决定，但受电压和硬件限制 |
| 是否能直接调幅 | 不能直接调，只能间接改频率/占空比/电压 | 可以调目标电流，但实际值可能饱和或触发保护 |
| 模型风险 | 容易高估电流不可控风险 | 容易低估电压饱和、控制延迟、器件热风险 |
| 适用阶段 | 原理粗筛、开环电流能力边界 | 控制策略、实车可实现性、保护逻辑评估 |

简单开关模型回答“这个电路自然会流多大电流”。逆变器模型回答“控制器想让它流这么大电流，实际能不能跟上，以及代价是什么”。

## 3. 三类模型中的推荐处理方式

### 3.1 MATLAB零维电路模型

零维模型应分成两个层级。

第一层是开环电压驱动：

- 输入：`Vdc`、`R_loop`、`L`、`f_sw`、`duty`。
- 输出：自然形成的 `I_peak_raw`、`I_rms_raw`、`I_pp_raw`。
- 用途：判断频率、占空比、电感、电阻对原始电流能力的影响。
- 口径：此时不能说“直接把电流调小”，只能说“通过改变频率、占空比或等效施加电压改变电流”。

第二层是目标电流/限流口径：

- 输入：`I_target` 或 `current_amplitude_scale`。
- 物理含义：代表控制器给出的电流目标或调制降额，不代表凭空把电流乘一个系数。
- 计算方式：先算开环可达电流，再按目标电流和限制边界得到实际电流。
- 输出必须同时给出：原始电流、目标电流、实际电流、有效缩放系数、限制来源。

推荐在零维模型中把 `current_amplitude_scale` 明确解释为“电流目标系数/调制降额系数”。如果后续参数更完整，应进一步替换为 `I_target_A`、`modulation_index` 或 `V_cmd`，并检查所需电压是否超过母线和调制上限。

### 3.2 COMSOL电路结构模型

COMSOL模型应按目的选择激励源，避免用一个理想源覆盖所有问题。

原理验证阶段可用受控脉冲电压源：

- 电路结构：电池OCV、支路内阻、电机等效电阻、电机电感、控制器等效脉冲电压源串联。
- 调节量：频率、占空比、等效输出电压幅值。
- 适用问题：观察电路瞬态、电流纹波、单支路发热和热耦合。
- 边界：这是开环电压驱动，不等于实际控制器已具备任意电流闭环能力。

如果要表达逆变器调幅，应使用受控电压源或电流闭环外层：

- 用电流目标与实测电流误差生成 `V_cmd`。
- 对 `V_cmd` 加母线电压、调制深度、最大/最小占空比和斜率限制。
- 最终仍用受限后的电压去驱动RL回路，而不是直接用理想电流源替代功率回路。

只有在做热量上限或控制目标扫描时，才可以临时使用受控电流源；但必须标注为“理想电流源假设”，不能用于证明实际逆变器一定能实现该电流。

### 3.3 Simulink/Simscape电路结构模型

Simulink模型应优先体现“目标电流 -> 控制器 -> PWM/电压矢量 -> 逆变器 -> 电机绕组电流”的链条。

推荐分层：

- L0：理想开关或平均值逆变器，用于快速验证电流闭环和保护逻辑。
- L1：平均值逆变器加电压饱和、采样延迟、PWM限幅，用于系统级联调。
- L2：IGBT/MOSFET开关级模型，加入死区、导通压降、开关损耗和器件热模型，用于最终损耗与风险校核。

Simulink中调节脉冲电流幅值时，推荐使用：

- `Id_ref` 或 `I_rms_ref` 作为控制目标。
- PI/PID电流环输出 `Vd_ref`、`Vq_ref` 或调制指令。
- PWM/SVPWM模块把电压指令转换成桥臂驱动。
- 逆变器和绕组物理网络产生实际电流。
- 保护模块独立监控峰值电流、RMS电流、母线电压、器件温度、电池边界和故障状态。

不建议在完整Simulink电气模型中直接用“理想电流源”代替逆变器输出，除非该模型只用于上层热模型或控制目标的离线估算。

## 4. 电流幅值调节应受哪些限制

逆变器可以调电流目标，但实际电流必须满足以下限制：

- 母线电压限制：所需绕组电压不能超过当前电池母线和逆变器可调制范围。
- 占空比/调制限制：存在最大调制深度、最小脉宽、死区时间和PWM分辨率。
- 电流环带宽限制：采样、计算、PWM更新和滤波会限制高频脉冲跟踪能力。
- 器件电流限制：IGBT/MOSFET、二极管、母线电容、铜排和线束有峰值、RMS和热限制。
- 电机限制：绕组温升、d轴电感饱和、永磁体退磁、转矩纹波和NVH风险。
- 电池限制：低温放电/回充能力、端电压跌落/抬升、析锂风险、SOC和BMS限流。
- 高压系统限制：继电器、接触器、熔断器、高压盒路径和支路不均流能力。

模型中只要引入“目标电流”，就必须同时输出“是否达到目标”和“未达到时的限制来源”。

## 5. 本项目推荐的统一建模口径

后续无论使用MATLAB、COMSOL还是Simulink，脉冲电流调控均按以下顺序处理：

1. 先定义拓扑和真实电流路径。
2. 先算开环电压驱动下的原始电流能力。
3. 再加入目标电流或调制降额。
4. 用逆变器、电机、电池和高压系统边界限制实际电流。
5. 输出原始电流、目标电流、实际电流、限制来源和热结果。

推荐统一输出字段：

| 字段 | 含义 |
|---|---|
| `f_sw_Hz` | 脉冲/PWM频率 |
| `duty` 或 `modulation_index` | 占空比或调制深度 |
| `I_raw_peak_A`、`I_raw_rms_A` | 开环电压驱动自然形成的电流 |
| `I_target_A` 或 `current_amplitude_scale` | 控制器目标电流或调制降额 |
| `I_actual_peak_A`、`I_actual_rms_A` | 限制后的实际电流 |
| `V_required_V` | 达成目标电流所需等效电压 |
| `V_available_V` | 当前母线和调制边界下可用电压 |
| `limiting_factor` | 限制来源，如电压饱和、MCU限流、电池限流、频率越界 |
| `P_battery_W`、`P_motor_W`、`P_inverter_W` | 电池发热、电机铜耗、逆变器损耗 |

## 6. 对当前零维模型的解释边界

当前4500-27零维模型中的 `current_amplitude_scale` 可以继续用于方案粗筛，但报告中应按以下口径解释：

> `current_amplitude_scale` 表示控制器目标电流或调制降额后的电流幅值系数，用于研究电流幅值变化对加热能力和安全边界的影响。它不是额外串联限流设备，也不是电路自然规律之外的任意缩放。最终可实现电流仍需由母线电压、逆变器调制能力、MCU电流限制、电机绕组和电池安全边界共同约束。

后续如果进入更细建模，应把这一系数拆成两个更物理的量：

- `modulation_index`：控制器实际施加电压能力。
- `I_target_A`：电流闭环希望达到的目标。

这样可以区分“电压给少了所以电流小”和“给了目标电流但硬件达不到”。

## 7. 风险提示

不能只用“电池电压/回路内阻”判断实际车上电流，因为实际中间有逆变器和电流闭环。也不能只用理想电流源证明方案可行，因为理想电流源会掩盖母线电压不足、PWM调制饱和、器件过热、BMS限流和控制跟踪失败。

本项目最稳妥的路径是：零维模型用于快速筛选频率、幅值和拓扑；COMSOL用于局部电路和热结构校核；Simulink用于验证逆变器控制、保护逻辑和可实现性。三者的电流调控变量可以不同，但必须共享同一个原则：电流目标不能脱离电压驱动能力和硬件边界。
