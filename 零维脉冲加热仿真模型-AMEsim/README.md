# AMESim 低温脉冲加热模型入口

日期：2026-07-03

本目录是车载动力电池-电机低温脉冲加热 AMESim 主线模型的预留目录。当前尚未形成可运行、已标定的 `.ame` 模型；后续任何自动生成或手工搭建的模型，先标记为结构骨架或工作副本。

## 建模目标

AMESim 侧优先承担一维工程系统骨架和部件级物理网络表达，包括：

1. 三包并联整体输出的电池等效电路和热模型；
2. 高压母线、DC-link、逆变器等效损耗和电机 R-L 支路；
3. 双电机同步、错相、轮换或降额策略的接口；
4. 电池发热、电机/逆变器损耗、支路电流、母线电流、温升和保护标志的 KPI 审计；
5. 与 MATLAB/Simulink 的参数、工况、控制量和结果接口。

## 首批产物建议

在创建 `.ame` 前，先补齐以下文本或表格资产：

1. `architecture_register.md`：系统架构、模块、连接关系和边界；
2. `parameter_register.xlsx` 或 `parameter_register.md`：参数名、单位、来源、可信等级和待确认项；
3. `variable_kpi_register.md`：变量、KPI、单位、输出位置和验收口径；
4. `study_matrix.md`：默认工况、敏感性工况、边界工况和失败工况；
5. `interface_contract.md`：与 MATLAB/Simulink 或外部脚本交换的参数、工况、控制量和结果格式。

## 关键约束

1. 不硬编码历史 demo 的 submodel、参数名、变量名、端口编号或 Data Path。
2. 变量名、端口、submodel 和 Data Path 必须从当前 AMESim 安装环境或当前模型动态发现。
3. 自动生成的 `.ame`、`.cir`、`.sim` 和结果文件必须说明来源、用途和成熟度。
4. 不能把 0D/L0.5 MATLAB 经验函数直接冒充 AMESim 物理网络。
5. 未完成参数核对、结构检查和最小工况验证前，不得把 AMESim 模型写成已标定或可交付模型。
