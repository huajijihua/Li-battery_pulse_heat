# M5 v2 准入与接口契约 v01

日期：2026-07-22

状态：`M5-00=COMPLETE`；`M5-01=ADMITTED, NOT_STARTED`；v2 唯一 `.slx` 尚未创建。除非用户明确启动 M5-01，否则不创建模型、脚本或结果。

本契约是 `车载脉冲加热仿真平台_v2/` 的唯一初始接口与参数准入入口。M4 的正式路线名称为 **CMI（Combined Module Invert，文献原称）架构下的开环低频 d 轴脉冲电流自加热路线**；M5 首版的正式路线名称为 **dSVPWM（dead-zone Space Vector PWM，死区空间矢量脉宽调制）换流型脉冲自加热路线**。本契约继承 M4 的账本语义和研究纪律，但不复制 M4 的控制方式。

## 0. 路线命名与连续性

M4 和 M5 研究的是同一条电池-DC-link-逆变器-PMSM-热响应路径。M4 通过低频 `Id_ref` 参考建立 Reference / Platform 基线并识别电池侧有效电流过低卡点；M5 在相同首版硬件边界内改用 dSVPWM 死区换流，直接验证电池侧正负交变电流形成机制。M4 名称中的“开环”仅限定加热参考层，M5 名称中的 `d` 表示 `dead-zone`，不是 d 轴。

具体换流解释、文献依据和 `bn/d0/a0` 参数化边界以 `00_项目管理/M5_文献调研与差异分析_v01.md` 第 8 节为准。

## 1. 首版系统边界

| 项目 | 首版约束 |
|---|---|
| 硬件 | 单包、三相三线、标准三相两电平逆变器、PMSM、DC-link。 |
| 机械工况 | 零速锁止作为首个诊断边界；行驶工况在控制接口验证后扩展。 |
| 主控制 | dSVPWM（死区空间矢量脉宽调制）换流型调制族，电压快速约束与温度慢速监督分层。 |
| 不纳入默认拓扑 | 电池中点、N 线、NPBH 开关、DMSI 电池分组。 |
| 不纳入默认结论 | 具体设备额定值、析锂安全、寿命、量产或法规结论。 |

## 2. M4 继承与排除

| 类别 | v2 处理 |
|---|---|
| 物理路径 | 迁移电池、高压路径/DC-link、逆变器、PMSM、热接口和必要机械约束的已验证系统边界。 |
| 审计口径 | 迁移 `I_batt`、`V_batt`、DC-link、相电流、`i_d/i_q`、温度、SOC、实际门极和固定窗口账本的区分与复算规则。 |
| 参数纪律 | 继承 `OFFICIAL`、`LITERATURE`、`ENGINEERING_DEFAULT`、`PLATFORM_DEFAULT`、`PLACEHOLDER` 及未知项传播规则。 |
| 不继承控制 | 不迁移 M4 的 CMI 架构下开环低频 `Id` 脉冲生成器、其频率/占空比/幅值结论或加热能力推断。 |
| 不迁移结论 | 不将 M4 的短时热账本、CMI 电流卡点或历史扫描升级为 v2 性能、安全或产品结论。 |

## 3. 信号接口契约

| 接口 | 角色 | M5 语义与准入要求 |
|---|---|---|
| `PWM_Gates[1:6]` | 调制层物理输出 | 是实际六路门极，不用命令侧 duty 替代；桥臂 `(1,2)`、`(3,4)`、`(5,6)` 必须独立检查互补和无重叠。 |
| `bn`、`d0`、`a0` | 主动控制候选 | 是 dSVPWM 调制执行量/约束，不直接等同于 `Id_ref`；首版应选择一个主执行量，其余作为约束、派生量或独立对照；每个量须有范围、来源和饱和语义。 |
| `V_batt`、`I_batt` | 电池端响应 | 保持 M4 功率正方向和固定窗口口径；M5 需以物理测量路径读回，不能由相电流或 RMS 代理替代。 |
| `V_dc`、`I_dc` | DC-link 响应 | 与电池端和相侧分别记录；用于电气子账本和换流路径审计。 |
| `i_abc`、`i_d`、`i_q`、转矩、转速 | PMSM 响应 | 与电池加热电流分开表达；零速锁止不等于行驶转矩解耦已验证。 |
| `T_pack`、`SOC`、环境/冷却边界 | 热/状态接口 | 初始值、单位、来源和适用范围必须显式记录；无目标数据时保留 `Reference / Platform`。 |
| `Z_batt(T,SOC,f)`、`Re(Z)` | 电热模型候选 | 使用前必须记录波形假设和来源；PWM/脉冲需谐波等效或时域交叉检查。 |
| `U_neg_est`、`f_boundary`、限流值 | 安全候选输出 | 只能由合格的负极/全电池数据和已登记假设生成；不得由全电池二阶 ECM 直接推导安全结论。 |
| `Energy_Ledger_Valid`、`Electrical_Ledger_Valid`、`DeviceProtection_Evaluable` | 审计状态 | 三者分别表达账本窗口与安全资格，不可互相替代。 |

## 4. 参数准入与来源登记

| 参数族 | 初始来源状态 | 准入规则 |
|---|---|---|
| M4 物理与账本基线 | `OFFICIAL` / `ENGINEERING_DEFAULT` | 仅作迁移基线；保留单位、来源和替换影响，不能写成具体设备参数。 |
| dSVPWM（死区空间矢量脉宽调制）换流机理 | `LITERATURE:Templock2024_*`、`LITERATURE:2024_IEEE_TTE_*` | 可定义结构和接口；频率、死区、占空比及电压阈值须重新扫描/整定。 |
| IEEE 双 PI 增益 | `LITERATURE_CONFLICT` | 正文 `Kp=0.01` 与 Table II `Kp=0.1` 冲突；禁止作为 v2 默认值，`Ki=0.9` 和 `Tc=1e-4 s` 也仅作参考。 |
| 频域热模型 | `LITERATURE:Zhang2015_*`、`LITERATURE:NPBH2026_*` | 结构可用，具体 EIS/ECM 数值须匹配目标电芯或明确标为平台假设。 |
| 安全候选 | `LITERATURE:Jian2023_*` | 需目标化学体系、负极阻抗/电位映射、SOC/温度范围和波形等效条件。 |
| 设备参数 | `UNKNOWN` 或 `PLACEHOLDER` | 无用户设备数据时不得转为额定值、包线或安全阈值。 |

## 5. 安全状态契约

| 状态 | 条件 | `DeviceProtection_Evaluable` |
|---|---|---|
| `UNKNOWN` | 缺少目标化学体系、EIS、负极映射、波形校核或适用范围。 | `0` |
| `CANDIDATE` | 已运行 NEPCS/TVCS 数学路径，但依赖文献比例或平台假设。 | `0` |
| `EVALUABLE` | 来源、单位、温度/SOC 范围、波形等效和独立验证全部满足。 | `1` |

首版初始化必须处于 `UNKNOWN`。任何有限的 `U_neg_est` 数值都不改变该状态。
